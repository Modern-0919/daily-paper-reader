## 摘要
幻觉是基于LLM的智能体的可靠性瓶颈，而事实归因验证器是最后一道防线——然而当前的验证器仅输出不透明的二元标签，使得智能体无法自我纠正，操作员也无法审计。我们提出SEVA，一种结构化验证智能体，它输出证据对齐、逐步推理链、校准后的置信度，以及带有可操作修复措施的六类错误诊断。使用强化学习训练这样的智能体并非易事：对多组件输出使用标准二元奖励会引发优势坍塌——组内奖励方差消失，GRPO梯度消失。我们通过一种过程奖励来解决这个问题，该奖励将验证质量分解为五个独立组件，以70/30的权重偏向过程信号，从而恢复梯度并引发隐式课程——智能体首先掌握验证行为（对齐度0.917->0.997，格式72%->100%），然后掌握结果（F1 64.9->69.0）。结构化输出进一步实现了验证->反思->探查->优化的自进化循环，在7B模型上经过四轮后揭示了一个意外的结构性发现：每一轮产生一个基准测试专家，而非通才模型（在同一模型上，HaluEval提升15个百分点，TruthfulQA下降10到14个百分点，在4倍数据量下持续存在）。在ClearFacts上，SEVA-3B与GPT-4o-mini性能相当（F1分数69.0 vs 69.8），同时产生更丰富、可审计的输出——确认了一个应具有普适性的原理：对于任何具有多组件生成的强化学习任务，奖励粒度必须与输出粒度匹配。

## Abstract
Hallucination is the reliability bottleneck for LLM-based agents, and fact attribution verifiers are the last line of defense -- yet today's verifiers emit only opaque binary labels, leaving agents unable to self-correct and operators unable to audit. We present SEVA, a structured verification agent that emits evidence alignments, step-by-step reasoning chains, calibrated confidence, and a six-category error diagnosis with actionable fixes. Training such an agent with RL is non-trivial: standard binary reward on multi-component output triggers advantage collapse -- within-group reward variance vanishes and the GRPO gradient disappears. We resolve this with a process reward that decomposes verification quality into five independent components weighted 70/30 toward process signals, restoring the gradient and inducing an implicit curriculum -- the agent first masters verification behavior (alignment 0.917 -> 0.997, format 72% -> 100%), then outcomes (F1 64.9 -> 69.0). Structured output further enables a Verify -> Reflect -> Probe -> Refine self-evolution loop, which over four rounds on a 7B model surfaces an unexpected structural finding: each round produces a benchmark-specialist, not a generalist (+15 pp on HaluEval, -10 to -14 pp on TruthfulQA in the same model, persistent at 4x data). On ClearFacts, SEVA-3B matches GPT-4o-mini (69.0 vs. 69.8 F1) while producing substantially richer, auditable output -- confirming a principle that should generalize: for any RL task with multi-component generation, reward granularity must match output granularity.

# 论文总结：SEVA

## 1. 核心问题与整体含义
- **研究动机**：现有事实归因验证器（如 MiniCheck、ClearCheck）仅输出二元标签（Attributable / Not Attributable），导致两个关键问题：
  1. 被验证的 LLM 智能体无法自我纠正（只知道错误，不知道错在哪里）；
  2. 人工操作员无法审计验证决策背后的推理过程。
- **整体含义**：不透明的验证器在金融、法律、医疗等高风险场景中无法提供可靠的安全保障。作者提出 SEVA (Self-Evolving Verification Agent)，通过结构化输出和过程奖励解决这一问题，使验证既可审计又可通过强化学习进行有效训练。

## 2. 方法论
- **核心思想**：将验证输出从单一二进制标签扩展为结构化描述，并设计一种**过程奖励**来替代标准二进制奖励，避免 GRPO 训练中出现的优势坍塌。
- **关键技术细节**：
  - **结构化输出 schema**：包含四个组件：
    - 证据对齐 `A`（claim spans ↔ source spans，状态：match / mismatch / not found）；
    - 推理链 `C`（逐步判断：supported / not supported / partially supported，附带自然语言解释）；
    - 标签 `y` + 校准置信度 `γ`；
    - 错误诊断（当标签为 Not Attributable 时）：分为六类错误（数值夸大、否定翻转、范围膨胀、时间偏移、实体替换、捏造）并给出修复建议。
  - **训练流程**：
    - 第一阶段：SFT。使用 GPT-4o-mini 自动标注 4,992 条 ANLI 样本生成结构化输出，微调 Qwen2.5-3B-Instruct（3 epochs，lr=2e-5）。
    - 第二阶段：GRPO 强化学习。基于过程奖励进行训练（G=8，T=1.2，β=0.001，lr=2e-6，5 epochs ≈ 350 steps）。
  - **过程奖励公式**：
    \[
    R = w_f R_f + w_a R_a + w_c R_c + w_l R_l + w_d R_d + R_{cal}

- **理论贡献**：

## 6. 主要结论与发现
- **过程奖励使 GRPO 在结构化输出上可行**：3B SEVA-GRPO 在 ClearFacts 上 F1 达 69.0（+4.1 vs SFT），匹配 GPT-4o-mini 的 69.8，同时输出更丰富、可审计的结构化结果。
- **隐式课程学习**：训练过程中，对齐和格式等行为指标在前 150 步饱和，而 F1 继续提升至 350 步，表明模型先学会“如何验证”，后学会“预测什么”。
- **自我进化产生专家而非通才**：在 7B 模型上四轮自进化后，HaluEval F1 提升约 +15 pp，TruthfulQA F1 下降约 -12 pp，且该不对称性在 4× 数据量下依然保持，说明探针分布驱动了专业化。
- **通用原则**：对于多组件输出的 RL 任务，奖励粒度必须匹配输出粒度；结构化输出既是审计资产也是训练诊断工具。

## 7. 优点
- **理论分析深入**：形式化证明了二进制奖励的优势坍塌（Proposition 1）和过程奖励的方差下界（Proposition 2），为方案提供了严谨基础。
