## 摘要
大语言模型、求解器和智能体团队越来越多地生成工作流操作、修复和计划，但生成的操作可能语法上有效，却可能过时、不可行、冲突或破坏触发修复的证据。我们引入了智能体事务处理（ATP），这是一种事务模型，将生成的操作视为不可信的提议，直到它们在声明且可执行的约束集C下通过确定性准入。该原则是双面的：提议不是真理，也没有提议能预见所有中断：任何事物都可以提议，但只有运行时接纳并提交，当未预见的中断发生时，它会响应式地在边界内进行修复，而不是信任新的提议。相对于C，提交状态正确性变得与提议层的胜任能力、诚实性或学习能力无关。我们在Mnemosyne中实现了ATP，Mnemosyne是一个运行时系统，具有仅追加的转换日志、有效状态投影、依赖安全的补偿和活跃的提交记录，并证明了相对于C的四个安全属性（权限分离、串行等价生成式准入、证据保持修复和义务约束），以及其局部化修复协议（LCRP）的有界响应式修复保证。一个可复现的工件在九个证伪测试中拒绝了目标违规，同时仍接受有效工作，投影和验证开销低于6%，且局部修复的编辑操作数量比全局重新计算少一个数量级。Mnemosyne是开源的：https://github.com/eyuchang/Mnemosyne/tree/arxiv-atp-rq1-rq9b-r8-v2。

## Abstract
LLMs, solvers, and agent teams increasingly generate workflow actions, repairs, and plans, but a generated action may be syntactically valid yet stale, infeasible, conflicting, or destructive of the evidence that triggered a repair. We introduce Agentic Transaction Processing (ATP), a transaction model that treats generated actions as untrusted proposals until they pass deterministic admission under a declared, executable constraint set C. The principle is two-sided: a proposal is not truth, and no proposal foresees every disruption: anything may propose, but only the runtime admits and commits, and when an unforeseen disruption strikes it repairs reactively within bounds rather than trusting a fresh proposal. Relative to C, committed-state correctness becomes independent of the competence, honesty, or learning of the proposing layer. We realize ATP in Mnemosyne, a runtime with an append-only transition log, effective-state projection, dependency-safe compensation, and active commitment records, and prove four safety properties relative to C (authority separation, serial-equivalent generative admission, evidence-preserving repair, and obligation containment) together with a bounded-reactive-repair guarantee for its localized repair protocol (LCRP). A reproducible artifact rejects the targeted violations across nine falsification tests while still admitting valid work, at under 6% projection-and-validation overhead, and bounded local repair edits an order of magnitude fewer operations than global recompute. Mnemosyne is open source: https://github.com/eyuchang/Mnemosyne/tree/arxiv-atp-rq1-rq9b-r8-v2.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：大语言模型、求解器和多智能体系统正越来越多地自动生成工作流动作、修复方案和计划。这些生成的动作在语法上可能完全正确，但可能过时、不可行（如依赖过期的交通时间）、相互冲突、或者修复方案会破坏触发它执行的证据（如删除失败测试用例来让 CI 变绿）。传统的 ACID 事务模型假设提交的工作单元是有意义的，不处理“该生成的动作是否应该被作为一个事务来接受”这一先决问题。
- **核心问题**：当生成者（LLM、求解器、智能体）本身不可信（可能产生幻觉、奉承、过时、对抗性输出）时，如何保证工作流状态正确性（committed-state correctness）独立于生成层的能力与诚实性？
- **整体含义**：提出 **Agentic Transaction Processing (ATP)** 模型，将生成的动作视为“不可信提议”，只有通过确定性约束集 C 的准入才能成为已提交的事务。状态正确性不再依赖智能层的能力，而是由数据管理层（事务日志、有效状态投影、准入门）保证。

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：
    - **“提议不是真理，没有任何提议能预见所有中断”**：任何 LLM、求解器、智能体、主动承诺等都可以提出动作，但只有运行时（runtime）通过确定性准入门（admission gate）后，提议才能成为提交的状态。
    - **权限分离**：智能层负责生成有用（或无用）的提议，事务层负责保证正确性（相对于约束集 C）。
    - **双防护**：静态准入（拒绝坏提议） + 响应式局部修复（处理已提交后发生的未预见中断）。

- **关键技术细节**：
    - **ATP 事务模型**：
        - 定义 Proposal（提议）、Committed transition（提交转换）、Committed-transition log (CTL，仅追加日志)、Effective state（有效状态，排除已补偿/已失效的记录）、Admission（确定性准入，检查有效状态、依赖安全、冲突范围、世界事实等）。
        - **Active Commitment Record (ACR)**：在 CTL 中存储的持久义务，触发后只能产生一个新提议，不能直接修改状态。
        - **约束集 C**：包括结构推导的规则（有限状态合法性、依赖有效性、补偿封闭性、幂等性、冲突范围）、声明式谓词（模式、证据、世界事实新鲜度）和少量应用验证器。
    - **准入算法 (Algorithm 1)**：
        1. 解析提议，检查租户、幂等键、操作键、冲突范围。
        2. 读取当前有效 StateView 和保留证据。
        3. 应用 C：有限状态、陈旧世界、依赖封闭、补偿安全；若为修复，要求触发证据保持有效或条件真的解决。
        4. 序列化或拒绝重叠冲突范围；ACR 唤醒只能提议，不能直接修改 CTL 或 StateView。
        5. 原子性追加到 CTL 并投影到 StateView，否则记录可查询的拒绝原因。
    - **局部级联修复协议 (LCRP)**：
        - 应对未预见的中断（如高速公路封闭、航班延误）。
        - 三个阶段：状态更新（记录中断为世界事实）、效应传播（沿着有效依赖向前传递影响）、有界重分配（仅修改受影响区域内的转换，受位移预算 δmax 和迭代上限 K 约束）。
        - LCRP 产生的修复本身是一个新提议，仍需经过正常准入。保证编辑半径仅覆盖依赖闭包，而非全量重规划。

- **公式与算法流程**（文字说明）：
    - **安全定理**：
        - 定理1（权限分离）：通过归纳证明，仅准入通过 C 的提议追加到 CTL，则所有提交状态相对 C 都正确，与智能层无关。
        - 定理2（串行等效生成准入）：冲突范围上的提议以某种串行顺序被准入，并发提议等效于串行历史。
        - 定理3（证据保持修复）：修复不能通过删除/补偿/隐藏触发证据来消解触发条件；必须真正解决条件或保留证据。
        - 定理4（义务约束）：ACR 创建是已准入的转换；唤醒只产生提议，不能直接修改状态。
    - **有界修复保证 (Proposition 1)**：LCRP 在 K 轮迭代内终止，累计位移 ≤δmax，修复时间 O(K(JM + JO_max))，不提交违反 C 的任何转换。

## 6. 主要结论与发现

- **权限分离成立**：在所有九类测试中，ATP 提交了零违规，而所有弱基线都提交了目标违规。说明 ATP 可以保证已提交状态相对 C 的正确性独立于提议生成者。
- **有效性与开销平衡**：ATP 在拒绝违规的同时仍接受有效工作，投影加验证开销 < 6%（在基础设施基准中为 4.96%–5.98%）。
- **局部修复有效性**：LCRP 的编辑操作数（平均 7.2）比全局重计算（平均 59.8）少一个数量级，且在成本感知目标 J 上更优（均值 58 vs 162）。表明在保持可行性的同时，LCRP 有效限制了中断的影响范围。
- **恢复期间恢复安全**：通过构造证明（Proposition 3），递归恢复退化为顺序恢复，任何嵌套深度都不会引入新的不安全性，因为未提交的修复和观察集是内存无关的。

## 7. 优点：方法与实验设计亮点

- **新颖的问题形式化**：将 AI 生成工作流的可靠性问题转化为数据库事务授权问题，提出了“提议不是真理”这一简洁核心原则。
- **形式化安全保证**：给出了四个可证明的定理，明确了在给定约束集 C 下的安全边界，而不是仅依靠启发式或 prompt 工程。
- **双机制防护层**：静态准入（防误提议） + 响应式局部修复（防未预见中断），两者都通过同一个确定性准入门，形成统一安全边界。
- **可重现的工件**：提供开源实现和预设的可重现实验，所有 9 个安全测试只需一个命令即可运行。
- **实验设计严谨**：对每种故障模式都设置了故意的“最宽松不安全基线”和对应的 ATP 结果，清晰地证明了 ATP 的优越性。
- **性能开销极低**：投影加验证开销不到 6%，局部修复的编辑操作比全局重计算少一个数量级，表明 ATP 实用可行。

## 8. 不足与局限

- **依赖约束集 C 的完整性和正确性**：ATP 的所有保证都只相对于 C。如果 C 不完整或错误，ATP 无法检测超出 C 的危险。编写一个完整的正确的 C 本身是需要额外应用工程工作的问题。
- **实验覆盖的真实性有限**：提议质量实验使用了确定性适配器模拟 LLM 故障模式，而非直接调用真实前沿模型（如 GPT-4o、Claude）。虽然论文声称故障模式是真实的，但可能在真实模型交互中出现未捕获的细微行为差异。
- **缺乏对分布式和跨区域部署的评估**：单存储节点下的线性化点清晰，但多区域或分片场景下保持串行等效准入和 outbox 幂等性需要额外机制（如共享共识日志），未被评估。
- **未与产品级工作流引擎进行直接吞吐对比**：Guardrail Comparator 是语义基线，不是与 Temporal、Cadence 等产品的性能基准。实际生产部署还需要更多工程化工作。
- **缺乏跨回合学习、后悔、抢占式规划方面的评估**：论文声明这些是未来工作，因此当前 ATP 不包含学习或预防性计划提升。
- **静态验证受限**：论文未用机器检查证明整个代码库无权限绕过，仅依靠测试和架构边界确保 gate closure 假设成立。可能存在未被测试覆盖的绕过路径。
- **修复的可判定性局限**：证据保持修复定理（Theorem 3）并不判定一个任意真实世界的故障是否真的被修复；只能保证如果无法验证修复解决了故障，则必须保留证据。实际中可能需要人工介入。
