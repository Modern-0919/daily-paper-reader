## 摘要
在让智能体在真实环境中运行之前，你能证明它使用了正确的证据吗？GroundEval将这个问题转化为一个确定性测试，检查智能体搜索、获取、引用以及被允许访问的内容。在一个案例研究中，两个前沿的LLM评判者对看似合理的智能体回答给出了高于0.85的评分。但跟踪记录揭示了另一番景象：该智能体从未检索其答案所依赖的工件，导致GroundEval得分为0.000。
我们提出了GroundEval，一个无评判框架，用于基于实体证据、时间限制和访问控制来评估智能体。GroundEval使用领域配置生成问题，让智能体自主选择回答方式，然后对最终答案及其产生的记录轨迹进行评分。该基准针对LLM作为评判的评估难以检测的三类失败：智能体在声称不存在之前是否进行了检查；是否仅依据行为者在相关时间可用的证据进行推理；以及是否使用了正确的因果机制而非似是而非的机制。这对应三个赛道：沉默、视角与反事实。GroundEval揭示了看似合理的答案可能基于无效证据路径的情况，并产出结构化的逐问题诊断报告，将工具活动与智能体的轮次叙述配对，使每个分数可审查而非仅仅报告。我们的案例研究表明，这种差距并非罕见的边缘情况，恰恰是最终答案和基于评判的评分从未设计去捕捉的盲点。

## Abstract
Before letting an agent operate over real context, can you prove it used the right evidence? GroundEval turns that question into a deterministic test of what the agent searched, fetched, cited, and was permitted to access. In one case study, two frontier LLM judges scored a plausible agent response above 0.85. But the trace told a different story: the agent had never retrieved the artifact its answer depended on, yielding a GroundEval score of 0.000.   We introduce GroundEval, a judge-free framework for evaluating agents against grounded, time-bounded, and access-controlled evidence. GroundEval uses a domain configuration to generate questions, lets the agent choose how to answer, and then scores both the final answer and the recorded trajectory that produced it. The benchmark targets three failures that LLM-as-judge evaluation struggles to detect: whether an agent checked before claiming absence, reasoned only from evidence available to the actor at the relevant time, and used the correct causal mechanism rather than a plausible one. These correspond to three tracks: Silence, Perspective, and Counterfactual. GroundEval exposes when plausible answers rest on invalid evidence paths, and produces structured per-question diagnostics that pair tool activity with the agent's turn-level narration, making each score inspectable rather than merely reported. What our case studies turned up is that this gap isn't some rare corner case. It's exactly the blind spot that final-answer and judge-based scoring were never built to catch.

- **研究动机**：案例中，两个前沿LLM评判者（Kimi-K2.6、ChatGPT-5.5）对同一智能体回答分别给出0.9和0.85的高分，认为推理“严谨且结构良好”，但GroundEval的轨迹检查显示该智能体从未检索回答所依赖的关键工件，得分0.000。这种差距不是边缘案例，而是最终答案评分和基于评判的评分固有盲点。
- **整体含义**：提出一种无需LLM评判的确定性评估框架，通过显式状态合约（事件日志、工件库、访问策略、评估配置）将“证据路径有效性”纳入评分，实现可审计、可复现的智能体质量度量。

## 2. 论文提出的方法论

### 核心思想
- 将评估从“最终答案匹配”扩展到“证据路径合规”：智能体不仅需给出正确答案，还需证明其答案是在规定的**时间边界、权限边界、因果机制和搜索空间**内产生的。
- 用**确定性状态合约**替代LLM的“似合理性”判断，合约由领域配置自动生成，无需人工标注预期轨迹。

### 关键技术细节
1. **四元组输入**：事件日志（时间戳流）、工件库（文档/记录）、访问策略（角色/子系统/时间可见性）、评估配置（因果链接规范、沉默对规范等）。
2. **双模式架构**：
   - **上下文模式（Context Mode）**：向智能体注入符合约束的上下文窗口，智能体需提交结构化答案及显式“证据工件”ID列表，通过检查引用合法性来评分。
   - **工具模式（Tool Mode）**：通过带门控的 `fetch_artifact` 和 `search_artifacts` 运行时记录完整工具调用轨迹，直接观察检索行为。
3. **三评估赛道**：
   - **视角（Perspective）**：测试智能体是否尊重特定行为者在特定时间点的知识边界（例如“Morgan在3月5日能否知道Acme有流失风险？”）。检测未来信息泄漏、跨用户泄漏、角色边界违规等。
   - **反事实（Counterfactual）**：测试智能体是否识别事件日志中正确的因果关系，要求原因和效果通过**联合键（如ticket ID）** 连接，而非仅凭时间邻近判定。
   - **沉默（Silence）**：测试智能体在声称“某事未发生”前是否搜索了所有预期位置（如“是否写了事后分析报告？”答案为“否”但需检查事后分析仓库）。
4. **双评分模型**：
   - **答案分数（Answer Score）**：基于结构化答案是否匹配地面真值（由事件日志和合约推导）。
   - **轨迹分数（Trajectory Score）**：基于工具调用序列是否满足访问权限、时间范围、搜索覆盖等约束。
5. **合规调整总分**：$S_{adj} = (w_a S_{answer} + w_t S_{traj}) \cdot (1 - v)^2$，其中 $v$ 为违规率（聚合角色门控、子系统、时间违规）。二次方乘数使高违规率模型即使答案正确也受重罚。

## 6. 论文的主要结论与发现

1. **确定性评分超越LLM评判**：在沉默赛道案例中，LLM评判因无法看到证据路径而给出0.85+高分，GroundEval因证据缺失评为0.000，证明其能检测LLM评判完全遗漏的“状态无效正确”问题。
2. **三类失败均可量化暴露**：
   - 沉默赛道：声称“不存在”但从未搜索关键工件（搜索覆盖率为0），轨迹分数0.273。
   - 反事实赛道：因表面主题不同而误判无因果链接，尽管已检索到包含连接键的工单，得分极低（答案0.062，轨迹0.637）。
   - 视角赛道：将物理出席混淆为子系统权限访问，导致7次角色门控违规，答案0.000，轨迹0.384。
3. **聚合结果**：总体合规调整后分数为0.350，违规率0.026（主要由视角赛道驱动）。零样本答案分数在沉默赛道上接近随机（0.487），在视角和反事实赛道上因表面线索仍有一定表现（0.200/0.220），说明问题生成需进一步控制表面规律。

## 7. 优点

### 方法亮点
- **确定性 & 可审计**：评分完全由事件日志和合约决定，无LLM主观偏差，同一合约同一轨迹得相同分数，适用于回归测试门控。
- **无黑盒依赖**：仅需外部可观测轨迹（工具调用或引用），无需思维链、模型内部状态或评判者解释，增强评估客观性。
- **自动诊断**：每一步违规（Actor门控、子系统、时间）都可定位并报告，使评分“可检查”而非仅“可报告”。
- **领域无关可复用**：配置为状态合约而非预期轨迹列表，用户仅需提供事件类型、访问规则等，评分逻辑不变，降低跨域迁移成本。

### 实验设计亮点
- 案例研究清晰展示了GroundEval与LLM评判的对比，直观说明盲点。
- 使用合成数据确保地面真值确定，避免自然数据标注噪声。

## 8. 不足与局限

- **实验覆盖有限**：仅测试一个模型（DeepSeek-V4-Pro），无跨模型泛化验证；未对比其他评估框架（如RAGAS、AgentRewardBench等）。
- **无消融研究**：未独立分析赛道权重、合规调整指数、门控模式等设计选择的必要性。
- **上下文模式观察力弱**：框架承认上下文模式无法确认智能体是否“暗中依赖”未引用的工件，存在盲区。
- **作者成本高**：领域配置编写是前端加载成本，虽承诺可复用，但初始构造仍需领域知识。
- **过滤问题生成偏差**：虽然包含多种反泄露控制，但零样本答案分数仍显示表面规律可被利用，说明问题生成有待改进。
- **仅限状态可约简任务**：不适用于创意、风格、主观质量等无法归约为状态约束的评估。
