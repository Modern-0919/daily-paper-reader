## 摘要
由大型语言模型（LLM）驱动的智能体AI系统正在迅速演变为自主决策系统，暴露了超越传统LLM漏洞的攻击向量。现有安全评估通常与特定实现或领域绑定，限制了跨异构系统的统一比较。为填补这一空白，我们提出RIFT-Bench，一种基于图表示的动态红队测试方法，支持跨不同智能体架构的统一评估。基于新颖的层次化表示，RIFT-Bench在两个自动化阶段运行：发现阶段提取系统结构，扫描阶段部署自适应对抗攻击并生成综合评估报告。它评估被检查系统本身，利用一组广泛且动态自适应的对抗探针跨多种攻击向量和目标。我们在覆盖多样化实现的45个智能体系统上展示了所提评估流水线的有效性，表明该方法能有效泛化到异构智能体架构。除系统和攻击外，RIFT-Bench还支持直接评估缓解策略。这些关键能力使RIFT-Bench成为智能体AI系统安全评估的可扩展基础。

## Abstract
Agentic AI systems powered by large language models (LLMs) are rapidly evolving into autonomous decision-making systems, exposing attack vectors beyond those of traditional LLM vulnerabilities. Existing security evaluations are often tied to specific implementations or domains, limiting unified comparison across heterogeneous systems. To address this gap, we introduce RIFT-Bench, a graph representation-driven methodology for dynamic red-teaming that enables unified evaluations across diverse agentic architectures. Building on a novel hierarchical representation, RIFT-Bench operates in two automated phases: Discovery, which extracts system structure, and Scanning, which deploys adaptive adversarial attacks and produces a comprehensive evaluation report. It evaluates the examined system itself, leveraging a broad set of dynamically adaptable adversarial probes across diverse attack vectors and objectives. We demonstrate the effectiveness of the proposed evaluation pipeline across 45 agentic systems spanning a diverse range of implementations, showing that the approach generalizes effectively to heterogeneous agentic architectures. Beyond systems and attacks, RIFT-Bench also supports direct evaluation of mitigation strategies. These key capabilities make RIFT-Bench a scalable foundation for security evaluation of agentic AI systems.

- **核心问题**：当前由大型语言模型（LLM）驱动的**智能体AI系统**正在快速部署，但它们不仅继承了传统LLM的漏洞（如提示注入、数据窃取），更因具备持久状态、工具调用、智能体间通信等系统级特性而**放大了风险**。现有的安全评估方法存在两大缺陷：
    1.  **缺乏标准化**：评估往往绑定特定实现框架、架构或领域，无法在**异构系统之间进行统一比较**。
    2.  **假设不匹配**：许多红队策略仍沿用LLM评估思路，将用户提示作为主要攻击向量，仅评估模型输出，**忽略了系统执行动态**（如工具调用、状态变更、智能体协调）中产生的漏洞。
- **整体含义**：为解决这一差距，论文提出了 **RIFT-Bench**，一种**基于图表示的动态红队方法**，旨在提供一个**实现无关的、可扩展的、标准化的基础**，用于对多样的智能体AI系统进行全面、可比较的安全评估。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：通过一个**统一的结构化表示（NodeSpec）**将对抗性评估从具体的实现中解耦出来，使攻击模板可以跨异构系统复用和自适应实例化。
- **关键技术细节**：
    1.  **NodeSpec（系统表示）**：一种**层次化的代码驱动图表示**，将智能体系统分解为组件（节点），每个节点包含5个维度：
        - **身份标识**（Identity）：名称、类型
        - **来源追踪**（Provenance）：代码引用、描述
        - **拓扑结构**（Topology）：父子关系、连接
        - **接口与配置**（Interface & Configuration）：输入输出、提示词、资源需求
        - **能力**（Capabilities）：代码执行、数据访问权限
    2.  **攻击分类学**：沿两个正交轴组织攻击：
        - **攻击表面（Attack Surface）**：用户输入、工具、内存、外部资源等入口点。组织为Surface -> Suite -> Probe -> Attempt 四层级别。
        - **对抗目标（Adversarial Objective）**：执行非期望操作、中断任务、泄露信息、资源过载。
    3.  **两阶段自动化流水线**：
        - **发现阶段（Discovery）**：通过**结构标识器**（一种结合LLM分析和确定性验证的模块化流水线）从目标系统的代码中自动提取NodeSpec。并支持**可配置的工具仿真**，以实现安全、低成本的评估。
        - **扫描阶段（Scanning）**：
            - **探针过滤**：根据NodeSpec中的能力标志，确定性过滤出适用于待测系统的攻击探针。
            - **攻击实例化**：利用NodeSpec中的组件信息（如工具名称、参数）填充探针占位符，生成具体的攻击尝试（Attempts）。
            - **执行与追踪**：在沙箱环境中独立执行每次攻击，并捕获完整的执行轨迹。
            - **轨迹评估**：使用探针关联的评估器对轨迹进行分析，测量多个指标。
    4.  **评估指标**：
        - **攻击激活率（AAR）**：攻击是否被执行（如被调用的工具）。
        - **攻击成功率（ASR）**：激活的攻击是否达成了目标（如调用了非预期工具）。
        - **任务效用（Utility）**：系统是否成功完成原始任务。
        - **执行漂移（ED）**：量化攻击导致的执行行为变化程度，公式：`ED = αΔ_comp + βΔ_out`，其中Δ_comp为组件级执行漂移，Δ_out为输出级漂移。
- **算法/流程说明**：发现阶段通过资源初始化、动态图构建、图验证、节点补全四步构建NodeSpec。扫描阶段通过探针过滤 → 尝试生成 → 系统适配 → 执行追踪 → 轨迹评估的流程进行。
