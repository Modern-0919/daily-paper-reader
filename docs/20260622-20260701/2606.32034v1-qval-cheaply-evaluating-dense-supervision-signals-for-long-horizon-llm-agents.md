## 摘要
LLM智能体越来越多地执行长期任务，单个轨迹可能包含数百或数千个动作。在这些场景中，仅基于结果的奖励提供的反馈过于稀疏，无法告知模型中间动作的好坏。密集监督方法旨在通过对中间步骤进行评分（从内在置信度到自蒸馏和嵌入相似性）来解决此问题。然而，评估这些方法的常见做法是通过衡量集成它们的训练流程的下游性能。这代价高昂，将监督质量与训练工程混杂因素混为一谈，并且使得需要不同训练设置的不同方法系列无法比较。因此，密集监督方法很少在共同基准上进行测试。我们引入了QVal，一个无需训练的直接评估密集监督信号的测试平台。给定一个状态-动作对，QVal衡量方法的评分与Q值对齐的程度：即它是否根据强参考策略的Q值对动作进行排序。这使得我们可以在任何训练运行之前比较信号，并将信号质量与其他工程选择分开。我们实例化QVal为QVal-v1.0，在四个不同的环境和七个方法论系列中基准测试了21种密集监督方法，跨越六个开放权重模型骨干进行了超过1200次评估实验。我们发现，简单的提示基线始终优于文献中最近的密集监督方法，并且性能按系列强烈聚类。这些发现在模型大小、环境和观察模态上均成立。QVal设计为易于扩展到新环境和新方法，使研究人员能够在任何训练运行之前迭代密集监督方法。

## Abstract
LLM agents increasingly act over long horizons, where a single trajectory can contain hundreds or thousands of actions. In these settings, outcome-only rewards provide too sparse guidance, failing to inform the model about the goodness of intermediate actions. Dense supervision methods aim to solve this problem by scoring intermediate steps, from intrinsic confidence to self-distillation and embedding similarities. However, it is common practice to evaluate them by measuring the downstream performance of a training pipeline that integrates them. This is expensive, conflates supervision quality with training engineering confounders, and renders different methodological families requiring distinct training setups incomparable. As a result, dense supervision methods are rarely benchmarked on common ground. We introduce QVal, a training-free testbed for directly evaluating dense supervision signals. Given a state-action pair, QVal measures how well a method's score is Q-aligned: whether it orders actions according to the Q-values of a strong reference-policy. This lets us compare signals before any training run and separate signal quality from other engineering choices. We instantiate QVal as QVal-v1.0, benchmarking 21 dense supervision methods across four diverse environments and seven methodological families, with over 1.2K evaluation experiments across six open-weight model backbones. We find that simple prompting baselines consistently outperform recent dense supervision methods from the literature, and that performance clusters strongly by family. These findings hold across model sizes, environments, and observation modalities. QVal is designed to be easily extensible to new environments and methods, enabling researchers to iterate on dense supervision methods before any training run.

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：QVal 通过衡量方法预测分数与**参考策略 Q 值**之间的排序一致性（**Q对齐**）来评估信号质量。一个高质量的信号应能正确排序动作，使其顺序与参考 Q 值一致。
- **关键技术细节**：
  - 对于每个环境，收集状态-动作对 `(s, a)`，并利用一个强参考策略 \(\pi\) 计算其 Q 值 \(Q^\pi(s,a)\) 作为标签。
  - 参考策略的选择：若存在最优策略（如 FrozenLake、OpenApps、ALFWorld 中的脚本策略），则直接使用；否则（如 TerminalBench）采用最大值蒙特卡洛（MVMC）采样，使用如 GPT-5.5 等强模型。
  - **Q对齐**的定义：信号函数 \(k(s,a)\) 若满足 \(k(s,a) = \phi(Q^\pi(s,a))\)，其中 \(\phi\) 是严格单调递增函数，则称其为完美 Q 对齐。
  - 评估指标：使用 **Spearman 秩相关系数 \(\rho\)** 和 **Kendall \(\tau\)** 来衡量预测分数与标签之间的单调一致性。针对输出排列的方法（如 ranking），采用**状态局部 Spearman**（state-local Spearman），在状态内计算平均秩相关。
- **算法流程**（文字描述）：
  1. 在目标环境中收集多样化的轨迹，采样状态-动作对。
  2. 对每个点，通过参考策略的蒙特卡洛（或确定性）滚动得到 Q 值标签（采用 MVMC 近似最优值）。
  3. 将待评估的密集监督方法应用于相同的状态-动作对，得到预测分数。
  4. 计算预测分数与 Q 值标签之间的秩相关系数（Spearman 或 Kendall），作为 Q 对齐分数。
- **特点**：每种方法使用相同的模型骨干、上下文信息，避免训练工程混杂因素。

### 6. 论文的主要结论与发现

1. **简单方法表现最好**：Ranking 和 Direct prompting（直接数值预测）在平均 Q 对齐上显著优于其他家族。
2. **复杂性不提升性能**：同一家族内，更复杂的变体（如 direct-16、codegen-avg）并不比简单版（direct-single）更好。
3. **方法族聚类明显**：同一家族的方法获得相似的 Q 对齐范围，验证了论文方法分类的有效性。
4. **性能不随环境难度单调下降**：Direct 方法在所有环境中保持正值，而 Code 和 Ranking 在开放环境（TerminalBench）中变差，Self-Distillation 反而在简单环境差、在 TerminalBench 较好。
5. **文本模态优于图像**：在同时提供文本和图像的环境上，文本观测获得更高的 Q 对齐。
6. **对参考值类型稳健**：Q 值与状态值 V 作为标签时，方法相对排序基本一致。
7. **MVMC 骨干影响小**：使用 GPT-5.5 或 Claude Opus 4.7 作为 TerminalBench 参考策略，方法排序类似。

### 7. 优点

- **无训练评估**：直接隔离信号质量，排除训练工程干扰，大幅降低成本。
- **标准化的基准**：提供 QVal-v1.0 和预标记数据集，便于扩展新方法和新环境。
- **细致的鲁棒性分析**：检验了模态、参考值类型、参考骨干等因素，支持结论可靠性。
- **方法分类清晰**：按信息源将 21 种方法分为 7 个家族，有助于理解不同系列的特性。
- **实用建议**：提出 Direct prompting 应作为基线，新方法需证明其复杂度带来的信号提升。

### 8. 不足与局限

- **实验覆盖有限**：仅四个环境，缺乏更多种类的长期 Agent 任务（如编程、游戏、多步骤推理；缺少多智能体或部分可观测环境）。
- **未验证下游训练效果**：Q 对齐是信号质量的代理指标，但高质量对齐并不保证后训练成功（可能与其他训练因素相互作用，或忽略探索激励）。
- **视觉方法劣势未深入解释**：文本优于图像可能只是因为状态抽象更直接，而非视觉信号本身差，论文承认这一点。
- **部分方法依赖内部模型信息**：如 ∆belief、sdpo 需要 token logprobs，限制了可评估的模型范围（如闭源模型可能无法使用）。
- **参考策略质量依赖**：TerminalBench 的标签基于 GPT-5.5 MVMC，虽然与 Claude 结果一致，但理论上不能保证是最优。
- **评价指标单一**：主要依赖 Spearman/Kendall，未考虑信号绝对值的重要性（如奖励塑造中缩放可能重要）。
- **未提供计算资源详情**：使得复现成本难以估算。
