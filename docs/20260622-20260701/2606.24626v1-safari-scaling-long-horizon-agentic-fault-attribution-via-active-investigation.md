## 摘要
随着自主智能体处理日益复杂的多步骤、多智能体任务，其执行轨迹已经超出了最大上下文窗口的限制。当前有效地诊断智能体故障的方法是将完整轨迹加载到大语言模型的上下文窗口中，但这会导致注意力稀释，并且当智能体痕迹不可避免地超出上下文限制时，方法失效。为了解决这一问题，我们提出了SAFARI（通过主动调查实现长视界智能体故障归因的规模化），该框架通过一个工具增强的诊断循环替代了线性上下文加载。通过为大语言模型配备专门的工具箱来读取和搜索轨迹片段，并配备持久的短期记忆以实现跨轮推理，SAFARI有效地将诊断准确性与架构上下文限制分离。我们的实验表明，在1M token预算下，SAFARI在Who&When数据集上的表现比当前最佳结果高出20%，在25K token预算下，在TRAIL GAIA子集上高出19%。最重要的是，即使目标故障位于模型原生上下文窗口的5倍之外，SAFARI仍能保持0.58的精确度，而传统评估器在这种情况下完全失效。

## Abstract
As autonomous agents tackle increasingly complex multi-step, multi-agent tasks, their execution trajectories have scaled beyond the constraints of even the largest context windows. Current methods for effectively diagnosing agent failures load the full trajectory into an LLM's context window, which suffers from attention dilution and fails when agentic traces inevitably exceed context limits. To address this, we introduce SAFARI (Scaling long-horizon Agentic Fault AttRibution via active Investigation), a framework that replaces linear context loading with a tool-augmented diagnostic loop. By equipping LLMs with a specialized toolbox to read and search trajectory segments alongside a persistent Short-Term Memory (STM) for cross-turn reasoning, SAFARI effectively decouples diagnostic accuracy from architectural context limits. Our experiments demonstrate that SAFARI outperforms state-of-the-art results by 20% on the Who&When dataset within a 1M token budget, and by 19% on TRAIL GAIA subset on a 25K token budget. Most significantly, SAFARI maintains a 0.58 precision even when the target fault resides 5x beyond the model's native context window, a scenario where traditional evaluators fail entirely.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：随着 LLM 驱动的自主智能体处理多步骤、多智能体任务，其执行轨迹长度已远超模型上下文窗口（如 2M tokens）。现有故障诊断方法（如一次性加载完整轨迹到 LLM）会导致 **注意力稀释**，且在轨迹超出上下文限制时完全失效。
- **整体含义**：提出一种能**解耦诊断精度与上下文限制**的框架，使故障归因可以在长于模型原生窗口的轨迹上仍然有效，从而提升复杂代理系统的可观测性和可靠性。

## 2. 方法论：核心思想、关键技术细节、算法流程
### 核心思想
用 **主动调查（Active Investigation）** 替代传统的被动线性上下文加载。LLM 作为 Investigator Agent，通过迭代调用专门的工具（read / search）选择性查询轨迹片段，并维护一个**短期记忆 (Short-Term Memory, STM)** 以保留跨轮推理的关键证据，最终在证据充分时输出故障步骤。

### 关键技术细节
- **工具集**：
  - `read(offset, limit)`：读取轨迹的连续片段（类似“滚动日志”）。
  - `search(pattern)`：在序列化轨迹上执行不分大小写的正则搜索，直接跳转到目标步骤或特定值。
- **短期记忆 (STM)**：四个字段：`task_goal`、`final_output + initial hypothesis`、`faults_identified_so_far`、`still_need`。STM 在每轮 LLM 调用末尾追加，当历史记录因上下文预算限制被丢弃时，STM 保留关键证据和推理链。
- **验证与决策**：
  - 收集足够证据后，Investigator Agent 将假设分解为至多 3 个**原子声明 (atomic claims)**，并附带引用证据。
  - 并行发送给 **Reasoning Evaluator LLM**（只有引用证据，无底层轨迹），返回每个声明的置信度分数（0-100）和反馈。
  - 平均置信度 ≥ θ（默认 70）则允许提交最终结果；否则根据低分声明的反馈，Agent 继续调查以补强证据。
- **迭代终止条件**：
  - 成功提交（有效结论）→ 输出预测的故障步骤 t' 和推理论据 R'.
  - 或达到最大迭代轮次 K=30 → 执行“强制终结”，综合已有证据给出最佳答案。

### 算法流程（文字说明）
1. **初始化**：设置系统提示和用户提示（包含轨迹总长度、初始元数据）。
2. **循环**：
   - Agent 观察当前历史 Hk 和工具模式 T，生成工具调用 a（read 或 search）。
   - 执行工具并观察结果 o，更新历史 Hk+1 = Hk ∪ {(a, o)}。
   - 若历史超限，丢弃最早轮次；STM 在每次 LLM 调用时追加于末尾，确保证据不丢失。
   - 当 Agent 认为证据充分时，形成假设并调用 evaluate 函数进行原子验证。
   - 根据平均置信度决定提交或继续调查。
3. **终止**：输出预测。

## 6. 主要结论与发现
- 在 **Who&When Hand-crafted**（1M token 预算）上，SAFARI 超 SOTA 20%（±1 宽容精度 0.414 vs RAFFLES 0.345）。
- 在 **TRAIL GAIA**（严格 25K token 预算）上，SAFARI 的超 SOTA 基线 19%（严格精度 0.5043 vs RAFFLES 0.4188）。
- 当故障位置**位于模型上下文窗口 5 倍之外**（ς=5），SAFARI 仍保持 ~0.58 精确度，而所有传统方法（需完整读入轨迹）完全失效。
- **短期记忆 (STM)** 是核心：在高资源预算下优势减小（因原始日志可完整容纳），但在低预算或超窗口场景下显著提升性能。
- 在代码密集的 SWE 子集上，SAFARI 性能下降（说明代码轨迹的简练记忆更难）。

## 7. 优点
- **新颖的主动调查范式**：通过工具调用选择性查询，而非被动阅读完整轨迹，显著节省 token 并突破上下文限制。
- **短期记忆 (STM)** 作为逻辑锚点，跨轮保留关键证据、假设和待办计划，防止信息在上下文丢弃时丢失。
- **原子验证机制**：将假设分解为可独立验证的声明，由独立评估器 LLM 检查，提高输出可靠性。
- **自适应的延迟特性**：在低预算下增加计算量换取精度，在高预算下快速收敛。
- **统一处理不同规模的轨迹**：从几千 tokens 到超过 2M tokens 均有良好表现。
- **实验结果全面**：覆盖多种预算、缩放因子、两个基准，并包含消融与延迟分析。

## 8. 不足与局限
- **实验覆盖不足**：
  - TRAIL SWE 子集仅 31 条，统计结论的稳健性较弱。
  - 仅使用单一骨干模型（Claude-Opus-4.6），未测试其他 LLM（如 GPT-4、Llama 等）。
- **性能退化场景**：
  - 当轨迹完全适合上下文窗口时（如 1M budget），SAFARI 略微差于 RAFFLES，因为 STM 的压缩可能丢失信息，且原始日志同时存在于历史中造成冗余/冲突。
  - 代码丰富的轨迹（SWE）压缩困难，SAFARI 效果不如 GAIA。
- **记忆化风险**：作者指出预训练数据集可能包含基准任务的正确答案，导致模型“知道答案”而非真正归因，在私有数据集上的表现未知。

- **未探索**：未测试在实时/持续监控场景下的效果；未扩展到故障归因以外的通用长程推理任务。
- **可解释性**：虽然引入了原子验证，但 STM 的错误积累和上下文丢弃可能导致不透明的推理链。
