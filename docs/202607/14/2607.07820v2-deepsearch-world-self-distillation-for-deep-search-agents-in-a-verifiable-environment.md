## 摘要
训练工具使用代理从自身经验中改进仍然具有挑战性，因为监督微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长程交互提供了弱监督。我们提出了DeepSearch-Evolve，这是一个基于DeepSearch-World构建的用于网络代理的自蒸馏框架，DeepSearch-World是一个确定且可验证的环境，具有可复现的搜索和页面阅读工具。DeepSearch-World包含从实体级随机游走构建的42万个多跳问答任务，并支持对自我进化有用的关键代理认知行为，包括进度验证、基于依据的反思和失败恢复。DeepSearch-Evolve迭代执行轨迹生成、过滤、数据混合和微调，以训练更强的代理。无需从更强大的模型中蒸馏，DeepSearch-World-9B在与开源代理相比时达到了竞争性性能，在BrowseComp上达到31.2%，在GAIA上达到61.5%，在HotpotQA上达到93.4%，表明可验证环境能够实现长程网络代理的可扩展自我进化。我们将发布环境、42万训练池、验证集、模型和代码，以促进未来对自我改进深度搜索代理的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有工具使用智能体（agent）的自我改进方法存在两个主要瓶颈：**监督微调（SFT）** 依赖固定的教师蒸馏轨迹，性能受限于教师能力且容易饱和；**强化学习（RL）** 使用稀疏的最终奖励，对于长程多步交互缺乏过程级监督，难以指导智能体在规划、工具选择、证据提取等中间步骤中做出正确决策。
- **研究动机**：为智能体提供一个**确定性、可验证的离线环境**，使得中间步骤的进度和失败可以被客观验证（而非依赖LLM评估），从而为自蒸馏提供稳定、细粒度的监督信号。
- **整体含义**：通过构建可验证的离线Wikipedia环境（DeepSearch-World）并结合自蒸馏框架（DeepSearch-Evolve），使得开源9B模型在无需更强教师模型（如GPT、Claude等）的情况下，能够从自身生成的已验证轨迹中迭代学习，在多个深度搜索基准上达到与依赖强模型蒸馏的开源智能体相当的性能。这项工作证明了**可验证环境是实现长程网络智能体可扩展自我进化的关键**。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：在确定性可验证的离线工具环境中，设计一个具有**脚手架（scaffold）结构**的教师智能体，生成包含进度追踪、证据记忆、反思恢复等认知行为的轨迹，然后将这些高质量轨迹转换为标准的ReAct格式，通过迭代SFT训练学生模型。整个过程不需要外部更强模型，仅依赖智能体自身在可验证环境中的探索。
- **关键技术细节**：
  - **DeepSearch-World环境构建**：
    - 基于Wikipedia超链接图，通过实体级随机游走生成42万个多跳QA任务（每问题隐去显式实体，需智能体通过搜索和浏览恢复目标实体）。
    - 构建离线Wikipedia语料库（约1000万实体），使用Pyserini的BM25索引实现确定性搜索（`web_search_wiki`）和基于SQLite偏移索引的页面浏览（`visit_wiki`）。
    - 环境内置**实体级进度验证**：对于每个问题，存储隐藏的目标实体集，智能体每次工具调用若匹配未解决实体则标记为成功；失败时提供分阶段规则化反思（第一次失败给通用提示，重复失败提供更明确指引如实体名称或模糊描述）。
  - **脚手架教师智能体（Scaffold Teacher Agent）**：
    - 三阶段交互流程：**Plan**（初始化进度状态，包含completed_list、todo_list、experience、information四个字段）→ **Act**（循环调用工具，更新进度状态，接收环境反思）→ **End**（当证据足够时生成基于已验证记忆的简洁答案）。
    - 轨迹中包含规划、记忆维护、失败恢复、证据归因等认知行为。
  - **自蒸馏训练循环（DeepSearch-Evolve）**：
    - 每一轮：当前模型πθ作为教师在环境中生成脚手架轨迹 → 通过拒绝采样（仅保留答案正确轨迹）和质量过滤（用Qwen3.5-9B去除冗余、弱对齐或不一致轨迹）→ **Scaffold-to-ReAct转换**：移除脚手架专用提示，将进度状态和反思重写为`<think>`块中的推理轨迹（包含计划、记忆、自我纠错等）→ 在转换后的ReAct轨迹上进行SFT更新 → 得到下一轮模型。
    - 采用**重要性采样数据混合**：每轮训练使用4k条轨迹，来自各轮按照衰减因子γ=0.5采样，优先用新数据但保留旧数据缓解灾难性遗忘。
    - 异步生成-训练架构：生成和训练解耦，一轮约2-3天（8GPU H20节点）。
  - **补充RL微调**：为进一步缩小离线到真实的差距，额外在1600条真实工具实例上使用GRPO（Google SerpAPI搜索 + Jina页面检索）进行训练。

### 6. 论文的主要结论与发现

1. **自蒸馏有效**：DeepSearch-World-9B在几乎所有基准上大幅超过基础模型Qwen3.5-9B-Instruct，提升幅度+23.8%（BrowseComp）、+37.6%（GAIA）、+48.1%（HotpotQA）等，证明可验证环境下的自蒸馏框架能显著提升智能体能力。
2. **无需更强教师**：仅依赖自身验证轨迹，达到与依赖强模型蒸馏的开源智能体（如Marco-DR、MiroThinker）相当的性能，表明环境验证可以替代部分外部教师监督。
3. **数据多样性关键**：420K训练池相比100K能维持更长的性能上升曲线，避免早期饱和和波动（图5），说明自蒸馏需要丰富且不断有新鲜实例注入。
4. **认知行为迁移**：通过Scaffold-to-ReAct转换，学生智能体学会了更长的交互（平均18.0轮 vs 4.7轮）、更多的浏览调用（5.4 vs 0.9）和更高的高级能力得分（70% vs 19%），证明渐进式规划、记忆维护和反思恢复等行为成功迁移。

### 7. 优点

- **环境设计创新**：首次提出**确定性+实体级进度验证**的离线Wikipedia环境，为长程工具使用提供稳定、可复现、细粒度的过程监督，无需昂贵LLM判断。
