## 摘要
虽然大语言模型（LLM）智能体在静态基准测试中表现出色，但它们在真实场景中的部署受到用户查询、工具集和交互动态等动态特性的阻碍。为弥补这一泛化差距，我们形式化了OpenAgent（开放世界中的工具使用智能体）问题设定，其特点在于查询、动作、观察和领域维度的分布偏移。为系统诊断其影响，我们构建了一个受控沙箱环境，在其中定义了跨四个层级（感知、交互、推理和内化）的细粒度环境偏移，并进行了一系列全面实验。我们的分析得出一系列关键见解，表明通过监督微调（SFT）和强化学习训练的智能体在面对开放环境偏移时都会遭受不同程度的性能下降。基于这些见解，我们提出扰动增强微调（Perturbation-Augmented Fine-Tuning），这是一种针对SFT的基于扰动的干预策略，为增强智能体在真实环境中的鲁棒性和实用性奠定了基础。我们的代码将在https://github.com/LAMDA-NeSy/OpenAgent发布。

## Abstract
While Large Language Model (LLM) agents demonstrate proficiency in static benchmarks, their deployment in real-world scenarios is hindered by the dynamic nature of user queries, tool sets, and interaction dynamics. To address this generalization gap, we formalize OpenAgent (Tool-Use Agent in Open-World), a problem setting characterized by distributional shifts across query, action, observation, and domain dimensions. To systematically diagnose its impact, we construct a controlled sandbox environment where we define fine-grained environmental shifts across a four-tier hierarchy, Perception, Interaction, Reasoning, and Internalization, and conduct a comprehensive series of experiments. Our analysis yields a series of key insights, demonstrating that agents trained via both Supervised Fine-Tuning(SFT) and Reinforcement Learning suffer from varying degrees of performance degradation when confronting open environmental shifts. Building on these insights, we propose Perturbation-Augmented Fine-Tuning, a disturbance-based intervention strategy for SFT that lays the foundation for enhancing agent robustness and utility in realistic environments. Our code will be released at: https://github. com/LAMDA-NeSy/OpenAgent.

- **核心问题**：当前训练范式（SFT 与 RL）是否能使智能体真正**泛化到开放世界**？如何系统诊断并缓解性能退化？
- **整体意义**：揭示静态训练的脆弱性，提出形式化问题设定 OpenAgent 和针对性干预策略，为提升智能体在非平稳环境中的鲁棒性奠定基础。

## 2. 方法论

### 2.1 核心思想
- 将开放世界挑战分解为**四维分布偏移**：查询意图偏移（ΔQ）、动作空间偏移（ΔA）、观测动态偏移（ΔO）、组合领域偏移（ΔD）。
- 构建**受控沙盒环境**，在封闭环境基线基础上系统性注入扰动，分四个层级（感知、交互、推理、内化）诊断模型脆弱性。
- 提出**扰动增强微调（Perturbation-Augmented Fine-Tuning, PAFT）**，通过在训练轨迹中注入可控扰动，迫使模型学习恢复行为而非记忆模式。

### 2.2 关键技术细节
- **沙盒环境**：基于 Python 的模拟工具系统，使用北京 POI 数据（匿名化），定义 12 个工具（如坐标查询、电话查找、距离计算等），支持多步轨迹。
- **PAFT 组成**：
  - **环境反馈扰动（EFP）**：在成功子轨迹中注入异常观测（如工具重定向、值废弃），监督模型输出矫正动作，将开环行为转为闭环恢复。
  - **可解性边界扰动（SBP）**：构造不可解负样本（致命错误、工具不可用），监督模型明确拒绝而非幻觉答案，打破“所有问题都可解”的隐含假设。
  - **符号表示扰动（SRP）**：对工具名称和文档进行同义改写或噪声注入，迫使模型依赖功能语义而非记忆的标识符。
- **公式**：$D_{\text{PAFT}} = (1-\alpha) D_{\text{clean}} \cup \alpha \cdot \mathcal{G}(D_{\text{clean}})$，其中 $\mathcal{G}$ 是扰动操作，$\alpha=0.3$。

## 6. 主要结论与发现

1. **SFT 的符号锚定（Symbolic Anchoring）**：SFT 模型过度依赖工具名称等表面符号，当名称改变（同义改写、噪声注入、语义陷阱）时准确率骤降，且随着训练步数增加退化更严重。RL 通过交互惩罚机制学会语义基础，表现更鲁棒。
2. **SFT 的轨迹惯性（Trajectory Inertia）**：在工具重定向等交互扰动下，SFT 模型无视反馈，继续按预计划执行甚至幻觉成功观测；RL 则解析指令直接切换到新工具。
3. **两者共同的推理脆弱性**：在逻辑反转（工具依赖关系颠倒）下，SFT 和 RL 均大幅退化，说明两者都过拟合了训练时的因果顺序，未能根据文档动态推理。
4. **边界盲视（Boundary Blindness）**：面对不可解任务（致命错误或信息缺失），SFT 直接忽略错误并输出幻觉答案；RL 能感知错误但仍倾向强行完成，体现了奖励设计（没有拒绝机制）导致的**目的论偏差**。
5. **PAFT 有效性**：在扰动注入后，PAFT 显著恢复各层级性能（早期阶段从大幅负Delta转为正Delta），消融实验证明三个组件各有针对性（EFP 提升交互，SBP 提升拒绝，SRP 提升感知/推理），且 α=0.3 效果最好。

## 7. 优点

- **问题形式化清晰**：明确定义 OpenAgent 的四个偏移维度，使诊断有据可依。
- **受控沙盒设计**：隔离真实 API 的不可控噪声，实现纯净的封闭基线，能精确测量每种扰动的影响。
- **细粒度诊断框架**：四个层级（感知→交互→推理→内化）覆盖了从输入解析到高层决策的全链条，且每个层级都设计了多个具体子实验。
- **训练动态分析**：监测整个训练过程中的性能变化，发现“过拟合越多，泛化越差”的退化趋势，有重要启示。
- **PAFT 方法可解释性强**：每种扰动针对诊断出的具体缺陷，且组件消融验证了其互补性，不是盲目的数据增强。

## 8. 不足与局限

- **任务域局限**：所有实验基于地理 POI 和简单计算场景，是否迁移到更复杂领域（如代码执行、网络交互、多模态）尚不明确。
