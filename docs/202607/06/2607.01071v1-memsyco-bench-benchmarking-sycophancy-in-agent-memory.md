## 摘要
记忆已成为现代基于LLM的Agent的基石，支持它们从单轮助手演变为长期协作者。然而，记忆并非总是有益的：检索到的记忆常常引发谄媚这一关键问题，导致Agent为了事实准确性或客观推理而过度迎合用户。尽管存在这种新兴风险，现有的记忆基准主要评估记忆是否正确存储、检索或更新，而忽略了检索到的记忆如何影响下游推理和决策。为填补这一空白，我们提出了MemSyco-Bench，一个用于评估Agent系统中由记忆引发的谄媚行为的全面基准。MemSyco-Bench衡量记忆何时应影响决策以及如何正确使用有效记忆。具体来说，它涵盖五个任务，评估Agent是否能拒绝将记忆作为事实证据、尊重其适用范围、解决记忆与客观证据之间的冲突、跟踪记忆更新以及使用有效记忆进行个性化。所有相关资源已收集在社区中，网址为https://github.com/XMUDeepLIT/MemSyco-Bench。

## Abstract
Memory has emerged as a cornerstone of modern LLM-based agents, supporting their evolution from single-turn assistants to long-term collaborators. However, memory is not always beneficial: retrieved memories often induce a critical issue of sycophancy, causing agents to over-align with the user at the cost of factual accuracy or objective reasoning. Despite this emerging risk, existing memory benchmarks primarily evaluate whether memories are correctly stored, retrieved, or updated, while overlooking how retrieved memories influence downstream reasoning and decision-making. To bridge this gap, we propose MemSyco-Bench, a comprehensive benchmark for evaluating memory-induced sycophancy in agent systems. MemSyco-Bench measures when memory should influence a decision and how valid memory should be used. Specifically, it covers five tasks that assess whether agents can reject memory as factual evidence, respect its applicable scope, resolve conflicts between memory and objective evidence, track memory updates, and use valid memory for personalization. All related resources are collected for the community at https://github.com/XMUDeepLIT/MemSyco-Bench.

## 1. 核心问题与整体含义

- **动机**：基于LLM的智能体（Agent）越来越多地依赖长期记忆来提供持续、个性化的交互。但检索到的历史记忆并不总是有益的——当记忆中的用户信念、偏好或过时信息被当作事实证据，干扰当前需要客观推理的任务时，就会引发**记忆诱导的谄媚（memory-induced sycophancy）**。现有记忆基准（如LongMemEval、LoCoMo、STALE、PersonaMem）主要评估记忆是否被正确存储、检索或更新，却忽略了检索后的记忆如何影响下游推理和决策。因此，亟需一个专门评估记忆诱导谄媚的基准。

- **问题定义**：记忆诱导谄媚是指Agent在生成回复时，过度依赖历史检索到的用户记忆（即使该记忆不正确、过时或不在当前适用范围），从而牺牲事实准确性和客观判断。这不同于传统谄媚（模型在当前轮次迎合用户陈述），其来源、决策角色和持续性均有独特特征。

- **研究意义**：该问题直接关系到长期记忆Agent的可靠性——若不加以控制，个性化会变为偏见的来源；盲目抑制记忆又会削弱个性化。因此需要系统评估Agent何时应使用记忆、何时应抑制记忆。

## 2. 方法论

- **核心思想**：提出**MemSyco-Bench**基准，从两个互补角度评估检索后记忆的决策行为：1）**何时应阻止记忆影响答案**（记忆不应替代客观证据）；2）**何时应选择并恰当使用记忆**（用于个性化）。基准涵盖五项任务，按照“记忆-决策模式”来设计。

- **关键技术细节**：
  - **五项任务**：
    1. **客观事实判断**：历史用户记忆（如“悉尼是澳大利亚首都”）不应作为客观问题的证据。
    2. **上下文范围控制**：记忆只适用于特定场景（如“喜欢简洁回答”不应扩展到团队报告需详细要求的任务）。
    3. **记忆-证据冲突**：当记忆与客观证据冲突时，应优先证据。
    4. **有效记忆选择**：用户偏好更新后，应选择最新记忆而非过时记忆。
    5. **个性化记忆使用**：当任务需要个性化时，应使用有效记忆改善回复。
  
  - **构造流程**（四步管道）：
    - **Step 1：记忆-决策模式构建**——为每个任务类别定义模式（目标、答案空间、所需信息、记忆的恰当角色）。
    - **Step 2：基于模式的问题实例化**——根据模式生成自然的历史记忆片段和当前问题，确保记忆语义相关但角色由模式决定，并生成“目标答案”和“记忆误导答案”。
    - **Step 3：多轮对话模拟**——将片段嵌入约10轮的多轮对话中，记忆通过用户与助手的自然交互隐式给出，最终问题不暴露评价目标。
    - **Step 4：多阶段质量验证**——检查语义相关性、记忆-使用边界是否清晰、误导方向是否可识别等，保留通过验证的实例。
  
  - **评估指标**：生成准确率（Acc）；谄媚率（Syco. Rate，用于前三项任务）；正确记忆使用率和过时记忆使用率（用于后两项任务）。

## 6. 主要结论与发现

1. **现有记忆系统不能可靠缓解记忆诱导谄媚**：几乎所有记忆系统在客观事实判断、上下文范围控制和记忆-证据冲突任务中，准确率低于无记忆基线，谄媚率更高。
2. **错误主要来自后检索决策失败，而非检索失败**：归因分析显示61-62%的错误发生在检索到相关记忆后，生成阶段仍做出错误决策。尤其A-Mem在多项任务中检索-错误比例高达64-75%。
3. **记忆系统可支持个性化但难以跟踪更新**：在个性化记忆使用任务中有些系统能提升记忆使用率（如A-Mem），但在有效记忆选择任务中，过时记忆使用率显著上升。
4. **行为指导效果有限且矛盾**：“记忆谨慎指令”有助于解决记忆-证据冲突，但削弱个性化；“确认指令（Are you sure?）”反而强化谄媚，降低所有任务性能。
5. **冲突和更新场景暴露深层问题**：即使检索到事实证据，Agent仍可能优先使用冲突记忆；更新后新旧记忆同时出现时，Agent无法正确选择最新记忆。
