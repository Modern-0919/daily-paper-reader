## 摘要
智能体系统的执行轨迹记录了其失败的方式，而改进此类系统而不改变模型权重的程序（轨迹选择、提示和工作流优化、运行时监控）会读取这些轨迹以获取反馈。然而，原始轨迹是积累这种反馈的糟糕媒介：冗长、特定于实例，并且缺乏用于重复失败的稳定词汇表。我们认为，智能体系统应该维护一个显式的失败表示，该表示由其自身行为归纳得出，并且在任何需要失败反馈的地方都可重用。AdaMAST通过将目标系统的轨迹转换为紧凑的、基于证据的失败分类法来构建这种表示：沿三个固定轴（系统级、角色特定和领域特定）组织的命名失败代码，每个名称、定义和证据模式都是从轨迹中归纳得出的；没有手工编写的代码，也没有人工标注的轨迹。该分类法不仅是事后诊断工具，还是一个共享的反馈接口，通过三种方式改进智能体。在智能体系统搜索中，使用分类法编码的失败候选诊断在所有五个测试基准上都优于自由形式的反思。在运行时，分类法反馈将SWE-agent在SWE-bench Verified Mini上的分辨率从60%（自由文本反思）提高到70%，并将Claude Code作为运行时技能从64.0%提高到70.7%。在轨迹选择中，基于归纳代码构建的验证器AdaMAST-Judge在Terminal-Bench 2.0上将最佳5次准确率相比Pass@1提高了8-15个百分点。词汇表本身是紧凑的（一个数量级的压缩，同时保留了轨迹区分性）、符合人类认知的（比手工制作的参考词汇表更接近专家失败注释），并且具有适应性（为不同领域归纳的分类法共享很少的代码）。适应性失败分类法闭环了智能体产生的轨迹与改进它们的程序之间的循环。

## Abstract
An agent system's execution traces record how it fails, and procedures that improve such a system without changing model weights (trajectory selection, prompt and workflow optimization, runtime monitoring) read these traces for feedback. Yet raw traces are a poor medium for accumulating that feedback: long, instance-specific, and lacking a stable vocabulary for recurring failures. We argue that an agent system should instead maintain an explicit representation of how it fails, induced from its own behavior and reusable wherever failure feedback is needed. AdaMAST builds this representation by converting a target system's traces into a compact, evidence-grounded failure taxonomy: named failure codes organized along three fixed axes (system-level, role-specific, and domain-specific), with every name, definition, and evidence pattern induced from the traces; no code is hand-authored, no trace human-annotated. The taxonomy is not merely a post-hoc diagnostic but a shared feedback interface, improving agents in three ways. In agent-system search, taxonomy-coded diagnoses of failed candidates outperform free-form reflection on all five benchmarks we test. At runtime, taxonomy feedback raises SWE-agent's resolution on SWE-bench Verified Mini from 60% with free-text reflection to 70%, and improves Claude Code from 64.0% to 70.7% as a runtime skill. In trajectory selection, AdaMAST-Judge, a verifier built on the induced codes, improves best-of-5 accuracy on Terminal-Bench 2.0 by 8-15 points over Pass@1. The vocabulary itself is compact (an order-of-magnitude compression that preserves trace distinctions), human-faithful (matching expert failure annotations more closely than a hand-crafted reference vocabulary), and adaptive (taxonomies induced for different domains share few codes). Adaptive failure taxonomies close the loop between the traces agents produce and the procedures that improve them.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 从目标系统的执行轨迹中自动归纳出**紧凑的、基于证据的失败分类法**，包含命名失败代码（名称、定义、证据模式），沿三个固定轴组织：
  - **A：系统级**（系统级架构、编排失败）
  - **B：角色特定**（多智能体架构中特定角色的失败）
  - **C：领域特定**（任务领域的推理/知识错误）
- 无需人工编写代码，无需逐条轨迹人工标注。
- 分类法一旦生成并验证，即可被多个下游程序无修改地使用，代替原始轨迹池。

### 关键技术细节
1. **生成分类法的四阶段管道**（共8次LLM调用）：
   - **分析阶段**：提取领域上下文、发现代理角色、识别反复行为信号。
   - **策展阶段**：分别沿A、B、C轴生成候选失败代码。
   - **整合阶段**：去重、修复命名违规、拒绝无证据代码、确保每个角色/子域有覆盖。
   - **注释者间一致性门**：四个LLM标注者独立标注保留轨迹，使用五阶段协商协议，目标均配对Cohen's κ ≥ 0.75（区域级）且覆盖≥0.70，否则触发合并/添加/重新标签编辑并重新运行。
2. **在线精炼**：当目标系统或轨迹分布变化时，可调用精炼轮次（合并、添加、重新标签）。
3. **三个消费者**：
   - **智能体系统搜索**（AdaEvolve）：突变提示接收基于分类法的诊断（触发代码+引用证据+跨问题模式）。
   - **运行时监控**：分类法进入智能体上下文，在检查点触发有限修复循环（≤3次尝试）。
   - **轨迹选择**（AdaMAST-Judge）：每个代码转为验证标准（语义失败→LLM评分提示，结构签名→启发式检查），并通过前向选择器保留最具判别力的标准。

## 7. 优点

1. **创新性**：首次提出将故障分类法作为持久基础设施，而非一次性事后工具，实现反馈的复用和摊销。
2. **通用性**：方法适用于任意智能体系统（单/多智能体），仅轴固定，内容自适应；下游程序无需修改。
3. **自动化和可扩展**：无需人工标注，仅需少量LLM调用；在线精炼支持系统演化。
4. **可验证性**：注释者间一致性门提供部署前质量保证；TRAIL外部验证确保人类相关性。
5. **压缩效用**：分类法可替代轨迹池作为上下文，节省1-2个数量级上下文令牌。
6. **实验设计公平**：在所有对比中严格保持周围程序固定，仅变化反馈内容，清晰分离词汇表效果与程序效果。

## 8. 不足与局限

1. **依赖LLM质量**：分类法生成、验证、消费者均依赖LLM（GPT-5.4-mini等），可能继承LLM的偏见或错误；未评估使用更弱/更强LLM的影响（仅部分跨模型复制）。
2. **人类验证有限**：TRAIL外部验证仅117条轨迹，且金标准为四位专家标注（可能存在主观性）；分类法的人类一致性尚未在大规模真实部署中验证。
3. **轨迹选择的结果解释**：Terminal-Bench结果通过特征选择器+启发式检查的中介，词汇表的直接贡献难以完全隔离（文中承认该点是部署模式研究，非纯净词汇表效果）。
4. **计算成本未完整报告**：缺乏GPU型号、训练时长、总计算量等细节，重复性受限（虽然每次运行成本小）。
5. **小基准敏感性**：TheoremQA、DROP等小基准种子变异性可能影响结论稳定性（已通过多种子部分缓解）。
6. **未评估对模型权重更新方法的适用性**：论文聚焦于不改变权重的程序，但未讨论分类法是否可用于强化学习/微调等场景。
7. **领域覆盖有限**：实验领域包括编程、数学、科学QA、终端任务，但未涵盖对话、规划、机器人等，通用性有待验证。
8. **可能过度拟合特定轨迹分布**：分类法从自身系统轨迹归纳，若系统性能变化剧烈（如换模型），可能需要频繁精炼。
