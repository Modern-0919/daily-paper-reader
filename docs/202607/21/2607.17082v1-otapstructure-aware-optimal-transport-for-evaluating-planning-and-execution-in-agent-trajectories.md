## 摘要
大型语言模型智能体通过生成交织了规划、工具调用和中间结果的轨迹来完成任务。当前的评估指标要么将轨迹简化为一个二元成功标志，要么通过与参考轨迹进行精确匹配来比较。成功标志无法区分可靠的解决方案与侥幸成功的方案，也无法说明失败运行出错的原因。精确匹配会惩罚那些有效但与参考轨迹顺序不同或分解方式不同的规划。我们将轨迹评估重新定义为智能体执行图与一组有效解图之间的距离，并通过在属性依赖图上求解非平衡融合Gromov-Wasserstein输运问题来实例化该距离。由此产生的评分称为\otap{}（面向智能体规划的最优传输），是一个伪度量，可证明对保持依赖关系的重新排序具有不变性，并对冗余步骤具有有界敏感性。其非平衡边际处理缺失或幻觉步骤而无需强制匹配，其软耦合适应规划粒度的变化。在受控扰动和三个公开基准测试中，当仅基于语义的指标得分低于随机水平时，\otap{}能够将有效轨迹与无效轨迹区分开来。当依赖图被精确恢复时，其准确率最高，仅当从自由文本轨迹中启发式推断图时准确率才会下降。

## Abstract
Large language model agents solve tasks by generating trajectories that interleave planning, tool calls, and intermediate results. Current evaluation metrics reduce such a trajectory to a binary success flag or compare it against a reference by exact matching. A success flag cannot distinguish a sound solution from one that succeeds by luck, and says nothing about why a failed run went wrong. Exact matching penalizes plans that are valid but reordered or decomposed differently from the reference. We reframe trajectory evaluation as a distance between the agent's execution graph and a set of valid solution graphs, and instantiate it via an unbalanced fused Gromov-Wasserstein transport problem over attributed dependency graphs. The resulting score, termed \otap{} (Optimal Transport for Agentic Planning), is a pseudo-metric that is provably invariant to dependency-preserving reorderings and has bounded sensitivity to redundant steps. Its unbalanced marginals handle missing or hallucinated steps without forcing a match, and its soft coupling accommodates variation in plan granularity. On controlled perturbations and three public benchmarks, \otap{} separates valid from invalid trajectories in a regime where semantics-only metrics score below chance. Its accuracy is highest when the dependency graph is recovered exactly, and drops only when the graph is inferred heuristically from free-text traces.

核心问题：智能体执行图（attributed DAG）与**一组有效解图**之间的距离应该如何度量？论文主张从“检查是否复制参考序列”转向**图结构对齐**。

## 2. 论文提出的方法论

### 核心思想

### 关键技术细节
1. **轨迹图表示**：每个步骤 \( s_i = (a_i, \tau_i, c_i, e_i) \) 包含动作描述、工具ID、上下文参数、执行效果。边表示数据或因果依赖。通过三种机制构建图：从plan-and-execute提取显式依赖图、从ReAct序列追踪、或退化为全序。
2. **节点成本矩阵**：对各字段独立编码，计算余弦距离，加权组合（含工具等价性距离和状态变化距离）。
3. **结构矩阵**：归一化符号化的成对依赖距离，正号表示祖先→后代，负号相反，零表示不可比较。
4. **UFGW目标函数**：
   - 第一项：节点属性对齐成本（Wasserstein项）
   - 第二项：结构距离差异平方和（Gromov-Wasserstein项）
   - 第三、四项：KL散度松弛边际约束（处理缺失/幻觉步骤）
   - 第五项：熵正则化加速计算

5. **多参考聚合**：通过温度控制的soft-min函数（式5），T→0时为最小操作，T→∞为均值。
6. **因果质量加权**：根据步骤在参考DAG中的关键性（移除是否破坏可达性）赋予不同质量。
7. **最终评分**：\( \exp(-L(P,R)) \in (0,1] \)，并输出精度/召回率（π, ρ）和最优传输计划。

## 6. 论文的主要结论与发现

- **Otap在区分有效/无效轨迹上显著优于所有基线**：在受控扰动中AUROC达0.820（最高基线0.674），在PlanBench、WebArena、GAIA上也领先（AUROC分别为80.4、85.9、69.8）。
- **结构项是关键**：去掉结构项（θ=0）后AUROC从82.0降至54.5（接近随机）。
- **多参考聚合至关重要**：单参考AUROC下降16.9个点。
- **依赖图精确性决定性能**：在PlanBench（精确PDDL）和WebArena（显式模板）上表现最好；在GAIA（启发式提取）上优势缩小但仍是最佳。
- **因果反转难以被纯文本指标检测**：Embedding基线在所有语料上AUROC低于随机，因为反转不改变词汇。
- **性能与计划长度温和相关**：短轨迹毫秒级，长轨迹（GAIA）需数十秒，但仍在可接受范围。
- **Otap不能替代形式化验证器**：在PlanBench oracle研究中，Otap AUROC（79.2）低于BLEU（86.7），说明当存在精确验证器时应优先使用验证器。

## 7. 优点

1. **结构感知**：通过Gromov-Wasserstein项显式建模依赖关系，对保持依赖的重排序不变（Proposition 2）。
2. **柔性对齐**：不平衡边际自然处理缺失/幻觉步骤，产生精度/召回率诊断。
3. **多参考支持**：软最小聚合避免单一参考偏向。
4. **可分解诊断**：提供属性、结构、边际各子损失，便于调试。
5. **伪度量理论保证**：满足非负、对称、三角不等式，且图同构时距离为零（Proposition 1）。
6. **计算可负担**：短轨迹毫秒级，比LLM裁判成本低且确定性强。
