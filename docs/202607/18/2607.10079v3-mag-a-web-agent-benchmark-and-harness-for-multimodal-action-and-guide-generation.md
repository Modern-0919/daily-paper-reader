## 摘要
数字采用平台（DAPs）是广泛嵌入在网页系统中的覆盖层，用于引导用户在页面内进行操作，帮助用户快速熟悉陌生界面。然而，完成一项实际任务很少意味着在单个页面上点击几个按钮：它需要一系列跨页面状态变化的操作序列。以往的研究也将自动化网络智能体操作与指南文本生成视为两个独立的问题，且大多数研究向模型输入文本化的页面表示（如DOM或可访问性树），而非人类实际操作的渲染屏幕。在本工作中，我们提出了MAG，这是首个将任务执行与指南编写统一为单一多模态动作与指南任务的基准，并基于截图提供了两种标注方案：标记集（Set-of-Mark）元素选择与原始像素坐标。此外，我们为这一复合任务构建了完整的测试框架，涵盖基于LLM辅助与人工验证的标注、训练、实时环境评估以及动作与指南的联合评估指标。利用该框架，我们评估了前沿API模型与开源多模态模型，并进行了详细分析。最后，我们设计了一种结合专家轨迹的GRPO训练方法，使监督微调的9B智能体成功率几乎翻倍（从6.9%提升至13.2%），同时提高了指南质量。即使是最强模型，任务完成率也低于40%，为未来研究留下了充足空间。

## Abstract
Digital Adoption Platforms (DAPs) are embedded overlays widely used on web systems to guide users through operations inside a page, helping them get started with unfamiliar interfaces quickly. Completing a real task, however, rarely means clicking a few buttons on a single page: it takes a sequence of actions that unfolds across changing page states. Prior studies have also treated automated web agent actions and guide text generation as two separate problems, and most of them feed models textual page representations such as the DOM or accessibility trees rather than the rendered screens that humans actually operate on. In this work we introduce MAG, the first benchmark that unifies task execution and guide writing into a single Multimodal Action and Guide task, with two grounding schemes over screenshots: Set-of-Mark element selection and raw pixel coordinates. We further build a complete harness for this compound task, covering annotation with LLM assistance and human verification, training, evaluation in live environments, and joint metrics for actions and guides. With this harness we evaluate frontier API models and open multimodal models, and report detailed analyses. Finally, we design a GRPO training method augmented with expert trajectories, which nearly doubles the success rate of a supervised 9B agent (from 6.9% to 13.2%) and improves guide quality at the same time. Even the strongest model completes fewer than 40% of the tasks, leaving ample room for future research.

- **核心问题**：如何让一个单一的智能体同时完成多步骤网页任务（实时环境、截图观测）并在每一步生成可供用户复用的引导文本？如何公平地评价这种双输出任务？

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 智能体在每个时间步 \( t \) 同时输出动作 \( a_t \) 和引导文本 \( g_t \)，动作基于截图（Set-of-Mark 元素编号或原始像素坐标），引导必须为用户可理解的纯文本（不包含标记ID或坐标）。
- 最终输出包含两个产物：可验证的任务结果（是否成功）和可部署的引导序列。
- 训练分为两阶段：SFT 学习输出契约和引导风格；专家增强 GRPO 通过注入缓存的外部专家轨迹解决常规 GRPO 在全失败组中梯度消失的问题。

### 关键技术细节
- **任务定义**：`(q, s_0, Φ)` 表示意图、初始页面状态和功能检查器。观测 `o_t` 为 1440×900 截图和候选菜单（包含标记元素列表）。动作空间包含 7 种动词（click/type/select/scroll/press_enter/go_back/finish），引导文本需独立于标记ID/坐标。
- **两种 grounding 方案**：Set-of-Mark（SoM）选择标记ID，或原始像素坐标。其他条件（提示、观测、评估指标）完全相同，用于公平对比。
- **数据集构建**：基于 WebArena 的 581 个成功轨迹（来自 OpAgent），先重放捕获 SoM 截图，再让 GPT-5.5 为每一步起草 think 和 guide，经过规则过滤和人工修正后得到 563 个已注释任务（4,760 SoM 步，5,779 坐标步）。
- **SFT（阶段 1）**：对 9B VLM 进行全参数微调，损失仅作用于输出 JSON 标记，目标：使输出符合契约（OFCR 从~19% 提升至~97%）并学会引导的祈使语态。
- **专家增强 GRPO（阶段 2）**：
  - 预先用 GPT-5.5 在训练任务上运行两遍，仅缓存成功轨迹（139 个任务，最多每个任务 2 条）。
  - 分组：每个被覆盖的任务 \( q \) 有 6 条在线策略 rollout + 至多 2 条缓存专家轨迹 → 组大小 \( G = 6 + |E_q| \)。
  - 奖励 \( R_i \in \{0,1\} \) 为任务成功；优势 \( A_i = R_i - \frac{1}{|G|}\sum_j R_j \)，无标准化；零方差组丢弃。
  - 损失函数：PPO 式的 token 级 clipped 目标，不对称 clip（\( \varepsilon_l=0.2, \varepsilon_h=0.28 \)），无 KL 项。
  - 专家轨迹的 token 概率使用策略 round-start log-probs 计算重要性比（off-policy）。
  - 训练共 10 轮，前几轮优先调度专家覆盖的任务，后续轮次只用策略自卷。

## 6. 主要结论与发现

1. **MAG 任务远未解决**：最佳配置（GPT-5.5 坐标）仅完成 37.4%；最佳 9B 智能体（GRPO SoM round 10）达到 13.2%。所有模型均存在引导税（SR - GGS 约 10-15 个百分点）。
2. **输出契约可学习，但任务能力难习得**：SFT 将 OFCR 从 ~20% 提升至 ~97%，但成功率几乎没变甚至下降（SoM 从 8.0% 降至 6.9%）。
3. **Grounding 偏好因模型而异**：Gemini 在 SoM 下显著优于坐标（+13.8 点）；GPT-5.5 和 Claude 无明显偏好；9B 模型仅在 SoM 下通过训练获得提升（坐标无改善）。
4. **专家增强 GRPO 是唯一有效的训练方法**：10 次无专家的 GRPO 配置均无持续增益；注入专家后 SoM 成功率从 6.9% 提升至 13.2%，GGS 翻倍（.036→.076），且引导质量同步提升（奖励从未直接优化引导）。
5. **能力提升来自新任务解决，而非锐化**：pass@6 从 14.4%（SFT）升至 21.3%（GRPO），CRPO 策略在采样时仍保持性能（pass@1 接近贪心）。
6. **长任务仍是主要瓶颈**：所有 9B 模型在超过 8 步的黄金轨迹上成功率接近 0，而 API 模型仍有 20-38%。
7. **引导与成功耦合**：成功轨迹的引导质量显著高于失败轨迹（GRPO round 10: .369 vs .256），且该差距随训练增强。

## 7. 优点

- **任务创新性**：首次将多步骤网页动作执行与指南生成统一为单一基准，并针对双输出设计联合评估指标（GGS），更贴近实际 DAP 产品需求。
- **公平的 grounding 对比**：在完全相同的任务、观测、提示下比较 Set-of-Mark 和原始坐标，控制变量严格，结论可信。
- **专家增强 GRPO 方法**：巧妙利用缓存成功轨迹解决全失败组梯度消失问题，是 off-policy guidance 在多模态网页 RL 领域的有效应用。
- **完整的工程框架**：从 LLM 辅助标注+人工验证、训练管线、实时评估到指标计算，全部开源，包含稳定的环境回滚、认证、并行 rollout 等细节，大幅降低复现难度。
- **深入的分析**：pass@k、置信区间、消融行动分布、长任务分割等，不仅报告结果，还深入剖析提升来源和剩余差距。

## 8. 不足与局限

- **测试集规模小**：仅 174 个任务，单点差异的统计不确定性较大（±2 点左右），部分对比可能处于噪声范围。
- **引导评估单参考**：使用 n-gram 重叠（BLEU/ROUGE）且只与一条黄金路径比较，无法奖励有效替代路径，低估了实际有用性。
- **任务来源单一**：所有任务来自 WebArena 的 6 个网站且大量共享模板，训练与测试模板高度重叠，泛化性（新网站、新任务模板）尚未验证。
- **领域偏差**：训练数据来自 OpAgent 先验成功轨迹，基准天然偏向该智能体可解的任务，无法代表 WebArena 的全部难度。
- **GRPO 结果基于单个种子**：未报告多次运行的平均值和标准差，仅有一个种子，稳健性证据有限。
- **专家模型单一**：仅使用 GPT-5.5 作为专家，未探索其他模型作为专家源的影响。
- **9B 模型以外的训练**：仅对 Qwen3.5 9B 进行了微调，未尝试更大模型或不同架构。
- **现实部署偏差**：自沙盒环境与实际网站存在差异，引导质量人工检验仅覆盖 50 例，且“有用率”由单作者评定，可能存在主观性。
