## 摘要
基于大语言模型的代理系统的失败归因，即识别失败轨迹中哪些步骤导致任务失败，对于调试和改进这些系统至关重要。现有方法要么依赖基于提示的流水线（计算成本高昂），要么需要对带有步骤级错误标注的失败轨迹进行后训练（收集成本高且难以扩展）。我们认为，一个实用的失败归因模型应当是轻量级的，并且能够在无需失败数据步骤级监督的情况下进行训练。为此，我们研究无监督失败归因，即仅使用成功轨迹进行训练，并在推理时给定失败轨迹识别错误步骤。我们提出OAT，将这一问题转化为使用神经控制微分方程的单类学习，在潜在空间中对成功轨迹的动态模式进行建模。推理时，失败轨迹中的每个步骤根据其与从成功轨迹学到的动态模式的偏差被赋予一个异常分数，进而用于形成一组错误步骤。实验表明，仅使用100条成功轨迹进行训练，OAT比基于提示的基线方法快200-5000倍，同时在域内和域外分布数据集上持续超越它们，F1分数分别提升+20%和+7%，证明OAT是诊断代理系统故障的一个有前景且高效的方案。

## Abstract
Failure attribution for LLM-based agentic systems, i.e., identifying which steps in a failure trajectory caused the task to fail, is critical for debugging and improving these systems. Existing approaches either rely on prompting-based pipelines, which are computationally expensive, or require post-training on failure trajectories with step-level error annotations, which are costly to collect and difficult to scale. We argue that a practical failure attribution model should be lightweight and trainable without step-level supervision on failure data. To this end, we address unsupervised failure attribution, i.e., training exclusively on successful trajectories and identifying error steps at inference time given a failure trajectory. We propose OAT, which casts this problem as one-class learning with neural controlled differential equations, modeling the dynamical pattern of successful trajectories in latent space. At inference time, each step in a failure trajectory is assigned an anomaly score based on its deviation from the dynamics learned on successful trajectories, which is then used to form a set of error steps. With training on only 100 successful trajectories, experiments show that OAT is 200--5000 $\times$ faster than prompting-based baselines, and, at the same time, consistently outperforms them in both in-domain and out-of-distribution datasets with +20% and +7% F1 scores, respectively, demonstrating that OAT is a promising and efficient direction for diagnosing agentic system failures.

- **问题**：LLM 代理系统的故障诊断（失败归因）需要定位失败轨迹中的具体错误步骤，是调试和改进的关键。现有方法要么依赖昂贵的提示流水线（需调用前沿 LLM），要么需要大量带步骤级错误标注的失败数据进行后训练，标注成本高且难以规模化。
- **动机**：作者提出 **无监督失败归因** 的新问题设定——仅使用成功轨迹训练，在推理时对失败轨迹识别错误步骤。成功轨迹容易收集，无需任何步骤级标注。
- **意义**：该设定大幅降低了部署门槛，为实时、轻量级诊断代理系统故障提供了新方向。

### 2. 论文提出的方法论

- **核心思想**：将无监督失败归因转化为 **单类学习** 任务，在 LLM 隐表示空间中建模成功轨迹的动态模式，推理时计算每个步骤与正常动态的偏离程度（异常分数），从而定位错误步骤。
- **关键技术细节**：
  - **步骤表示提取**：对每个步骤，使用生成轨迹的 LLM（Qwen3.5-27B）的最后一层隐藏状态做平均池化，得到固定维度向量 `ht`。对查询也提取表示 `hQ`。
  - **连续轨迹建模**：将步骤序列视为连续隐路径的观测值。采用 **神经控制微分方程（Neural CDE）**：先通过三次样条插值步骤表示构建控制路径 `X(u)`，其导数 `dX/dv` 在每个积分步驱动隐动态 `z(u)` 的演化。训练损失为预测表示 `\hat{h}(u_t)` 与真实表示 `h_t` 的 L2 距离。
  - **门控控制路径**：为应对分布外（OOD）场景，引入一个轻量门控网络 `q`，根据控制路径导数的异常程度自适应地抑制其信号，提高鲁棒性。
  - **推理检测**：
    - 对失败轨迹，用训练好的 Neural CDE 预测每一步的期望表示，计算重建误差作为异常分数 `e_t`。
    - 两种检测策略：① **Top-k**：固定 k，选择分数最高 k 步；② **置信预测（CP）**：用保留的成功轨迹校准阈值 `δ`，超过阈值的步骤视为错误。CP 提供分布无关的覆盖保证。

### 3. 实验设计

- **数据集与场景**：
  - **域内（ID）**：MCP-Atlas（工具调用基准），基于 Qwen3.5-27B 生成，含 103 条成功轨迹（训练用）和 88 条失败轨迹（测试），人工标注错误步骤。
  - **域外（OOD）**：Who&When（多代理协作基准），由 GPT-4o 生成，共 184 条失败轨迹，自带步骤级标注。该数据集与 MCP-Atlas 在代理系统、基座模型、任务类型上差异很大。
- **基准方法**：
  - 随机选取一步、第一步、基于 GPT-4o 和 GPT-5 的 Prompt 方法（修改自 Zhang et al. [10] 的 all-at-once prompt，输出候选错误步骤）。
- **评估指标**：精确率、召回率、F1、命中率（是否至少命中一个错误步）、AUROC、AUPRC。
- **实现细节**：Neural CDE 向量场为 3 层 MLP（隐层维度 64），控制路径用自然三次样条，欧拉求解器。PCA 降维至 64 维。训练仅用 80% 成功轨迹（约 82 条），其余 20% 用于验证/校准。

### 7. 优点

- **轻量级**：仅需轻量 3 层 MLP，<1GB VRAM，训练仅需百条成功轨迹。
- **无监督**：无需任何失败数据或步骤级标注，成功轨迹容易收集。
- **高效**：推理零 token 成本，延迟毫秒级，支持实时诊断。
- **通用性**：能跨域（不同代理架构、基座模型、任务类型）泛化，且支持代理 LLM 设置。
- **可解释**：提供步骤级异常分数，可辅助人工审查，并能捕获未标注的早期异常信号。

### 8. 不足与局限

- **固定阈值问题**：置信预测的硬阈值可能漏掉因果重要但分数略低于阈值的步骤（如案例 3）；Top-k 则需预设 k 值，缺乏自适应性。
- **早期推理错误检测不足**：对隐含在推理文本中的早期错误敏感度较低（如案例 2 中的第一步），容易漏检。
- **依赖 LLM 表示质量**：步骤表示的质量直接影响模型性能；若使用不同基座模型（代理 LLM），性能可能轻微下降。
- **实验范围限制**：仅在 MCP-Atlas 和 Who&When 两个基准上测试，任务类型有限；未在更广泛的代理系统（如多轮对话、复杂环境交互）上验证。
- **标注主观性**：MCP-Atlas 的错误步骤由作者人工标注，虽经交叉验证，但仍存在一定主观性和歧义，可能影响评估公平性。
- **未探索更高级的 NDE 变体**：论文仅使用标准 Neural CDE，未尝试 Normalizing Flow、Neural SDE 等可能提升性能的扩展。
