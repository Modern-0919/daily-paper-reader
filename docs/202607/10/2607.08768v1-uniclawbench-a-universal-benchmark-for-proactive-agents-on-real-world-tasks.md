## 摘要
大型语言模型和多模态大型语言模型的快速发展加速了能够操作日常工具并在真实环境中协助用户的主动智能体的出现。然而，现有基准难以有效评估此类智能体，因为它们通常依赖于沙盒环境和单轮评估范式。此外，基于场景的任务分类将多种模型能力混杂在同一任务类别中，使得难以识别智能体失败的根本原因。为解决这些限制，我们提出了UniClawBench，这是首个在动态真实世界环境中评估主动智能体的能力驱动型基准。UniClawBench围绕五项基础模型能力构建：技能使用、探索、长上下文推理、多模态理解和跨平台协调。基于这些能力，我们设计了400个双语真实世界任务。与以往依赖静态预录答案的基准不同，我们的基准在实时Docker容器中通过细粒度的逐步完成检查点来评估智能体。此外，我们设计了一个闭环评估策略，包括一个执行智能体、一个隐藏监督智能体和一个用户智能体，以模拟真实的多轮人类反馈，同时避免泄露评分标准。为区分基础模型能力与框架级设计选择，我们在多种智能体框架下评估了最先进的模型。通过对模型和框架的全面比较，我们展示了基础模型能力和智能体框架设计如何共同影响真实世界环境中的性能。为促进未来研究，我们在https://github.com/HKU-MMLab/UniClawBench公开了我们的基准和代码。

## Abstract
The rapid development of large language models and multimodal large language models has accelerated the emergence of proactive agents capable of operating everyday tools and assisting users in real-world environments. However, existing benchmarks struggle to evaluate such agents effectively, as they often rely on sandboxed environments and single-turn evaluation paradigms. Moreover, their scenario-based task taxonomies mix multiple model capabilities within the same task category, making it difficult to identify the root causes of agent failures. To address these limitations, we introduce UniClawBench, the first capability-driven benchmark designed to evaluate proactive agents in dynamic, real-world settings. UniClawBench is built around five foundational model capabilities: Skill Usage, Exploration, Long-Context Reasoning, Multimodal Understanding, and Cross-Platform Coordination. Based on these capabilities, we design 400 bilingual real-world tasks. Unlike previous benchmarks that rely on static, pre-recorded answers, our benchmark evaluates agents in live Docker containers using fine-grained, step-by-step completion checkpoints. Furthermore, we design a closed-loop evaluation strategy comprising an executor agent, a hidden supervisor agent, and a user agent to simulate realistic multi-turn human feedback without leaking grading criteria. To disentangle base model capabilities from framework-level design choices, we evaluate state-of-the-art models under multiple agent frameworks. Through comprehensive comparisons across both models and frameworks, we show how base model capabilities and agent framework designs jointly shape performance in real-world environments. To facilitate future research, we make our benchmark and code publicly available at https://github.com/HKU-MMLab/UniClawBench.

- **研究动机**：大语言模型和多模态大语言模型已发展为能在真实环境中执行复杂多步任务的自主智能体（proactive agents）。然而，现有的评估基准存在三个结构性缺陷：
  - **环境不真实**：大多依赖沙盒（如WebArena、OSWorld）或静态缓存页面，与实际部署存在鸿沟。
  - **评估范式陈旧**：采用单轮评估，忽略人类与智能体之间迭代反馈的闭环本质。
  - **任务分类混淆**：按应用场景（如“办公”“研究”）组织任务，导致多种能力混杂（如视觉感知、长上下文推理、工具使用），失败时难以定位根本原因。
- **整体含义**：亟需一个能力驱动的、运行在真实动态环境中的基准，能够诊断智能体失败的具体能力瓶颈，并支持多轮人机交互评估。

## 2. 论文提出的方法论：核心思想与关键技术

- **核心思想**：将智能体能力解耦为五个基础维度，设计能力导向的任务，并通过隐藏检查点评估和闭环比策略模拟真实交互。
- **关键技术细节**：
  - **能力导向任务分类**：
    - **技能使用（Skill Usage）**：要求智能体正确选择、调用并执行特定工具/API（如OCR、SQLite查询、Git审计）。
    - **探索（Exploration）**：在信息不完整或误导的情况下进行开放式调查（如反向工程未文档化的Web API、审计配置冲突）。
    - **长上下文推理（Long Context）**：跨异构来源聚合并维持一致性证据（如综合多个网页、PDF、视频）。**本质困难在于全局一致性与冲突消解**，而非单文档长度。
    - **多模态理解（Multimodal）**：从图像、视频、音频中提取并生成基于非文本内容的信息（如从视频中截图、重构图表）。
    - **跨平台协调（Cross-Platform）**：同步和编排异构应用间的状态和信息（如PDF→Zotero→Obsidian的工作流）。
  - **任务设计**：人工构建400个双语任务（每种能力40个英文+40个中文），每个任务为一个完整的“任务包”，包含指令、上下文、输入文件/资源、工具、技能、预期输出和隐藏评估参考。
  - **闭环评估策略（Three-role Closed-loop Evaluation）**：
    - **执行者（Executor）**：被评估的智能体，运行在真实Docker容器中，仅获得公开任务包（工具、文件、网页访问）。
    - **隐藏监督者（Supervisor）**：基于隐藏的逐步骤检查点评分标准，全面评估轨迹和产物，返回通过/失败/继续状态及分数。**仅向用户模拟器传递粗略的进度信号**（如通过/可恢复/分数），不暴露评分细则。
    - **用户模拟器（User Simulator）**：模拟真实用户，仅基于可见轨迹和进度信号生成自然语言反馈，引导执行者迭代改进。**通过信息防火墙隔离**，避免泄露答案或评估标准。
  - **动态评估机制**：放弃预录答案，采用细粒度的检查点（step-by-step completion checkpoints），使自动监督者知道每一步应检查哪些中间推理和执行证据。
  - **执行系统**：基于Docker容器，提供真实浏览器、命令行、GUI应用、本地文件和服务。任务有全局超时（常规30分钟，长上下文45分钟）和单轮超时（常规20分钟，长上下文30分钟）。
- **公式/算法流程**：论文未涉及数学公式，但流程可描述为：
  - 初始化：准备任务包、隐藏参考、运行环境。
  - 循环（最多2次用户跟进）：
    1. 执行者在Docker中执行任务，保存轨迹和产物。
    2. 监督者评估得分和状态（pass/fail/continue）。
    3. 若为continue，用户模拟器根据可见信息生成反馈，反馈经改写后发送给执行者，进入下一轮。
  - 终止：达到pass、fail不可恢复、跟进次数上限或超时。
  - 输出最终指标：任务级通过率（Pass Rate）和检查点平均分数（Average Score）。

## 3. 实验设计

- **数据集/场景**：UniClawBench基准本身，包含400个真实世界双语任务，覆盖5种能力维度，每个任务运行在实时Docker容器中（包含真实网站、桌面应用、文件系统等）。
- **对比方法**：
  - **模型对比**：在相同框架（OpenClaw v2026.3.11）下评估10个模型——Kimi-2.6、Qwen-3.5-Plus、GPT-4.1、GPT-5.4-Mini、Gemini-3.0-Flash、Gemini-3.1-Flash-Lite、Claude Sonnet-4.6、Gemini-3.1-Pro、GPT-5.4、Claude Opus-4.8。
  - **框架对比**：在GPT-5.4、Claude Opus-4.8、Kimi-2.6三个模型上，评估三种框架的性能——OpenClaw、EDICT、Nanobot。
  - **评估指标**：通过率（Pass Rate, PR）和平均分（Average Score, AS）。
  - **可靠性验证**：随机抽取50个完成轨迹，由3名人类专家独立评估PR（多数投票）和AS（平均分数），与自动评估比较：PR一致性92.0%，AS的Pearson r=0.71，Spearman ρ=0.68。

## 6. 论文的主要结论与发现

- **框架选择比模型选择对任务性能影响更大**：OpenClaw（单智能体架构）整体通过率最高，EDICT（多智能体编排）平均分高但通过率低（协调摩擦导致），Nanobot（极轻量）令牌效率高但复杂任务性能受限。
- **长上下文和多模态理解是当前智能体最大的瓶颈**：所有模型在Skill Usage和Exploration上表现相对较好，但在Long Context、Multimodal和Cross-Platform上通过率显著下降。
- **“半途失败”现象严重**：多数模型获得较高的中间检查点平均分，但最终通过率远低于平均分，表明智能体在长的执行链中常出现不可逆错误。
- **多轮用户反馈显著提升性能**：经过两轮交互后，通过率和累计最大分数逐步提升（图4c）。
- **闭源模型领先但差距缩小**：Claude Opus-4.8和GPT-5.4总体通过率最高（分别为0.475和0.407），但均未超过50%，凸显任务难度。开源模型（如Qwen-3.5-Plus、Kimi-2.6）已十分接近，甚至在某些维度超越部分闭源模型。
- **令牌消耗差异**：EDICT消耗令牌最多，Nanobot最少；长上下文任务输入令牌数最高（平均2.3M），探索任务最低（平均0.5M）。

## 7. 优点

- **创新性的能力驱动分类**：首次将任务按基础能力解耦，使失败诊断精确到具体维度，而非笼统的场景。
- **真实环境评估**：在实时Docker容器中使用真实软件、网页和服务，消除沙盒与真实部署的差距。
- **闭环多轮交互**：设计执行者-隐藏监督者-用户模拟器三重隔离角色，既模拟真实用户迭代反馈，又避免泄露评分标准（信息防火墙）。
- **逐步骤检查点评估**：适应动态环境（如网页价格变化），避免预录答案过时问题。
- **区分模型与框架的影响**：通过跨模型和跨框架对比实验，揭示了框架架构对性能的显著影响，为智能体系统设计提供指导。
- **全面的实验覆盖**：10个模型、3个框架、400个任务、5种能力，加上可靠性验证和令牌分析，使结论有较强支撑。

## 8. 不足与局限

- **任务规模有限**：仅400个人工构建任务，可能无法完全覆盖真实世界任务的多样性和复杂度。
- **实时环境不稳定**：网页、API等动态因素可能导致结果波动，增加评估噪声。
- **LLM评估偏差**：监督者和用户模拟器都基于GPT-5.4的Codex，可能引入系统性偏差或与执行模型重叠的偏差。
- **语言局限**：仅支持中英双语，未涉及其他自然语言场景。
- **框架版本限制**：实验基于特定版本的OpenClaw、Nanobot、EDICT，框架更新后结论可能变化。
- **硬件资源有限**：仅使用CPU和少量内存，未探究GPU加速或更大内存对性能的影响。
- **未研究训练过程**：基准仅评估推理性能，不涉及模型训练或微调策略，对模型能力增强的指导有限。
- **可能鼓励过度优化**：存在针对基准特性（如特定检查点）过拟合的风险，而非提升真实泛化能力。
