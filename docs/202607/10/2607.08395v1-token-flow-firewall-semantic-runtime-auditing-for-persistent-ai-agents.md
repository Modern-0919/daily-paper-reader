## 摘要
持久化AI代理将大型语言模型（LLMs）从单轮交互扩展为长期运行的软件系统。与传统聊天助手不同，这些代理中的不安全内容会通过持久化状态、可复用技能和工具中介交互传播，形成更大的语义攻击面。我们观察到，此类代理中大多数安全关键交互通过自然语言的token流传输，包括内存更新、工具参数、检索文件以及组件间通信。这一观察结果催生了一种新的安全方案：可以在不安全行为到达特权运行时宿点之前，将其拦截为有风险的语义流。基于此洞察，我们提出了TokenWall，一种作为代理token流语义防火墙的运行时防御框架。TokenWall对这些流进行边界感知的语义审计，构建结构化的源-宿审计记录，在执行前应用轻量级本地检查，并选择性地将模糊的高风险案例升级到更强的仲裁模块。与依赖稀疏审计或远程大模型监督的先前方法不同，TokenWall实现了全覆盖的执行前调解，同时减少了远程仲裁和延迟。在CIK-Bench上的实验表明，TokenWall将攻击成功率降至12.5%，同时在没有人工确认的情况下保持了97.4%良性可执行通过率。TokenWall在良性案例上仅引入0.69秒的额外延迟，表明语义运行时遏制可以为持久化AI代理实现实用的安全-效用折衷。

## Abstract
Persistent AI agents extend large language models (LLMs) beyond single-turn interaction into long-lived software systems. Unlike traditional chat assistants, unsafe content in these agents can propagate through persistent state, reusable skills, and tool-mediated interactions, creating a substantially larger semantic attack surface. We observe that most security-critical interactions in such agents are transmitted through natural-language token flows, including memory updates, tool arguments, retrieved files, and inter-component communications. This observation enables a new security formulation: unsafe behavior can be intercepted as risky semantic flows before reaching privileged runtime sinks. Based on this insight, we propose TokenWall, a runtime defense framework that acts as a semantic firewall over agent token flows. TokenWall performs boundary-aware semantic auditing over these flows, constructing structured source-sink audit records, applying lightweight local inspection before execution, and selectively escalating ambiguous high-risk cases to stronger arbitration modules. Unlike prior approaches that rely on sparse auditing or remote large-model oversight, TokenWall enables full-coverage pre-execution mediation while reducing remote arbitration and latency. Experiments on CIK-Bench show that TokenWall reduces attack success rate to 12.5% while maintaining a 97.4% benign executable pass rate without human confirmation. TokenWall further introduces only 0.69 seconds of additional latency on benign cases, demonstrating that semantic runtime containment can achieve a practical security-utility trade-off for persistent AI agents.

  - 基于规则的审计（如OpenGuardrails）高效但粗粒度，难以捕获隐式语义威胁。

- **核心洞察**：安全关键的状态转变都通过自然语言token序列在系统边界之间传递，因此可在token流**跨边界进入持久态或外部执行之前**进行预执行语义检测，从而阻断不安全行为。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将运行时安全中介建模为对**语义token流**的边界感知审计，在流到达受保护宿点（内存、工具、外部接口）前执行裁决。
- **Token流抽象**  
  形式化定义：  
  `f = (x, s, t, c, b)`  
  其中`x`为分段后的载荷（span），`s`为源，`t`为宿，`c`为运行时元数据，`b`为被跨越的边界类型。
- **审计阶段与组件**：
  1. **轻量级预检（Precheck）**：确定性规则过滤，如硬阻断（高风险模式：向外部暴露私密端点、破坏性操作无授权等）和软标记（为语义审计提供证据）。
  2. **本地小模型语义审计（Local Small Auditor，默认Qwen3-4B）**：
     - 将token流规范化为结构化源–宿记录，包含安全面类型、源/宿元数据、载荷span、预检信号、所有者意图摘要。
     - 输出结构化报告：决策（允许/重写并继续/推迟给人类/阻断）、重写后的载荷、5个归一化分数（风险、不确定性、可利用性、影响、重写完整性）、残差风险类别、是否需要大模型复核等。
     - 决策逻辑：对上下文流、权限流、能力流三类风险分别评估；允许或重写仅当风险可分离且保留良性语义。
  3. **升级仲裁（Fallback Arbiter，远程大模型）**：仅当小模型报告高不确定性/高风险/不完整重写时触发，做出最终决策。
- **算法流程**（见论文Algorithm 1）：
  1. 预检：若硬阻断则直接执行；否则进入小模型审计。
  2. 小模型审计：若提议重写则验证合法性，否则视为失败封闭（阻断）。
  3. 若满足升级谓词（表面特定条件+阈值），则调用大模型仲裁；否则直接执行小模型决策。
- **升级谓词**：基于表面类型、残差风险类型、可利用性、影响、重写完整性等，使用固定阈值表（见附录B.3）判断是否需要升级。

## 6. 主要结论与发现

- TokenWall在所有安全面上均达到最低ASR（总体12.5%），优于最强基线ClawKeeper（14.7%），同时大幅降低延迟（16.9s vs 64.3s）和人工干预率（33% vs 69.3%）。

- 架构消融证明各组件均至关重要：去小模型导致LCR升至94.3%、延迟74.7s；去升级使ASR升至20.4%；去重写增加HR和LCR。

- 结构化报告（含表面类型、不确定性、残差风险）对实现低LCR和高安全性至关重要；仅输出决策将导致ASR升至28.4%。

## 7. 优点

- **问题定义新颖**：将持久代理安全形式化为对跨边界token流的预执行审计，比动作级审计更根本。
- **实用性与安全兼顾**：通过三级流水线（规则→小模型→大模型）实现全覆盖预执行中介，同时控制成本和延迟。
- **轻量本地审计与选择性升级设计精巧**：利用小模型处理80%以上用例，仅在模糊/高风险时调用大模型，显著降低远程依赖和隐私暴露。
- **支持可恢复重写**：对可分离风险使用重写而非阻断或人工，保留任务连续性，降低人工率。
- **实验设计严谨**：消融充分、阈值固定、所有基线在统一环境下对比，使用LLM-as-judge减少人工偏差。
- **开源生态友好**：基于OpenClaw框架，方法可直接应用于现行代理系统。
