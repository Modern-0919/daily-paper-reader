# Agent Step Value: Probing the Observer Effect in Black-Box Traces

Abstract 

Final-answer scores hide which agent transi-tions helped or harmed a trace. We intro-duce Agent Step Value (ASV), a replay frame-work that scores before/after states with a state-less LLM evaluator over a fixed candidate set. ASV reports entropy movement and Bayesian surprise measure belief movement, while of-fline gold-margin gain measures movement toward a reviewed target. It also quantifies evaluator-channel sensitivity by replaying the same frozen transitions under changed projec-tion, rationale, prompt, or scoring rules. In a 100-question open-QA study with live PubMed retrieval and DeepSeek log-probability scor-ing, ASV evaluates 1,100 transitions. Entropy movement is 0.000 while mean Bayesian sur-prise is 2.693, exposing near-one-hot belief piv-ots. Under a 128-token rationale-conditioned protocol, mean gold-margin gain is -2.335 (95% CI [-3.395, -1.272]); direct one-token scoring on the same traces gives +4.033. A100-transition component audit traces the re-versal to short generated rationales over full states. ASV turns prompt sensitivity into a mea-sured channel effect and localizes the largest rationale-conditioned losses to extraction and audit. 

1 Introduction 

Language-model agents act through traces. They classify requests, plan, retrieve evidence, call tools, draft answers, audit intermediate products, and re-vise. Final-answer evaluations can rank systems, but they rarely say which transition made the state better or worse. That gap matters for black-box analysis because a retrieval call can add useful ev-idence, a synthesis step can drop it, and an audit step can change the answer while weakening the state that supported it. We introduce Agent Step Value (ASV) to mea-sure this transition object. Given an observed transi-tion (xi−1, a i, x i), ASV asks how a specified evalu-ator’s belief over candidate outcomes changes from the before-state to the after-state. The actor can be a ReAct-style loop (Yao et al., 2023), a DSPy-like stateless module graph (Khattab et al., 2024), or a structured domain workflow. ASV only requires bounded state projections and a fixed candidate set. A state-grounded LLM evaluator scores every candidate in every state and stores raw log scores. ASV then separates three questions often col-lapsed into one score, whether uncertainty changed, whether the belief distribution moved, and whether that movement favored a reviewed target over its nearest competitor. This matters because a con-fident wrong-to-right pivot and a confident right-to-wrong pivot can both have near-zero entropy movement. Evaluator deliberation creates a measurement problem. If a judge reasons before answering, first-token log probabilities score the reasoning pream-ble. ASV therefore separates deliberation from scoring, i.e., the evaluator first writes a compact label-free rationale over stable candidate identifiers, then a second call appends the physical option map-ping and scores one option token. The rationale here is part of the measurement protocol. ASV values are indexed by the protocol that pro-duces them. The transition is the object, while the prompt, rationale policy, projection, label map-ping, and scoring call form the measurement chan-nel. ASV reports protocol sensitivity by replay-ing frozen states under alternative channels and measuring signed shift, absolute shift, and sign dis-agreement. In our study, a short generated rationale reverses the measured direction of the same tran-sitions, while longer or extractive buffers reduce the reversal, making prompt sensitivity a measured channel effect. 2 Related work 

ASV connects tool-using agents, trajectory-level evaluation, process supervision, and LLM-as-judge evaluation. Tool-using and modular systems such as ReAct, Toolformer, ToolLLM, and DSPy orga-nize language-model applications as traces (Yao et al., 2023; Schick et al., 2023; Qin et al., 2024; Khattab et al., 2024). AgentBench, WebArena, Mind2Web, SWE-bench, AgentBoard, and Agent-ProcessBench evaluate behavior at the system or trajectory level (Liu et al., 2024; Zhou et al., 2024; Deng et al., 2023; Jimenez et al., 2024; Ma et al., 2024; Fan et al., 2026). ASV is therefore comple-mentary. When a workflow can export before/after states, it turns the trace into measurable evaluator belief updates without predefined progress labels. Process supervision gives the closest method-ological motivation. PRM800K labels intermediate reasoning steps (Lightman et al., 2024), and later process-reward or process-error benchmarks ex-tend step-level supervision to reward construction and error localization (Wang et al., 2024b; Zheng et al., 2025). Agent-as-a-Judge and AgentEval ex-tend step feedback to agentic workflows (Zhuge et al., 2025; Guo et al., 2026). ASV uses a smaller contract: a transition, a candidate set, and candidate log scores. Its damage/repair view also connects to Self-Refine and to evidence that intrinsic self-correction can degrade answers without external feedback (Madaan et al., 2023; Huang et al., 2024). LLM-as-judge work shows that language mod-els can approximate human preferences while re-maining sensitive to prompt format and option or-der (Liu et al., 2023; Zheng et al., 2023; Wang et al., 2024a); trained or rubric-conditioned judges further motivate evaluator reliability audits (Kim et al., 2024). Because ASV uses one-token op-tion log probabilities, it is also related to multiple-choice selection bias and first-token/text-answer mismatch (Zheng et al., 2024; Wang et al., 2024c). ASV treats the evaluator as a measurement instru-ment whose protocol must be specified and audited, recording raw scores, floor-score events, rationale leakage flags, and label mappings. Our retrieval-grounded open-QA setting relates to FEVER, KILT, PubMedQA, BioASQ, and RAG evaluation frame-works such as RAGAS and RAGChecker (Thorne et al., 2018; Petroni et al., 2021; Jin et al., 2019; Tsatsaronis et al., 2015; Es et al., 2024; Ru et al., 2024). ASV instead measures transition-level be-lief updates. The entropy and Bayesian-surprise diagnostics connect to calibration work (Yu et al., 2026) and accounts of surprise as belief change (Itti and Baldi, 2009). 

3 Agent step value 

ASV is a replay measurement procedure. It takes an observed trace, a fixed candidate set, and an eval-uator protocol, then computes how evaluator belief changes at each transition. It separates gold-free belief movement, offline target-directed movement, and evaluator quality flags, so the trace can be au-dited without treating the evaluator as ground truth.

Large protocol sensitivity limits claims about protocol-independent agent quality, but it is itself a black-box evaluation result, that the evalua-tor channel is changing how the same transition is read. 34 Experimental setup 

We use the experiment to answer two questions. First, can ASV localize which trace transitions change evaluator belief? Second, when the same frozen transition is replayed under different evalua-tor channels, which part of the channel carries the reversal? The study contains 100 open-QA tasks from intervention, risk, mechanism, diagnostic, and insufficient-evidence settings. Each task has four reviewed candidates. The source traces come from a PubMed evidence-retrieval workflow. We evaluated all 1,100 agent steps with a 128-token rationale-conditioned DeepSeek scorer, then re-played the same frozen states under label permu-tations and a direct one-token scorer. Appendix A gives the provider settings, actor coverage, storage format, and prompt details. For channel attribution, we replayed the 100 clean largest-gap transitions under alternate buffer policies and state slices. Clean excludes floor scores, fallback scores, missing option labels, and rationale leakage flags. 

5 Results 

5.1 The fixed protocol gives negative step values 

Table 1 gives the fixed-protocol insight. Mean gold-margin gain is -2.335 with trajectory-bootstrap 95% CI [-3.395, -1.272]. The estimate stays negative under B = 4 label permutations, after removing floor-score steps, and after removing the one tra-jectory containing all 11 gold-leakage warnings. Leave-one-trajectory-out means range from -2.467 to -2.180. Under this candidate set, projection, actor stack, and rationale-conditioned evaluator, many transitions move evaluator belief away from the reviewed target. Label order does not explain the fixed-protocol result. Direct one-token scoring gives +4.033 on the same 1,100 transitions, while the 128-token rationale-conditioned protocol gives -2.335. The mean signed channel effect is -6.368, the mean ab-solute effect is 14.853, and signs disagree in 70.1% of nonzero paired moves. After quality filtering, direct scoring remains positive (+3.939), rationale scoring remains negative (-2.221), and sign dis-agreement remains 68.1%. 

5.2 Entropy misses near-one-hot pivots 

Entropy movement is zero for every step, but mean Bayesian surprise is 2.693 and the largest sur-prise reaches 15.194. A temperature-scaling probe shows that, at the native scale, the mean top can-didate probability is 1.000 before and after transi-tions. At τ = 5 , it drops to about 0.948 and 12.9% of steps show nonzero entropy movement. Gold-margin gain supplies the missing direction that 77 steps are positive, 179 are negative, and 844 are unchanged. 

5.3 Loss localizes after evidence enters the trace 

The most negative step-type means occur at extract (-7.790), audit (-7.392), validate-plan (-4.664), and retrieval (-3.096); post-audit (0.758) and finalize (0.394) are mildly positive. Because medians are zero and extrema reflect saturated flips, we treat these means as localization hints and inspect dam-age/repair events. Figure 2 preserves this ordering under both the fixed layout and the B = 4 label-permutation au-dit, so physical option order does not explain the pattern. Under the rationale-conditioned protocol, extraction damages initially target-directed states more often than it repairs them (24% versus 4%); audit shows the same imbalance (22% versus 3%). The largest measured loss appears after evidence has entered the workflow and the agent begins com-pressing or criticizing its intermediate state. 

5.4 The reversal is carried by rationale compression 

Table 2 asks which evaluator-channel component carries the reversal. On the top-gap clean transi-tions, full-state direct scoring is positive on average (+7.750), while the 128-token generated-rationale channel is sharply negative (-21.741). The re-versal shrinks with a 512-token rationale (-1.981) and largely disappears with an extractive evidence buffer (+1.160). Evidence-only direct scoring re-mains positive (+5.421), while evidence-only 128-token rationale scoring is only mildly negative (-2.333). Candidate coverage does not explain the full-trace reversal. Coverage averages 0.546 over-all and is slightly higher in sign-disagreement steps (0.590). A cue-retention pass points to the same mechanism, that 63 of 100 top-gap transitions are labeled as compression loss, compared with 3 la-beled as state damage. 6 Discussion and limitations 

ASV is a replay diagnostic, and should not be treated as a causal reward estimator. Its claim is that a frozen trace, a fixed candidate set, and an explicit evaluator protocol are enough to measure local belief updates. The direct/rationale reversal is part of it, that a step value without its evalua-tor channel can mix actor behavior with artifacts introduced by projection, compression, option ren-dering, or scoring. The negative mean gold-margin gain is evidence about this measured stack under the rationale-conditioned protocol. The structural result is that extraction and audit have the strongest damage rates, and both occur after external evidence has entered the workflow. This resembles failures re-ported for intrinsic self-correction, where model-generated verification can destabilize answers with-out outside feedback (Huang et al., 2024). ASV gives a trace-level way to locate that destabilization. The entropy result gives a second lesson. When a black-box scorer is near one-hot, uncertainty reduc-tion can be blind to large belief pivots, so Bayesian surprise and gold-margin gain are needed together. The label-free rationale buffer should be treated as a measurement channel. It makes evaluator deliberation explicit while preserving one-token likelihood scoring, but it can also compress away the cue that made a state useful. The compo-nent audit shows how to use ASV in practice, that agreement across direct, longer-rationale, and extractive-buffer channels is a stronger signal; dis-agreement redirects inspection to state projection, rationale compression, option rendering, or scorer calibration. The scope remains limited to one 100-question open-QA run, one partly live actor stack, and one DeepSeek evaluator family. ASV logs raw scores, leakage and floor events, permutations, protocol comparisons, state-slice checks, and cue-retention labels, but those audits do not make the judge objective or reveal internal token-level mech-anisms.

A Reproducibility details 

The implementation stores trajectories as JSONL records containing task metadata, candidate speci-fications, and ordered state transitions. Each transi-tion contains an action name, a before-state projec-tion, an after-state projection, and optional cost and trace metadata. Evaluator outputs are stored sepa-rately as candidate log scores, normalized beliefs, rationale quality flags, permutation identifiers, and metric records. Gold candidate identifiers, success labels, final scores, and physical option labels are never rendered into the label-free state projection. This separation makes it possible to rerun evalua-tor protocols from preserved trajectories without rerunning the actor. The 100-question open-QA experiment used four candidates per task, including exactly one 

none-of-the-above . ASV restricts scoring to a small option set because first-token top-log-probability APIs expose only a bounded set of candidate tokens. The source trajectories were col-lected with PubMed retrieval through NCBI Entrez. The actor used DeepSeek where configured: classi-fication used LLM mode for 97 of 100 trajectories, planning for 67, synthesis for 52, verification for 95, and revision for 50; the remaining substeps used deterministic or fallback paths recorded in the trace metadata. No model training was performed. The evaluator used deepseek-chat , a 128-token label-free rationale pass, temperature zero, one generated token for the scored pass, log probabilities enabled, and top-log-probabilities requested for the option labels. The main reported estimate uses a fixed option layout. The B = 4 robustness audit renders the same trajectories under four deterministic phys-ical label mappings and normalizes scores back to stable candidate identifiers before computing mar-gin gain. The protocol-sensitivity audit reuses the same frozen states with the rationale pass disabled and the same one-token option scoring rule. The attribution audit samples the 100 clean largest-gap transitions and replays them under alternate buffer policies and state slices, without rerunning the ac-tor. 

B Evaluator prompt skeleton 

ASV uses two evaluator prompts. The first prompt generates a compact rationale buffer without phys-ical labels. The implementation renders the task question, the redacted state projection, and a candi-date manifest using stable candidate identifiers and candidate texts. 

> You are evaluating an agent state. Generate a com-pact label-free rationale buffer. Use stable can-didate identifiers only. Never mention physical option labels such as A/B/C/D, gold labels, an-swer keys, success flags, or scores. Compare the evidence against each candidate identifier and sep-arate supporting evidence, contradicting evidence, missing evidence, and unresolved ambiguity. Do not output a final option label.

The second prompt appends the physical label mapping only at the tail and scores one token. 

> [Redacted state projection.] [Candidate manifest with candidate identifiers and texts.] [Label-free rationale buffer.] Current physical option map-ping: A = candidate_id: [id]. B = candidate_id: [id]. C = candidate_id: [id]. D = candidate_id: [id]. Output exactly one uppercase option label.

The scored call is read from the log probabilities of the first generated option token. Label tokens are normalized by stripping whitespace and simple punctuation, so variants such as A, leading-space 

A, newline-prefixed A, and A. map to the same op-tion; when multiple returned token variants map to the same option, their scores are combined with log-sum-exp. The scored prompt is not allowed to emit free-form reasoning before the option label be-cause that would make first-token log probabilities measure reasoning preambles. 8
