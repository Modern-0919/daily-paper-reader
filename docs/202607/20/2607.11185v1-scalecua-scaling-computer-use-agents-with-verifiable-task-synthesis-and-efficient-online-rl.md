## 摘要
计算机使用智能体（CUA）正成为通过视觉感知和GUI执行来自动化复杂数字工作流的强大接口。带有可验证奖励的在线强化学习（RLVR）已成为扩展其能力的关键方向。然而，这一范式受到可验证数据稀缺和在线强化学习效率低下的瓶颈。为突破这些障碍，我们引入了ScaleCUA，一个通过可验证任务合成和高效训练来扩展CUA在线强化学习的统一框架。在数据层面，我们设计了VeriGen，一个通过迭代Docker交互和多智能体反馈循环生成可验证强化学习任务的端到端框架。通过共享的Docker交互探针扩展到100多个并发智能体工作进程，该流程生成了超过24K个可验证任务和近3000个高质量强化学习任务。为了最大化样本效率，我们提出了Frontier Sampling，它跟踪每个任务的能力，并将轨迹分配到当前学习前沿。在训练方面，我们进一步设计了Visual Context Segmentation，一个在最近视觉上下文上的滑动窗口，平衡轨迹和训练引擎压力，相比逐步分解实现了2.83倍的训练加速。综合来看，ScaleCUA在OSWorld上达到68.7%，在ScienceBoard上达到54.0%，在开源计算机使用智能体中树立了新的最先进性能。代码、模型和数据集可在https://github.com/THUDM/SCALE-CUA获取。

## Abstract
Computer use agents (CUAs) are emerging as a powerful interface for automating complex digital workflows through visual perception and GUI execution. Online reinforcement learning with verifiable rewards (RLVR) has emerged as a key direction for scaling their capabilities. However, this paradigm is bottlenecked by verifiable data scarcity and online RL inefficiency. To break these barriers, we introduce ScaleCUA, a unified framework that scales online RL for CUAs via verifiable task synthesis and efficient training. At the data level, we design VeriGen, an end-to-end framework for generating verifiable RL tasks through iterative docker interactions and a multi-agent feedback loop. Scaled to 100+ concurrent agent workers via a shared docker interaction probe, this pipeline produces 24K+ verifiable tasks and nearly 3K high-quality RL tasks. To maximize sample efficiency, we propose Frontier Sampling, which tracks per-task capability and allocates rollouts to the current learning frontier. On the training side, we further design Visual Context Segmentation, a sliding window over recent visual context that balances rollout and training-engine pressure, yielding a 2.83x training speedup over step-wise decomposition. Together, ScaleCUA achieves 68.7% on OSWorld and 54.0% on ScienceBoard, establishing new state-of-the-art performance among open-source computer use agents. Code, models, and datasets are available at https://github.com/THUDM/SCALE-CUA.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
> 将可验证任务自动合成、自适应任务采样与高效的滑动窗口多轮训练三者结合，构建从数据到训练的闭环。

### 关键技术细节

#### (1) VeriGen：可验证任务生成引擎
- **输入**：环境知识文档 + 模型 rollout 轨迹。
- **流程**：多智能体循环（提议器、评判器、检查器）与实时 Docker 容器交互。
  - **提议器**：生成任务指令及可执行 Python 评判函数。
  - **评判器**：静态语义审查指令和评判逻辑。
  - **检查器**：动态执行“干跑”，验证任务可达性及评判正确性。
- **轨迹引导合成**：从已训练模型的 rollout 中，对失败任务切分为更简单的子任务；对成功任务组合为更难的复合任务。
- **并行扩展**：通过共享 Docker 交互探针（env probe API），支持 100+ 并发数据工人和 100+ Docker 环境，生成 **24K+ 可验证任务**，其中近 **3K 高质量 RL 任务**。

#### (2) Frontier Sampling：前沿任务采样策略
- 维护每个任务的 EMA（指数移动平均）成功率 \( \hat{p}_i \)。
- 过滤掉成功率超出 [0.1, 0.9] 的任务后，计算采样权重：
  \[
  w_i = \exp\left(-\frac{(\hat{p}_i - \mu)^2}{2\sigma^2}\right)
  \]
  其中 \(\mu=0.5\)（目标成功率），\(\sigma=0.25\)。
- 预留 20% 的批次用于从剩余任务中均匀采样（保持探索）。
- **效果**：集中 rollouts 于学习前沿，加速收敛。

#### (3) Visual Context Segmentation：视觉上下文分割
- **问题**：完整保留多轮截图会导致 rollout 引擎负担重；按步分解则训练条目数量膨胀。
- **方法**：在 rollout 过程中维护一个 token-ID 流（含损失掩码和 log 概率），使用滑动窗口 \(K\)（保留最近截图数）和最小移除阈值 \(\Delta\)。
- 当截图数超过 \(K+\Delta\) 时，保存当前段为训练样本，移除最旧的 \(\Delta\) 张截图，重新分词剩余历史。
- **效果**：将 O(nT) 训练项压缩为 O(nT/K)，在 K=5 时获得 **2.83 倍端到端训练加速**，且不损失任务成功率。

## 7. 优点：方法或实验设计上的亮点

- **数据-训练闭环设计**：VeriGen 利用模型自身的 rollout 轨迹反哺生成新任务，形成自改进循环。
- **实用且高效的工程实现**：共享 Docker 探针、并行数据工人、token-ID 流等设计支撑大规模实际部署。
- **采样策略轻量有效**：仅需 EMA 统计和简单高斯加权，无额外开销，但大幅提升训练信号纯度。
- **训练加速不牺牲精度**：Visual Context Segmentation 既解决 token 膨胀又保持多步推理能力，具有实际落地价值。
- **验证数据质量和安全性**：通过专家审计、重叠审查、错误分析等多维度评估，增强了结果可信度。
