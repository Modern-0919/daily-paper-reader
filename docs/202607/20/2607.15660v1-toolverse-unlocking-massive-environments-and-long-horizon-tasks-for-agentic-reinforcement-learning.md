## 摘要
虽然大语言模型（LLM）智能体在紧凑且定义明确的场景中展现出强大的推理能力，但在面对需要无缝工具集成的大规模、多样化和动态的真实世界环境时，它们在保持鲁棒性和有效性方面仍存在困难。为解决这一差距，我们引入了ToolVerse，一个全面的框架，该框架扩展了智能体强化学习环境，使智能体能够在工具集成推理（TIR）任务中执行复杂的长期推理。首先，ToolVerse从近400个包含约4500个工具的真实世界模型上下文协议（MCP）中自动构建大规模可执行的智能体训练环境。其次，我们提出了一种基于工具依赖图的任务设计策略，利用动态解锁采样算法生成长期任务，并生成GUST（图解锁采样任务）数据集。第三，为缓解长期智能体强化学习中的信用分配问题，我们提出了一种细粒度的回合感知相对优势算法。我们使用ToolVerse进行了广泛的智能体强化学习训练，并在多个智能体基准上评估了我们的框架。实验结果表明，我们的框架显著增强了LLM在长期工具使用方面的能力，实现了显著的性能提升，并在动态环境中展示了稳健的推理能力。

## Abstract
While LLM agents demonstrate strong reasoning abilities in compact and well-defined scenarios, they struggle to maintain robustness and effectiveness when faced with large-scale, diverse, and dynamic real-world environments that demand seamless tool integration. To address this gap, we introduce ToolVerse, a comprehensive framework that scales up agentic RL environments and enables agents to perform complex long-horizon reasoning in Tool-Integrated Reasoning (TIR) tasks. First, ToolVerse automatically builds the massive executable agent training environments from nearly 400 real-world Model Context Protocols (MCPs) that contain about 4500 tools. Second, we propose a task design strategy based on a tool dependency graph, utilizing Dynamic Unlocking Sampling Algorithm to generate long-horizon tasks, and produce GUST (Graph Unlocking Sampling Tasks) dataset. Third, to alleviate the credit assigment problem in long-horizon agentic RL, we propose a fine-grained Turn-Aware Relative Advantage algorithm. We conduct extensive Agentic RL training using ToolVerse and evaluate our framework on serveral agentic benchmarks. Experimental results demonstrate that our framework significantly strengthens LLMs' capabilities in long-horizon tool use, achieving a marked performance boost and showcasing robust reasoning within dynamic environments.

## 2. 论文提出的方法论
### 核心思想
- 通过自动化管道将真实世界 MCP（Model Context Protocol）工具定义转化为可执行 Python 环境（含字典数据库）；基于工具依赖图（Tool Dependency Graph, TDG）和动态解锁采样（Dynamic Unlocking Sampling, DUS）算法生成长效、因果一致的多步任务；并设计回合感知相对优势（Turn-Aware Relative Advantage, TARA）算法，用于细粒度信用分配。

### 关键技术细节
#### 2.1 规模化可执行智能体环境
- **输入**：原始工具集 Sraw = {S₁,...,Sₙ}（每个 Si 对应一个功能域）。
- **流程**：
  1. **Schema Refactoring**：将原始 schema 转换为标准结构 ˆSi，去除噪声。
  2. **字典数据库构造**：针对每个工具集构建伪数据库 Di（如用户、订单等状态变量）。
  3. **代码生成**：利用 MCP 工具库生成可执行函数 Fi，实现查询/更新操作。
  4. **单元测试**：自动生成测试 Ti，仅保留通过语法和功能验证的工具集（V(Fi,Ti)=True）。
- **结果**：从约 400 个真实 MCP 构建出 4438 个可执行工具的 422 个环境。

#### 2.2 图引导的长周期任务合成 —— GUST 数据集
- **Tool Dependency Graph (TDG)**：节点 = 工具，边 = 输出-输入依赖或顺序依赖。图由 LLM 推断并经过环去除、DAG 净化。
- **动态解锁采样算法 (DUS)**：
  - 维护入度为 0 的就绪队列 Q。
  - 每步采样子集 St ⊆ Q（批量大小 k = min(|Q|, N)），移除已处理节点，更新后继节点入度。当某节点入度降为 0 时加入 Q。
  - 生成的轨迹 T = [S₁, S₂, ..., Sₘ] 自然呈现递增复杂度。
- **逆上下文重构**：将工具骨架填充具体参数（依赖参数从前序输出获取，上下文参数从数据库获取），形成可执行黄金轨迹（Golden Trace）。然后用 LLM 将黄金轨迹转化为用户任务（保留依赖边信息），并通过 LangGraph 执行校验和教师模型 Pass@8 过滤。
- **统计**：GUST 数据集包含 2987 条数据项，每个数据项平均 3.8 个子任务，每个环境平均 12.2 个工具。

#### 2.3 回合感知相对优势估计 (TARA)
- **动机**：标准 GRPO 给整个轨迹赋单一优势，无法区分中间步骤的正确/错误。
- **二元奖励**：每个回合 ri,t ∈ {0,1}，检查工具调用 JSON 是否覆盖黄金轨迹集合（字典层匹配）。
- **局部优势 Alocal**：组内标准化当回合奖励。**门控未来优势 Afuture**：引入一致性门 δi,t = ri,t，仅在当前步正确时才累加未来折扣奖励；同样进行组内标准化。
- **总优势**：Atotal = Alocal + λ·Afuture（默认 λ=0.5）。该信号分配给该回合所有 token。
- **理论证明**：方差比完整未来收益估计小（λ² 倍），且可抑制干扰动作（门控阻断未来幸运回报）。

## 6. 主要结论与发现
- ToolVerse 训练显著提升所有规模模型在三个工具使用基准上的性能，最高提升达 +15.15（Qwen3-8B on ACEBench-Agent）。

- 组件消融验证：局部与未来优势结合+门控机制达到最佳效果。
- 超参数 λ=0.5、γ=0.5 为最优，中等程度的未来信用传播平衡效果与稳定性。
- 环境数目从 100 扩展到 422 持续提升泛化性能，说明多样性是关键。

## 7. 优点
1. **环境规模化自动化**：通过 MCP 自动重建真实工具集，将 Agentic RL 从单一工具扩展到通用域。
2. **因果一致的长周期任务生成**：基于 TDG + DUS 算法，自然模拟依赖链，生成逻辑连贯的多步任务，避免随机游走。
3. **细粒度信用分配**：TARA 通过回合级二元验证、局部/未来优势分解、门控机制，有效降低方差并抑制干扰动作，理论分析支持其有效性。
4. **通用性**：在多种模型（思考/非思考、不同尺寸）上均取得一致增益；训练环境覆盖面广（8 大宏域、众多实体）。
5. **工程细节完善**：包括 TDG 净化、逆上下文重构、Pass@8 过滤、零方差处理等，保证数据质量。

## 8. 不足与局限
