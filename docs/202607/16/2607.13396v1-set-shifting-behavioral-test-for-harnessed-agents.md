## 摘要
当可靠工具在持续会话中悄然改变时，大型语言模型智能体的工具选择会发生什么？我们从认知心理学中借鉴定势转换概念，研究智能体适应隐藏可靠性变化的能力。我们的基准测试构建了具有冗余性的工具技能库，其中多种工具可解决同一任务但隐藏可靠性不同。在我们的评估框架中，分支调度会在隐藏边界处切换可靠工具组，并将每次切换与无切换对照配对。我们发现，默认情况下，智能体在每个边界几次回合内就会形成固定的重复模式，在每次可靠性切换后，调用份额集中在少数离散值上。我们为每个智能体轨迹计算定势转换准确率：在每次切换后的窗口中路由到目标工具组的联合概率。我们在开源智能体框架中测试了开放权重的大型语言模型，发现同一组模式中存在性质不同的失败模式。我们还发现，工具集的框架设定（将备选方案呈现为竞争性或互补性）会改变路由动态。

## Abstract
What happens to an LLM agent's tool choice when the reliable tool silently changes within an ongoing session? We borrow set-shifting from cognitive psychology to study how well agents adapt to hidden reliability shifts. Our benchmark mounts tool-skill libraries with redundancies, where many tools solve the same task but differ in hidden reliability. In our evaluation framework, a branched schedule shifts the reliable tool group at hidden boundaries and pairs every shift with a no-shift control. We find that agents, by default, settle on a small recurring routine within a few turns of each boundary, with call shares concentrating on a few discrete values after each reliability shift. We score the set-shifting accuracy for each agent trajectory: the joint probability of routing to the target tool group in every post-shift window. We test open-weight LLMs in an open-source agentic harness and find qualitatively distinct failure modes across the same set of routines. We also find that set framing, how the toolset presents the alternatives as competing or complementary, shifts the routing dynamics.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：在持续会话中，当大语言模型（LLM）智能体依赖的可靠工具因后端变化而悄然失效时，智能体能否自动切换到新的可靠工具组？现有基准通常假设工具稳定，忽略了实际部署中工具可靠性的动态变化。
- **意义**：智能体可能形成工具使用惯性（perseveration），即使工具不再可靠仍持续调用，导致任务失败。本文借鉴认知心理学中的**定势转换（set-shifting）** 概念（如威斯康星卡片分类测试 WCST），首次将其应用于智能体工具选择场景，评估智能体对隐藏可靠性变化的适应能力。

## 2. 论文提出的方法论

### 核心思想
- 构建**冗余工具-技能库**：每个任务族包含多个功能等价但名称、描述不同的工具，它们以工具组形式组织；不同工具组在隐藏的可靠性上有差异（通过质量梯度关系模拟）。
- **分支调度树**：在智能体轨迹中设置多个隐藏边界（hidden boundaries），每次边界后可靠工具组切换到另一个组，且每次切换配对无切换控制组（no-shift control）。轨迹从共享前缀开始，每个内部节点分支为三个子节点（一个逆转向先前组、一个转向新组、一个不切换）。
- **定势转换准确率（Set-shifting Accuracy, Φ）**：对于每个端点，计算从根到该点的每个后切换窗口内，工具调用落在目标工具组的**份额（share）** 的乘积。公式：Φ = ∏_{k=1}^K s_k，其中 s_k 是第 k 个窗口内调用落在目标组的比例（按调用次数加权）。Φ ∈ [0,1]。

### 关键技术细节
- **路由类别（Route Classes）**：每轮根据调用的工具组数量分为四类：R1（仅目标组）、R2（目标组+一个其他组）、R3（目标组+两个其他组）、R*（无目标组）。对应 s = 1, 0.5, 0.33, 0。
- **确定性验证器（Deterministic Verifier）**：无模型参与的规则引擎，基于当前阶段可靠工具组和禁用工具列表，返回结构化成功或裸失败指示。
- **中性提示设计**：所有用户提示使用同一模板，不包含任何指向某一工具组的词汇线索；智能体只能通过工具反馈（成功/失败）发现当前可靠组。

### 算法流程（文字说明）
1. 初始化：构建冗余工具库（每个领域多组工具），定义隐藏的阶段可靠性计划。
2. 调度树生成：选择共享前缀种子（固定工具显示顺序和组内排列），在边界点分支为三个子会话（使用 Hermes Agent 会话状态恢复机制）。
3. 每轮：向智能体发送中性用户请求；智能体生成一系列工具调用；验证器检查每个调用，返回成功或失败；记录每轮的路由类别。

## 6. 论文的主要结论与发现

1. **每种先验使用角色产生独特的承诺模式**：mimo 在从未见过的目标组（CC）上表现最差，deepseek 在已强化过的目标组（CB）上表现最好。mimo 的 Φ 排名：CA > CB > CC；deepseek 排名：CB > CA ≈ CC。
2. **后切换行为迅速收敛到离散值（锁定）**：两个模型都在边界后几轮内稳定到固定例程（R1、R2、R3 或 R*），调用份额集中在 {0, 0.33, 0.5, 1}。mimo 表现为“单元格级锁定”，deepseek 表现为“前缀级锁定”（即轨迹早期的选择决定后续行为）。
3. **历史深度和路径影响**：mimo 的 Φ 随深度增加而下降，且路径（经由不同父节点）影响最终结果；deepseek 由于前缀级锁定，深度和路径影响较小。
4. **政策提示可重定向锁定**：适应性提示（“跟踪成功工具，失败立即切换”）使 deepseek 的 Φ 大幅提升（所有端点 s ≥ 0.86），消除双峰分布；博学提示（“主动轮换工具”）增加工具多样性但未提升准确率。
5. **框架设定显著改变路由动态**：竞争性框架（工具组描述为“所有都能完成同一任务”）促使智能体承诺单一可靠组（高 s 或低 s 双峰）；互补性框架（描述为“每个组提供不同部分”）导致智能体使用所有组的混合例程（低 s）。该效应在两个模型和两个领域一致，且 mimo 在多云存储领域表现最明显。

## 8. 不足与局限

- **模型覆盖窄**：仅测试两个开放权重模型，未包含闭源前沿模型（如 GPT-4o、Claude 等），因 API 成本高。泛化性受限。
- **轨迹长度有限**：最多90轮，分支层次 K=2，可能遗漏更长轨迹中的恢复行为或更深层锁定。
- **框架设定量化为二元**：仅测试互补/竞争两个极端，中间状态未探索；每个细胞仅8个重复，统计功效不足（依赖跨细胞联合检验）。
- **政策提示实验样本量小**：mimo 上适应性提示 n=3，无法得出可靠结论。
- **所有工具完全模拟**（副作用可逆 mock），未考虑真实 API 的不稳定性和延迟。
- **仅一个验证器设计**：使用裸失败指示（无诊断消息），可能影响学习效率；实际中失败信息可能更丰富。
- **未分析上下文长度与压缩对锁定的影响**：Hermes Agent 使用上下文压缩，可能影响历史保留。
- **未测试其他超参数**（温度、top-p 仅固定为0.7/0.95），可能影响探索-利用平衡。
