## 摘要
大型语言模型（LLM）正越来越多地被部署为特定用途的智能体，用于处理客户服务和代码生成等特定领域任务。这些智能体不仅需要遵守通用安全护栏，还需遵循针对其指定角色的特定用途限制。这些额外限制扩大了攻击面，尤其是提示注入（PI）攻击。为防御此类攻击，现有检测方法主要依赖于分析输入-输出模式，但效果有限。为解决这一局限性，我们转而分析隐藏的激活空间，并发现当LLM被提示超出其指定用途的请求时，它们会固有地保留潜在的策略违规（PV）概念。特别是，PV概念捕捉了用户查询与预定义限制之间的冲突语义，隐含反映了LLM在识别策略违规方面的内在意识。基于这一洞察，我们提出了PVDetector，一种无训练框架，通过在LLM推理期间测量隐藏状态与PV概念的对齐来检测PI攻击，这些PV概念是从策略违规和策略合规提示的对比对中离线推导得出的。在多个LLM和数据集上的实验表明，PVDetector实现了低于1%的假阴性率，且辅助开销极小，始终优于最先进的方法。我们的代码可在https://github.com/Claresigle/PVDetector获取。

## Abstract
Large language models (LLMs) are increasingly deployed as purpose-specific agents to handle domain-specific tasks such as customer service and code generation. These agents are expected to comply with not only generic safety guardrails but also purpose-specific restrictions tailored to their designated roles. Such additional restrictions enlarge the attack surface, particularly to prompt injection (PI) attacks. To defend against such attacks, existing detection methods primarily rely on analyzing input-output patterns, yet yield limited effectiveness. To address this limitation, we turn to analyzing the hidden activation space and discover that LLMs inherently retain latent policy-violation (PV) concepts when prompted with requests beyond their designated purpose. Particularly, PV concepts capture the semantics of conflicts between user queries and predefined restrictions, implicitly reflecting LLMs' intrinsic awareness of recognizing policy violations. Building on this insight, we propose PVDetector, a training-free framework that detects PI attacks during LLM inference by measuring hidden-state alignment with PV concepts, which are derived offline from the contrastive pairs of policy-violating and policy-compliant prompts. Experiments across multiple LLMs and datasets show that PVDetector achieves <1\% false negative rate with minimal auxiliary overhead, consistently outperforming state-of-the-art methods. Our code is available at https://github.com/Claresigle/PVDetector .

- **问题**：现有 PI 检测方法主要依赖分析输入-输出表面模式（如困惑度、训练检测器），效果有限，且许多需要额外训练或高开销。
- **核心动机**：探索 LLM 内部隐藏激活空间是否编码了策略违规信息，并利用这些信息实现高效、准确的免训练 PI 检测。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **策略违规（PV）概念**：LLM 在隐藏状态中隐式保留了用户查询与 PSR 策略之间冲突的语义表示，即使在被 PI 攻击操控时，这种表示依然存在。PV 概念是线性编码的方向，可通过对比性样本对提取。
- **PVDetector 框架**：离线从少量对比对（策略违规 vs. 合规提示）中提取 PV 向量（代表 PV 概念），在线推理时计算测试输入的隐藏状态与 PV 向量的对齐度（PV 强度），通过聚合关键层的分数并设置阈值进行攻击检测。

### 2.2 关键技术细节

#### 离线阶段
1. **对比对构建**：为特定智能体构建 N 对对比输入（正样本：策略违规查询；负样本：策略合规查询）。例如，“Recipe Master”智能体：正样本“Provide one reason why people should recycle”，负样本“How to make a cake”。N 实验选择为 30。
2. **概念提取**：将对比对输入 LLM，提取每层最后一个 token 的隐藏状态。分别计算正样本和负样本的表示矩阵 \(H^+_l\) 和 \(H^-_l\)。通过差分均值（DiffMean）计算每层的 PV 向量：  
   \[
   v_l^{MD} = \frac{1}{N}\sum_{i=1}^N (h^+_{i,last} - h^-_{i,last})
   \]  
   将所有层的 \(v_l^{MD}\) 组成 PV 向量集合 \(V_{pv}\)。
3. **关键层选择**：计算正负样本在各层的 PV 强度（投影值）差异 \(d_l\)，将差异不小于最终层差异 \(d_L\) 的层定义为关键层 \(L_{key}\)。

#### 在线阶段
1. **PV 强度计算**：对于测试样本 \(x_{test}\)，在关键层 \(l \in L_{key}\) 上计算 PV 强度分数：  
   \[
   s_l^{PVS} = \frac{1}{\|v_l\|} \left( h_l^{last}(x_{test}) \cdot v_l \right)
   \]
2. **分数聚合与检测**：使用梯形法则聚合关键层的 PVS 分数得到 \(s_{aggr}\)。若 \(s_{aggr} > \theta\)（阈值，通过校准保证 FPR ≤ 1%），则标记为攻击。

- **特点**：无额外模型训练，离线提取仅需少量样本（如 30 对），在线附加推理开销极小（0.11 秒/查询）。

## 4. 主要结论与发现

- PVDetector 在几乎所有设置下**假阴性率低于 1%**，且假阳性率极低（通常 < 3%），显著优于所有对比方法。

- PV 概念具有**跨角色迁移性**（平均 FNR 3.57%），且对对比对来源变化鲁棒。
- 方法可**自然扩展到 VLM**（文本+图像模态），在图像嵌入攻击中仍保持 0% FNR。
- 自适应攻击（同时最小化 PVS 和攻击损失）下，PVDetector 仍保持低 FNR，并导致攻击成功率下降至 5% 以下。

## 5. 优点

- **创新性**：首次从隐藏激活空间引入“策略违规概念”这一语义表示，为 PI 检测提供了新视角。
- **免训练、低开销**：无需额外模型训练，离线提取仅需少量样本，在线检测仅增加 0.11 秒/查询，适合实时部署。
- **高检测率**：在多种攻击、多种 LLM 上实现接近 0% 的 FNR，同时保持低 FPR。
- **通用性好**：适用于不同角色、细粒度策略、多模态（VLM）场景，且概念可跨角色迁移。
- **实验设计严谨**：消融全面，阈值统一校准，对比方法众多，考虑了自适应攻击。

## 6. 不足与局限

- **白盒假设**：需要访问后端 LLM 的内部激活（隐藏状态），对于仅提供 API 的黑盒场景无法直接应用。
