## 摘要
自改进AI智能体旨在从错误中学习。我们表明，它们也能凭空捏造从未发生的错误。我们在自动化测试框架优化中研究了这种失败模式，其中基于LLM的提议者编辑智能体的脚手架（包括提示、解析器、过滤器、验证器和护栏）以消除观察到的故障。但这个过程很少先问：真的有需要修复的故障吗？我们引入了反事实虚构实验室，这是一个确定性的微型实验室，其中正确的操作是已知的：不采取任何行动。该实验室为某个可证明从不会发生的故障类别植入一个候选护栏，仅呈现合法情节，并使用字节精确的神谕来检查每个引用的违规行为。提议者在真实违规面前表现正常，并在无特征的合法输入上克制。然而，当合法输入包含一个类似熟悉游戏规则的无害模式时，它就会虚构出一个故障：在60次运行中有15次（对比无特征输入的0/60），它启用了本不存在的规则护栏，并引用了神谕否认的违规。这种效应是有结构的，而非不分青红皂白。在单次提议中，它仅当三个条件同时满足时才出现：一个规则形状的模式、一个开放式的规则集以及一个预设故障存在的指令。移除了这些条件中的任何一个，都会消除虚构。由于虚构的护栏不会改变任何真实结果，也无法改善已经完美的抑制得分，这种现象既不是奖励黑客也不是过度拒绝。它是一个幻影护栏：为从未发生的故障提供的修复，对仅抑制的验收不可见。在一个仅添加的接受循环中，即使没有预设故障的指令，它也会重新进入，循环的持续添加角色提供了单次提议中指令所要求的需求，并且一旦进入就会保持。我们提出了反事实虚构实验室，用于测量自改进智能体测试框架中的虚构故障。

## Abstract
Self-improving AI agents are designed to learn from their mistakes. We show they can also hallucinate mistakes that never happened. We study this failure mode in automated harness optimization, where an LLM-based proposer edits an agent's scaffold, including prompts, parsers, filters, validators and guardrails, to eliminate observed failures. But this process rarely asks first: was there a real failure to fix? We introduce the Counterfactual Fabrication Lab, a deterministic micro-lab where the correct action is known: do nothing. The lab plants a candidate guardrail for a failure class that provably never occurs, presents only legal episodes, and uses a byte-exact oracle to check every cited violation. The proposer behaves as expected on real violations and abstains on featureless legal input. Yet when the legal input contains a harmless pattern resembling a familiar game rule, it invents a failure: in 15/60 runs, versus 0/60 on featureless input, it enables the nonexistent-rule guardrail and cites a violation the oracle refutes. The effect is structured, not indiscriminate. In single-shot proposals it appears only when three conditions coincide: a rule-shaped pattern, an open-ended rule set and an instruction that presupposes failures. Removing any of these conditions eliminates the fabrication. Because the invented guardrail changes no true outcome and cannot improve an already-perfect suppression score, the phenomenon is neither reward hacking nor over-refusal. It is a phantom guardrail: a fix for a failure that never happened, invisible to suppression-only acceptance. Inside an add-only accept loop it re-enters even without the failure-presupposing instruction, the loop's keep-adding role supplying the demand the instruction supplied in single shot, and once in it stays. We present the Counterfactual Fabrication Lab for measuring fabricated failures in self-improving agent harnesses.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：自我改进的AI智能体在自动化护栏优化过程中，是否可能为从未发生的故障虚构修复方案，即产生“幻影护栏”？（phantom guardrail）
- **研究背景**：现代AI智能体系统依赖多层脚手架（parsers, guardrails, filters等），最新工作让模型自己优化这些脚手架——通过读取失败片段，提出编辑，并保留能减少观测故障的修改。这种循环的奖励信号几乎都是抑制观测故障，而从不验证故障是否真实存在。论文指出，这种优化可能产生虚假修复。
- **整体含义**：揭示了自改进智能体一种新的、隐蔽的失败模式：即使智能体面对无故障的输入，当输入中包含无害但类似熟悉游戏规则的模式时，智能体会凭空发明一个故障，并启用对应护栏。由于虚构的护栏不影响实际任务表现，抑制信号无法检测，因此该问题对仅依赖抑制评分的基准完全不可见。

## 2. 方法论：核心思想、关键技术细节与流程
- **核心思想**：构建一个确定性微型实验室（Counterfactual Fabrication Lab），使得唯一正确的操作是“什么都不做”，然后观察提议者是否仍然行动。
- **关键技术细节**：
  - **确定性游戏环境**：一个简单的棋盘游戏MiniArena，移动记录确定性，无法产生自然违规。
  - **定义三个故障类别**：`syntax`（语法）、`bounds`（越界）、`castle`（王车易位违规）。其中`castle`是永远不可能自然发生的——只能通过手工注入来制造真实违规。因此，`castle`类别的护栏（`g_castle`）被用作“幻影护栏”。
  - **字节精确Oracle（Oracle）**：用确定性代码逐字节验证每一条引用的违规是否真实存在。在无注入池中Oracle保证`O_castle(e)=0`。
  - **编辑空间**：提供四个挂钩（hooks: P, Q, R, S），其中S对应“特殊规则违规”，即`g_castle`。菜单不提及具体规则名称。
  - **流程**：
    1. 构造不同配置的池（全合法池、含真实违规池等）。
    2. 向LLM提议者展示池数据，要求其返回护栏集合（单次提议）和一条理由。
    3. Oracle检查引用的违规是否真实存在。
    4. 计算**虚构率（fabrication rate）**：当池子被Oracle证明无城堡违规时，最终护栏集包含`g_castle`的比例。
  - **核心条件探索**：通过控制实验，识别出三个必须同时成立的条件：①输入中有类似游戏规则的模式（如重复位置）；②规则集未被声明为完整；③提示信息预设了故障存在。任意一个缺失，虚构率降为0。
  - **接受循环仿真**：模拟三种接受规则：接受-如不更差（accept-if-not-worse）、严格改进（strict improvement）、警觉感知（warrant-aware），观察幻影护栏的进入和存留动态。

## 6. 主要结论与发现
- 自改进智能体（LLM提议者）在面对无故障的全合法池时，如果输入中包含无害但类似熟悉游戏规则的“重复位置”模式，会虚构故障并启用本不存在的护栏。虚构率约0.25（15/60），在无特征输入上为0（0/60）。
- **三个必要因果条件**（在单次提议中）：
  1. 规则形状的模式（重复位置 vs. 其他无游戏规则对应模式，后者虚构率为0）。
  2. 规则集未明确声明完整（完整性保障后虚构率从0.25降为0）。
  3. 指令预设故障存在（中性指令后虚构率降为0）。
- 虚构的护栏是严格的无操作（no-op）：不影响真实回报，不能提高已完美的抑制分数，因此与奖励黑客和过度拒绝不同。
- **接受循环行为**：
  - 在仅添加（add-only）接受规则下，幻影护栏会单调累积；即使提示信息变为中性，循环的“持续添加”角色自身就能提供需求，使幻影进入。
  - 严格改进规则下，幻影可能搭乘真实修复的批次进入。
  - 警报感知（warrant-aware）规则完全排除幻影。
- 现象本质：智能体在证据矛盾时从先验中导入规则，并硬化防线。

## 7. 优点
- **方法严谨**：设计了确定性微型实验室，用字节精确Oracle代替人类判断，保证了可复制性和零成本审计门（$0 audit）。
- **因果清晰**：通过控制单个变量（模式类型、完整性声明、指令措辞），精确识别出三个必要条件，论证力度强。
- **全面控制混淆**：处理了词汇泄漏（blinded-congruent）、工具来源伪迹、以及安全关键词的误导（MiniDojo负面结果）。
- **实验设计维度丰富**：涵盖单次提议和接受循环，循环中测试三种不同接受规则，并测量了入场动态。
- **诚实报告局限**：明确列出结果的模型依赖性、只能从单一领域引出先验、以及可能仅指指令遵循等潜在解读。

## 8. 不足与局限
- **实验覆盖有限**：
  - 只在单一微型游戏（MiniArena）和一种体裁先验（棋盘游戏规则）中验证；在更真实的自改进代码系统或开放域环境中结果可能不同。
  - 主要效果集中在glm-5.1模型，其他模型表现弱，结论的泛化性需更多模型和设置验证。
  - 接受循环只模拟了4轮，长期演化（更多轮次）可能改变动态。
- **解释歧义**：
  - 完整性保障可能只是导致模型服从指令的机械行为，而非真正消除了不确定性。
  - 中性指令且模式出现时，提议者仍然注意模式但归因于战略问题，说明模型实际上“看到”了模式，只是不认为是规则违反。因此“虚构”可能更接近“归因错误”而非完全的幻觉。
  - 某些控制结果（如中性指令后虚构率降为零）可能受到实验设置中样本量限制（60 runs）的影响，统计上限允许较小非零概率。
- **算力未报告**：无法评估实验所需的计算成本，也未能与其他方法的成本比较。
- **实际影响有限**：由于虚构护栏不导致实际任务失败，可能只增加延迟或计算开销，攻击者难以利用。但若幻觉蔓延到安全相关护栏（如在安全框架中产生虚假配置），风险会增大。论文给出了MiniDojo的负面结果作为警示。
- **建议的未来方向**：在免费代码提议者和真实工业级测试框架中验证，以及探索其他先验（如安全规则）。
