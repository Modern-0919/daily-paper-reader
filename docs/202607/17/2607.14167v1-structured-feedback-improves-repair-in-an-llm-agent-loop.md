## 摘要
LLM智能体通常在外部验证拒绝候选方案后重试，但验证与后续模型调用之间的接口仍不明确。我们引入了VeriHarness，这是一个代码控制的智能体循环，模型生成候选方案，而外部验证器控制接受、预算和追踪。我们使用它来比较原始诊断与识别故障位置、观测值和可接受替代方案的反馈。在四次调用上限下的50个配对TextWorld游戏中，包含所有三个字段的反馈将Qwen2.5-Coder-14B的最终成功率从14/50提升至36/50（+44个百分点），Llama-3.1-8B从8/50提升至29/50（+42个百分点）。消融实验显示大部分增益来自可接受替代方案：仅包含位置和观测值的反馈仍接近原始诊断基线。以散文形式而非键值JSON记录呈现完整的修复信息，成功率几乎相同，没有证据表明JSON语法本身改善了修复。这种排序在测试的调用预算和一次采样解码设置中保持一致。

## Abstract
LLM agents often retry after external validation rejects a candidate, but the interface between validation and the next model call remains underspecified. We introduce VeriHarness, a code-controlled agent loop in which models generate candidates while external validators control acceptance, budgets, and traces. We use it to compare raw diagnostics with feedback that identifies the failure location, observed value, and admissible alternatives. Across 50 paired TextWorld games under a four-call cap, feedback containing all three fields raises terminal success from 14/50 to 36/50 for Qwen2.5-Coder-14B (+44 percentage points) and from 8/50 to 29/50 for Llama-3.1-8B (+42 points). Ablations locate most of the gain in the admissible alternatives: feedback containing only the location and observed value remains near the raw diagnostic baseline. Presenting the complete repair information in prose instead of a keyed JSON record yields nearly the same success, providing no evidence that JSON syntax itself improves repair. The ordering persists across the tested call budgets and one sampled-decoding setting.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：LLM智能体（agent）在生成候选方案后，常由外部验证器（validator）进行检查；若失败则重试。但验证器应返回怎样的反馈信息以帮助模型修复，这一接口设计缺乏系统研究。
- **核心问题**：
  - RQ1：结构化的验证器反馈（包含失败位置、观测值、可替代方案）相比原始诊断信息，是否能在相同计算预算下提升修复成功率？
  - RQ2：任何增益是来自键值表示（JSON）、修复值本身，还是二者共同作用？
- **整体含义**：明确的修复信息（尤其是可替代方案）比仅告知“失败”更有效；JSON语法本身并不比自然语言更优。这为设计 agent 循环中的验证器输出提供了具体指导。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将候选方案的生成与外部验证分离，使用代码（而非纯提示）控制 agent 循环状态、预算和追踪。提出 **VeriHarness** 框架。
- **关键技术细节**：
  - **架构**：一个主协调器构建紧凑状态包，分派LLM worker；每个worker返回结构化候选；外部“门”（gate）进行模式、工件和环境行为验证。只有门能接受候选，模型自身的自评估无效。
  - **反馈策略对比**（以TextWorld为示例）：
    - **RawDiag**：原始验证错误信息，无显式修复字段。
    - **SameNL**：用自然语言（散文）提供失败位置、观测值和可替代方案。
    - **LocObs**：仅包含失败标签、位置和观测值（JSON格式），无替代方案。
    - **TypedFields**：包含失败标签、位置、观测值和可替代方案（JSON键值对）。
  - **流程**：每次失败后，failure encoder 将门特定细节映射为通用接口（如稳定标签、命令索引、可接受动作列表等）。新一次调用时，模型接收到紧凑任务状态和当前修复反馈。
- **与现有方法的区别**：Reflexion、Self-Refine 等依赖LLM自身产生和使用批评；VeriHarness 强调外部确定性验证器应返回结构化信息，并评估信息中不同字段的贡献。

## 6. 论文的主要结论与发现

1. **结构化反馈显著提升修复成功率**：包含位置、观测值和可替代方案的反馈（TypedFields）相比原始诊断（RawDiag），在Qwen2.5-14B上提升44个百分点（95% CI: 28-60, p<0.001），在Llama-8B上提升42个百分点（95% CI: 28-56, p<0.001）。大多数最终差异出现在重试之后。
2. **增益主要来自可替代方案**：仅含位置和观测值的LocObs几乎与原始诊断性能相同；添加可替代方案后成功率提升约36-40个百分点。
3. **JSON语法不优于散文**：SameNL（散文）与TypedFields（JSON键值）的成功率几乎相同（Qwen: 70% vs 72%；Llama: 58% vs 58%），差异无统计显著性。但TypedFields使用略少的调用次数（平均每游戏少0.34次），可能因信息结构更清晰便于模型解析。
4. **预算效应**：在15游戏子集上，RawDiag从4次调用增加到8次调用无新游戏被解决，而TypedFields持续获益（Llama尤其明显，从2/15升到11/15）。
5. **采样解码下顺序保持一致**：Qwen在温度0.3下，TypedFields vs RawDiag 提升45个百分点（95% CI 23-67, p=0.005）。
6. **HumanEval边界条件**：当可见测试始终通过时，无修理机会，所有策略结果相同（14/15）。说明结构化反馈仅当验证器暴露有用失败时才能发挥作用，无法修复验证器未检测到的错误。
