## 摘要
在软件系统中可信地部署基于LLM的智能体需要评估它们在特定应用工作流上的表现，且粒度要足够精细以定位成功与失败的环节。然而现有的智能体评估机制存在局限性：基准测试对特定应用工作流和环境的构念效度较低，而复制评估环境成本高昂且易发生漂移。我们提出写时复制（CoW）评分框架，该框架直接在应用环境中评估智能体操作，利用PostgreSQL级别的写时复制机制隔离智能体的写入操作。CoW评分生成会话级和操作级评分，突出显示智能体在特定应用环境中数据库写入操作的成功与失败位置，从而实现对智能体控制台和工具界面的低成本评估与迭代。我们在开源项目管理平台Plane上演示了该框架，分析揭示了工具界面中的具体问题，相应的修复措施在受影响的模型上带来了可衡量的改进。Python库：https://github.com/trail-ml/agent-cow-python

## Abstract
Trustworthy deployment of LLM-based agents in software systems requires evaluating how they perform on application-specific workflows, with enough granularity to localize where they succeed and fail. Yet existing agent evaluation mechanisms are limited: benchmarks have low construct validity for application-specific workflows and environments, and replica evaluation environments are expensive and prone to drift. We propose Copy-on-Write (CoW) Scoring, a framework that evaluates agent operations directly within application environments using a PostgreSQL-level Copy-on-Write mechanism to isolate agent writes. CoW Scoring produces session- and operation-level scores that highlight where agents' database write operations succeed and fail in a given application environment, enabling inexpensive evaluation and iteration on agent harnesses and tool surfaces. We demonstrate the framework on Plane, an open-source project-management platform, where analysis surfaced specific issues in the tool surface, and corresponding fixes produced measurable improvements on affected models. #### 2. 方法论：核心思想、关键技术细节

- **核心思想**：借鉴操作系统中的写时复制（Copy-on-Write）概念，将其应用于数据库层面。智能体在评估过程中的所有写入操作不是直接修改生产数据，而是被重定向到一个隔离的“变更表”（changes table）中。这样，通过对比智能体执行的工作流（agent session）与人类专家执行的“黄金标准”工作流（GT session）在数据库中的最终写入状态，来实现评估。

- **关键技术细节：CoW机制**
    1.  **数据库重构**：将每个原始应用表拆分为三部分：
        -   **基表（Base Table）**：存放原始的生产数据，智能体无法直接写入。
        -   **变更表（Changes Table）**：结构同基表，增加 `session_id`、`operation_id` 等元数据列，用于存储智能体在每次会话中的写入。
        -   **视图（View）**：一个SQL查询，将基表和变更表合并，并以原始表名对外提供服务。智能体的所有读写操作都通过这个视图进行。
    2.  **触发器（Triggers）**：在视图上定义 `INSTEAD OF` 触发器。当智能体试图通过视图写入时，触发器被激活，自动将写入操作重定向至对应的变更表，并记录会话和操作元数据。这样，基表（生产数据）始终保持不变。

- **关键技术细节：CoW评分框架**
    - **输入**：一个黄金标准（GT）会话和一个智能体（Agent）会话的数据库变更记录。
    - **评分维度**：分为**结构评分**（写了哪些表、行、列）和**内容评分**（写入了什么具体值）。
    - **评分粒度**：
        - **会话级（Session-level）**：
            - **结构分 (s_struct)**：计算智能体写入的行与GT写入的行之间的匹配率。$$s_{struct} = \frac{N_{matched}}{N_{matched} + N_{missing} + N_{extra}}$$
            - **内容分 (s_content)**：对所有匹配的行，计算其字段值的相似度（基于数据类型，如文本用SequenceMatcher）的平均值。$$s_{content} = \frac{1}{N_{matched}} \sum_{(g,a) \in matched} sim(g, a)$$
            - **总体分 (s_overall)**：上述两个分数的平均值。
        - **操作级（Operation-level）**：评估每个操作对达成最终结果的有用性。
            - **结构效用 (u_struct)**：该操作执行前后，会话级结构分的变化量。
            - **内容效用 (u_content)**：该操作写入的、最终与GT匹配的行的内容相似度平均值。
