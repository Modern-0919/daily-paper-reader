## 摘要
部署用于IT运维的AI智能体通常是永久性成本中心，因为每次执行都需要完整的LLM推理，即使对于先前已解决的问题也是如此。本文引入了渐进式结晶，这是一种将智能体探索视为发现机制而非永久执行模型的生命周期。它定义了一个三阶段执行分类法，从完全智能体编排到混合再到完全确定性工作流，同时结合基于证据的晋升机制，将反复验证的智能体行为转化为更便宜、更可复现的确定性工作流，并自动降级表现退步的工作流。在每月处理数万起事件的云端网络AIOps系统上进行评估，该方法在八个月内将确定性执行从0%提升至45%，尽管事件数量翻倍，但每起事件的智能体成本降低了70%以上，并且通过更强的可复现性和可审计性提升了安全性。本文还介绍了执行分类法、晋升与降级标准、轨迹提取方法、经济模型、安全考量，并讨论了局限性和有效性威胁。

## Abstract
AI agents deployed for IT operations are typically permanent cost centers because every execution requires full LLM inference, even for previously solved problems. This paper introduces progressive crystallization, a lifecycle that treats agent exploration as a discovery mechanism rather than a permanent execution model. It defines a three-stage execution taxonomy, from fully agent-orchestrated to hybrid to fully deterministic workflows, together with an evidence-based promotion mechanism that converts repeatedly validated agent behaviors into cheaper and more reproducible deterministic workflows, while automatically demoting workflows that regress. Evaluated on a production cloud networking AIOps system processing tens of thousands of incidents per month, the approach increased deterministic execution from 0% to 45% over eight months, reduced per-incident agent costs by more than 70% despite doubling incident volume, and improved safety through greater reproducibility and auditability. The paper also presents the execution taxonomy, promotion and demotion criteria, trace extraction methodology, economic model, safety considerations, and discusses limitations and threats to validity.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想：渐进式结晶（Progressive Crystallization）

将工作流执行划分为三个类型，形成从完全智能体编排（随机、昂贵）到完全确定性（零token、可复现）的演化谱系，并通过基于证据的晋升与降级机制，让已验证的行为逐步固化“结晶”。

### 关键技术细节

- **执行类型分类法**（Table I）：
  - **Type 3（Agent-orchestrated）**：智能体在限定范围内自由探索，自主读操作+确定性检查点+人工审核写操作；确定性约50%；token消耗高（1万~5万）。
  - **Type 2（Hybrid）**：固定步骤结构，仅特定阶段调用LLM进行解释/分类；确定性约90%；token消耗低（1千~5千）。
  - **Type 1（Deterministic）**：预编码逻辑，运行时无LLM调用；确定性100%；token消耗为零。

- **结晶生命周期**（四个阶段）：
  1. **发现（Discovery）**：新事件触发Type 3执行，记录完整轨迹。
  2. **捕获（Capture）**：成功执行后，从轨迹提取可复用的Type 3模板（解析工具调用顺序、分支条件、输入输出schema、构建DAG、参数化实例值、标记人工审批点）。
  3. **晋升至混合（Promotion to Hybrid）**：基于多次成功运行，用确定性规则替换稳定分类步骤，用精简prompt替换推理步骤；自动化生成验收测试。
  4. **晋升至确定性（Promotion to Deterministic）**：进一步将有限输出集合或可规则化的决策改为确定性等价物，形成Type 1工作流。

- **晋升与降级门槛**（Table II）：
  - Type 3→2：≥10次成功运行、零安全违规、≥90%执行产生相同动作序列、所有自动验收测试通过、最近窗口无人工覆盖。
  - Type 2→1：≥50次成功混合运行、LLM分类一致性≥99%、确定性规则覆盖所有输入变化、完整回归测试无需LLM通过、专家审查确定逻辑。

- **降级机制**：每个已晋升的工作流持续监控，一旦发生执行失败、安全违规或验收测试退化，电路断路器自动降级到更高类型。例如：固件更新改变命令输出格式导致解析器失败，自动降级到Type 2让LLM处理新格式，之后重新晋升。
