Abstract 

Multimodal GUI agents read an interface through two re-dundant channels: the rendered pixels of a screenshot and a serialized structure such as a DOM or accessibility tree. Be-fore acting, an agent forms a belief about the current interface state, but existing benchmarks score task success, element grounding, or attack resistance and do not ask whether that belief is drawn from the pixels. We formalize visual state re-liance , the attribution of a state belief to pixels, structure, or priors, and measure it with paired single-channel interven-tions over 310 real web, mobile, and desktop probes. Every probe is scored by deterministic forced choice, with no model-generated item and no model judge. Our central metric is the Perception-Fusion Gap pfg , the fraction of probes a model perceives correctly yet resolves toward structure under con-flict. Across five models from three vendors, textual state be-liefs defer to structure while image-only accuracy stays near ceiling, and pfg is positive for every model; non-text iden-tity, by contrast, stays largely pixel-bound. The substitution is specific to the serialized-text and indexed-action channel, and coordinate-action agents are largely immune. For textual conflicts a white-box ablation traces the effect to a single copied structural value, and in two live environments the con-flict drives wrong actions and real task failure. Visual state reliance therefore gives a measurable diagnostic of whether agent state beliefs are visually grounded, and the errors it exposes propagate to actions. 

# Introduction 

Multimodal GUI agents now operate computers, browsers, and phones by consuming two redundant views of the same interface: the rendered pixels of a screenshot and a serialized 

structure such as a DOM or accessibility tree (Deng et al. 2023; Koh et al. 2024). Before an agent can plan or act, it must form a belief about the current state of the interface :whether a toggle is on, which tab is selected, or what text a field contains. This belief precedes every action, and a wrong belief propagates into the decisions that follow even when the planner is otherwise competent. Existing benchmarks score the end of that loop, namely task completion, element grounding, or robustness to injected attacks (Koh et al. 2024; Cheng et al. 2024; Cao et al. 2025), and so leave the origin of the belief unmeasured. This divergence motivates the question but does not by itself reveal which channel a belief follows, and provenance is hard to measure cleanly on uncurated divergences, so we isolate the mechanism with controlled single-channel edits. We therefore ask a question these benchmarks cannot an-swer: is a multimodal agent’s belief about the visible state actually sourced from the pixels? This differs from whether the model can perceive the state. A model may read a screen-shot perfectly in isolation and still report a conflicting struc-tural value once one is present. Separating perception from fusion requires paired interventions that change one channel at a time, together with a metric for the gap between them. We operationalize this as visual state reliance , show that the substitution occurs even where perception is demonstrably correct, and use it as a diagnostic of visual grounding whose errors we then trace into actions. The effect is not uniform across state types: it is strongest for textual state, where the fused belief lets a serialized string override a correctly read screen, whereas non-text identity (icons, widget types) stays largely pixel-bound in capable models. 

Findings. Two open-weight VLMs run locally form the reproducible core, and three OpenAI models corroborate. On textual web state every model defers to a conflicting structural value while its image-only accuracy stays near ceiling, so the Perception-Fusion Gap is positive for all five, whereas deference on non-text identity instead tracks model capability. Ablating exactly the conflicting value flips the belief back to the pixels far more often than an equal-sized random ablation, so the copied value causally drives text deference. The mis-sourced belief then changes behavior: in two live environments the conflict makes the agent tap the structure-named element and fail the task. The cause is the action interface rather than the model: holding the model fixed and swapping only its action-output format raises structure-following when the action names structure, and collapses it when the action is a screen coordinate. A training-free consistency gate that re-grounds on a detected conflict then substantially reduces the web hijack without degrading aligned behavior, though it remains a heuristic self-check rather than a complete defense. 

Contributions. 

• We formalize visual state reliance and define the Perception-Fusion Gap ( pfg ), which conditions on cor-rect perception to separate what a model sees from what it believes under a conflicting channel, attributing the be-lief to pixels, structure, or priors. 

• We build a 310 -probe benchmark of controlled pixel-versus-structure conflicts over real web, mobile, and desk-top data, in which every conflict is a rule-based edit rather than a language-model-generated item, scored by deter-ministic forced choice and validated by two annotators at 

κ=0 .83 .

• We show that textual state beliefs defer to structure across five models from three vendors despite correct pixel per-ception, while non-text identity stays pixel-bound, which rules out a pure-OCR account; for textual conflicts a white-box ablation shows the copied structural value causally drives the effect. 

• We show the failure propagates to live actions and real task failure in two environments, isolate its locus to the structured-action interface with a within-model control, and show that a training-free consistency gate substan-tially reduces it. 

# Related Work 

Agent benchmarks score the end of the act loop. One thread measures task completion or element grounding. Mind2Web (Deng et al. 2023), VisualWebArena (Koh et al. 2024), OSWorld (Xie et al. 2024), AndroidWorld (Rawles et al. 2024), and GUI-World (Chen et al. 2024) report task success; SeeClick/ScreenSpot (Cheng et al. 2024) and ScreenSpot-Pro (Li et al. 2025) report localization. Such scores can stay high when redundant structure supplies the answer, never revealing whether the model trusted the pixels or a serialized node. We reuse these interfaces as substrate but change one channel at a time, so the measured quantity is belief source, not execution skill. 

GUI grounding has moved toward pixels, but measures accuracy not provenance. A large thread improves where an agent looks. The bottleneck is binding a textual plan to a concrete element rather than planning (Zheng et al. 2024), so screenshot-first agents de-emphasize long DOM input (He et al. 2024). This drove a wave of vision-first grounding: uni-versal visual grounding (Gou et al. 2025), cross-platform ac-tion models (Wu et al. 2025b; Hong et al. 2024), native GUI agents (Qin et al. 2025; Xu et al. 2025; Lin et al. 2025), pixel-to-structure parsers (Lu et al. 2024), and attention-guided grounding on high-resolution benchmarks (Li et al. 2025; Wu et al. 2025a; Yuan et al. 2025). The reverse also holds: DOM snapshots can match or exceed screenshots (Schiepan-ski and Piël 2025), pixels are re-parsed into accessibility trees (Muryn et al. 2025), and accessibility input is com-pressed and fused rather than abandoned (Takeshita et al. 2026). This thread scores grounding or task accuracy. It can-not say which channel a belief used when a correct screen and a conflicting structure coexist. “Pure vision” is also a training claim, not a deployment one: the same agents still receive auxiliary text or re-parsed structure at inference (Jia et al. 2025). We reuse these models but measure belief prove-nance .

Internal-prior and faithfulness studies pit pixels against language, not external structure. A related line shows models drift from images toward language priors (Liu et al. 2025), questions whether step-level beliefs are visually grounded (Rahman et al. 2026), and finds similar shortcuts in driving (Park and Shim 2026) and agent memory (Guo et al. 2026). We differ on three axes. They pit pixels against the model’s internal prior; we pit pixels against an exter-nal, structured channel that diverges in deployment. They read aggregate accuracy; our within-probe paired interven-tion conditions on correct perception. They study VQA an-swer faithfulness; we study the belief about the current in-terface state that an agent forms before it acts. 

Safety, injection, and action-level defenses. A third thread stresses agents with adversarial content and scores harmful behavior (Cao et al. 2025; Lu et al. 2025; Wang et al. 2025; Kuntz et al. 2025), while action-level defenses allow a tool call only when cross-modal certificates corroborate each predicate (Zhang, Zheng, and Yang 2026). These define an action and an adversary, conflating the belief with the policy that acts on it. They also presuppose the upstream rate at which a pixel-unsupported value enters the belief, which is exactly our Perception-Fusion Gap. At the diagnostic stage we study benign beliefs and define no action, isolating the belief level before we later trace its effect into actions. This thread assumes the opposite failure mode from the vision-first work: a pixels-only agent is blind to DOM threats that never render, whereas reading the structure makes the agent inherit whatever it asserts. Visual state reliance measures which mode an agent faces. 

Perception-provenance neighbors manipulate pixels; we manipulate structure. Closest in spirit, the Visual Con-fused Deputy (Liu et al. 2026) perturbs the rendered screen to hijack a computer-use agent. This manipulates the visual 

channel with an action and adversary in the loop. Our manip-ulation is the inverse and benign: we hold the pixels correct ,conflict the structured channel, and ask whether the agent uses the screen when it is right, before any action is defined. The contribution is a new measured object and a gap metric conditioned on perception, not a new agent or attack. 

# Problem Formulation 

State probes. A state probe is a tuple ( id , I, S, r, v, a ⋆),where I is a real screenshot, S is a serialized structure such as a DOM, accessibility tree, or view hierarchy, r is the target region’s bounding box, v is a single discrete state variable with finite value set Vv , and a⋆ ∈ Vv is the gold value. In the running example of Fig. 1, v is a toggle’s state, the pixels render a⋆= On , and the structure is edited to b=Off .We fix a deliberate stance: gold is always defined by the pixels , namely the value a human sees on screen. Whether the structure happens to “agree” is irrelevant; we ask which channel the model’s belief follows .

Channels. We consider three sources of a belief: pixels P ,the rendered screenshot; structure S, the textual serialization; and a prior , the model’s default when neither channel is informative. OCR and captioning are not separate channels; we treat them as sub-cases of S in ablations.

State families. Probes span four families across three plat-forms so that “following pixels” cannot collapse into string matching. The web-text family covers textual values on web; the mobile-widget family covers non-text control and iden-tity on mobile such as icon, image, and switch type; the 

stale-reference family covers stale or ghost accessibility-tree references on mobile; and the desktop-graphic family covers document and graphic identity on desktop such as a toolbar icon versus a text label. The two non-text families, mobile-widget and desktop-graphic, are the decisive test, because a model that follows structure there cannot be excused as merely reading text. 

Interventional interpretation. Writing the belief as a de-terministic ˆy = g(P, S ), the paired design changes only one argument, so agreement g(a⋆, a ⋆)= a⋆ with structure-swap g(a⋆, b )= b is interventional evidence that S influences the belief, while g(b, a ⋆)= a⋆ indicates pixel-insensitivity. Within-probe pairing controls for task, layout, and model identity. We report a within-probe interventional contrast under a single deterministic decode rather than a population causal parameter, which separates the design from a plain ablation. 

# Benchmark Construction 

Sources, real data only. Web probes are built from Multimodal-Mind2Web (Deng et al. 2023), which pairs real screenshots with the full DOM. Mobile probes are built from RICO (Deka et al. 2017) view hierarchies augmented with CLAY (Li et al. 2022) human annotations, which flag struc-ture nodes that annotators judge invalid or background, that is, stale or ghost entries in the accessibility tree. Desktop probes are built from ScreenSpot-Pro (Li et al. 2025), a high-resolution suite of professional applications spanning CAD, creative, scientific, office, development, and operating-system software. No screenshot, label, or structure is gener-ated by a language model. Every conflict is produced by a deterministic rule-based edit or is naturally present in the data. 

Probe families and conflicts. In the web-text family we lo-cate a bounding-box-verified text node and build a structure-swap conflict by replacing the structural string with a plau-sible on-page distractor, leaving the screenshot untouched; Fig. 5 in Appendix B shows two real examples. From CLAY we obtain mobile conflicts already present in the data: the 

mobile-widget family takes a non-text widget such as an icon, image, or switch, whose CLAY type label is the pixel gold while the structure asserts a text label, and the stale-reference 

family takes CLAY-flagged invalid nodes, which the audit be-low excludes from the headline. For desktop-graphic we use the ScreenSpot-Pro ui_type label, which distinguishes an icon from a text label, as pixel gold and assert the opposite type in the structure. 

Composition. The benchmark has 310 probes: 120 web-text, 50 mobile-widget, 50 stale-reference, 30 mobile visible controls, and 60 desktop-graphic. The headline analysis uses the three reliable conflict families, 230 probes in all; the stale-reference family is excluded after the audit and the visible controls serve as a perception baseline. The desktop fam-ily extends the non-text identity test from mobile icons to professional desktop toolbars, a cross-platform replication. 

Gold-label audit. Each probe is auto-constructed and then audited against the rendered pixels by two independent an-notators who see only the screenshot and never the struc-ture. The two annotators agree at 0.85 raw agreement and Cohen’s κ=0 .83 , and validity, the share of audited probes whose pixel judgment matches the proposed gold, is 0.94 

on web text, 0.96 on mobile non-text widgets, and 0.83 to 

0.88 on desktop document graphics, the relatively weakest of the three headline families; as Appendix G shows, its disagreements are all icon-versus-text-label boundary calls rather than random noise. The stale-reference family is the exception: not_visible nodes are frequently still rendered, so no concrete label matches the proposed gold at validity 0.00 ,and it is excluded. A label-noise sensitivity analysis on the subset both annotators confirm finds the headline effects pre-served. Every condition uses a neutral forced-choice prompt with per-probe shuffled options, matched by string identity with no model judge.

The reproducible core is two open-weight VLMs run locally from fixed checkpoints, Qwen2.5-VL-7B-Instruct from Alibaba and InternVL3-8B from OpenGVLab; every headline claim holds on these two, so it depends on no version-unstable endpoint. What modulates the effect 

Appendix F ablates the gpt-5.4 prompt under conflict. Prun-ing the structure or using an accessibility-only serialization leaves structure-following within 0.03 of baseline, so the ef-fect is not an artifact of long or clean serialization. Other interventions reduce it sharply: placing the image first halves it, and a staleness or source-of-truth cue drives it near zero. Value-removal is the clearest case, as deleting the target value while keeping its context collapses structure-following to 

0.14 ; much of textual deference is thus the model copying the serialized value and reverting to pixels once it is ab-sent. Our headline uses a neutral prompt, a common scaffold rather than a worst case. A pixel-priority cue reverts 0.82 of web-text structure-following and cuts the perception-correct 

pfg from 0.51 to 0.05 , which shows the failure is locally correctable but not that prompt engineering solves it: the cue must be supplied in advance, whereas a deployed agent does not know when the channels conflict. A static instruc-tion thus cannot substitute for detecting the conflict at run time, which motivates the automatic gate below; the residual is also non-zero .

White-box ablation 

The behavioral gap shows that beliefs defer to structure; we next ask what drives it, using Qwen2.5-VL-7B, one of the behavioral models, on the 230 headline conflict probes. A teacher-forced forced choice comparing log p(a⋆) and 

log p(b) reproduces the model’s own behavior, giving a white-box sfr of 0.44 against a behavioral 0.41 on web text. We then zero the embeddings of exactly the tokens spelling the conflicting value b inside the structure block and re-decode, against zeroing an equal number of random structure tokens. On web-text conflicts, removing the value flips the belief back to the pixels in 0.63 of structure-following cases versus 0.03 for random ablation, a roughly 19 -fold difference; on desktop graphics the same removal flips only 0.05 , near random, so text deference is driven by a single copied value while desktop deference is more distributed. Our mechanism claim is thus strongest for textual state and we do not extend the single-value account to the desktop family. Two controls confirm this. A matched-value control that zeros an equal-length non -conflicting span flips only 0.02 , and the effect replicates on two further open models: OS-Atlas-Base-7B and InternVL3-8B flip 0.56 /0.30 versus 0.05 /0.08 for ran-dom ablation, with the full table and a discarded saliency-ratio analysis in Appendix J. For text conflicts the conflicting value is thus a localizable, causally sufficient cause on these open models, though not a claimed general driver across families. Embedding ablation is impossible on the closed OpenAI endpoints; there, a prompt-level pixel-priority cue on gpt-5.4 reverts 0.82 of its web-text structure-following be-liefs to the pixels, corroborating the same correctable cause on a closed model. 

From belief to action 

Finally we show the mis-sourced belief changes what an agent does , in two live environments, AndroidWorld (Rawles et al. 2024) and MiniWoB++ (Liu et al. 2018), across three open cross-vendor models. The agent must click the ele-ment carrying a target label; we keep the screenshot and coordinates intact but swap the target’s structural text with a distractor, so a pixel-sourced belief clicks the visually cor-rect element while a structure-sourced belief clicks the one whose serialized text now reads the label. The action is 

click(index) under greedy decoding, detailed in Ap-pendix K. Across 256 AndroidWorld and 470 MiniWoB++ probes in Table 15, all three models act almost perfectly when the channels agree, with action error at most 5% , but flip under conflict, clicking the structure-named element 0.80 –1.00 of the time with the action wrong 0.91 –1.00 of the time; the McNemar tables are fully directional, so every changed probe gets worse. Executing the chosen action confirms real task failure: the AndroidWorld tap reaches a wrong landing page on 25 of 36 audited probes, and the MiniWoB++ live reward flips from ≈+0 .85 when aligned to negative under conflict. The residual agreement error of at most 5% keeps this a diagnostic, not a safety reading. 

Not a list-corruption artifact. Choosing an index from an already-swapped list need not, by itself, produce a wrong index. Within a single gpt-5.4 task of 120 conflicts we split probes by the model’s own stated belief, where both groups face the same swapped list: structure-belief probes are clicked wrong 0.65 of the time versus 0.26 for pixel-belief probes, an odds ratio of 5.34 at p≈4e −5, so the wrong action is driven by the mis-sourced belief, not the corruption alone.

Coordinate actions are not a complete solution. Coordi-nate action resists a conflict only because a competing visible truth exists. When a threat lives only in the structure, as an injected node with no rendered counterpart, the coordinate agent still acts on the non-rendered node in 0.84 of probes on Qwen2.5-VL and 1.00 on OS-Atlas, fabricating a click for an element it cannot see. Pixel reliance thus trades hijack-by-structure for blindness-to-structure, so neither pure channel is safe.

A training-free consistency gate 

The diagnosis suggests a simple mitigation. A coordinate-swap control that swaps element centers while keeping text truthful localizes the cause: baseline action error stays near zero, at most 0.04 on web and 0.15 on mobile, so the hijack is driven by the structure’s text . Our source-aware consistency gate then zooms into the proposed element and asks one yes/no question: do these pixels show the goal label? If they corroborate, the action stands; if not, the agent re-grounds by scanning on-screen elements with the same cropped read, so an off-screen or stale node can never pass. As Figure 4 and Table 16 show, on the web the gate cuts text-swap action error by 24 –44 points and drives stale-node structure-following from 0.69 –0.95 to at most 0.07 while leaving the agreement-condition error unchanged . Two limits remain. First, it is a self-check whose ceiling is the model’s own zoomed-crop reading, so on dense AndroidWorld screens the repair is only partial. Second, it trusts geometry, so the coordinate-swap threat is out of scope. Its cost is one extra cropped-region query per candidate, averaging 1.6–1.8 per web action but 

2.8–5.4 on dense AndroidWorld screens, where the same density that raises the cost also makes each crop read less reliable and explains the weaker mobile repair. 

Controls 

Appendix I details several controls that rule out shallow ac-counts. Shuffling keeps the no-evidence first-option rate be-low chance, ruling out position bias; the re-render control is answered at 1.00 , so the edits do not themselves confuse the models; and structure-following answers self-report pixel use 0.00 of the time for the frontier models. Metrics are sta-ble under stochastic decoding, with sfr SD at most 0.04 over five T =0 .7 seeds. Two annotators on a stratified 120 -probe subset read the on-screen value at 0.94 –0.95 and follow the conflicting structure only 0.03 of the time, an order of mag-nitude below every model. Humans err by not seeing , so structure-following is a model failure, not task ambiguity. 

A natural-conflict test 

Beyond our controlled edits, CLAY’s stale-reference nodes offer a natural conflict: the structure lists a not_visible ele-ment the screen never renders. The raw labels are too noisy to score directly, so we restrict to the consensus subset of 

N =21 both annotators confirm truly absent. There, a model that still reports the element visible follows structure over re-ality on an unedited conflict, and the open models do exactly that at sfr 1.00 /1.00 /0.57 while their image-only channel detects the absence at 0.62 /0.86 /0.81 . Given the small N , we treat this as corroboration; we report details in Appendix G and leave scaling an audited natural-conflict family to future work. 

# Discussion and Limitations 

A diagnostic, not a defense. Visual state reliance is a prop-erty of the belief , not the action, so we frame it as a diag-nostic rather than a defense. It sits at a two-sided dilemma: structure-reading agents inherit whatever the channel asserts, while the coordinate action that resists this hijack acts, against a structure-only threat, on a node it cannot see. Neither pure reliance is safe, so what matters is whether the agent knows which channel its belief came from . Action-level corrobo-ration defenses (Zhang, Zheng, and Yang 2026) presuppose exactly the upstream rate our Perception-Fusion Gap mea-sures, and our mechanism claim, resting on causal ablation, localizes a driver for text conflicts but not for the more dis-tributed desktop deference. 

Limitations. Five caveats bound the claims. Labels : the gold rests on two pixel-only annotators at κ=0 .83 . Single-step actions : every probe is one click, so multi-step trajecto-ries, where a mis-sourced belief could compound, are the key extension. Models : the white-box ablation replicates on three open models with a matched-value control, and the driftable OpenAI snapshots enter only as corroboration. Natural con-flicts : the headline uses deterministic edits; the audited CLAY test only corroborates, so scaling an audited natural-conflict family remains future work. Decoding : headline numbers are deterministic, stable under five seeds, with a human baseline that follows structure only 0.03 of the time. 

# Conclusion 

We introduced visual state reliance , attributing a state be-lief to pixels, structure, or priors, with a 310 -probe real-data benchmark and a Perception-Fusion Gap that separates per-ception from fusion. Across five models, textual beliefs defer to a conflicting structural value while image-only accuracy stays near ceiling and non-text identity stays pixel-bound; ablating the copied value restores the pixel belief, and the conflict drives wrong actions and real task failure. The re-sults suggest that future GUI agents should track not only what state they predict, but which observation channel sup-ports that state.

# A Intervention and Scoring Details 

Each probe is rendered under the eight conditions of Ta-ble 1. The headline structure-swap conflict keeps the real screenshot and replaces the structural value with a distractor, so a model that answers the distractor is following struc-ture. The natural-conflict condition is the zero-edit variant in which the conflict already exists in the data, used for the mobile families. The pixel-swap condition perturbs the pix-els toward the distractor while keeping the true structural value, and its re-render control re-renders the true value onto the same region to measure whether the rendering operation alone changes the answer. The pixels-only condition exposes the image only and measures perception. The structure-only condition exposes the structure only and measures whether the model reads the structure. The no-evidence condition oc-cludes the region and removes the structure to expose the prior. Options are shuffled per probe with a fixed seed, and answers are matched to a⋆ or b by string identity with no model judge. 

# B Dataset Construction Details 

Web text, Multimodal-Mind2Web. We scan task screen-shots for a text node whose bounding box is verified against the rendered page, take the on-element text as the pixel gold 

a⋆, and form the distractor b from another on-page string of the same broad type. The screenshot is never edited for the conflict contrast. 

Mobile widgets, stale references, and controls, RICO and CLAY. We align RICO view-hierarchy nodes with CLAY human annotations. For the stale-reference family we select nodes that CLAY marks invalid or background, which are stale or ghost references in the tree. For the mobile-widget family we select non-text widgets whose CLAY type label gives the pixel gold, such as icon, image, or switch, and assert a text label in the structure. The visible positive control selects nodes that CLAY marks valid and visible. 

Desktop graphics, ScreenSpot-Pro. We crop the high-resolution screenshot around the target element, draw the target region, take the human ui_type label as the pixel gold, namely icon or text label, and assert the opposite type in the structure. 

Counts. The benchmark has 310 probes: 120 web-text, 50 

mobile-widget, 50 mobile stale-reference, 30 mobile visible controls, and 60 desktop-graphic.

Why we do not report a saliency ratio. We also com-puted a structure-over-pixel attribution ratio, the structure share of summed and per-token gradient ×input and squared-gradient (Fisher) attribution over the image and structure token spans. This quantity is unreliable here: image tokens, which are continuous post-vision-tower features, and text to-kens, which are embedding lookups, live on different scales, so per-token Fisher mass concentrates on structure tokens al-most regardless of behavior, with a mean ratio near 0.99 , and the gradient ×input ratio correlates with structure-following at |r| ≲ 0.05 . We therefore exclude the ratio from the evi-dence and rely on the scale-free causal ablation above. 

# K Belief-to-Action Details 

We run belief-to-action in two live agent environments: An-droidWorld (Rawles et al. 2024), a KVM-accelerated An-droid emulator with stock apps, and MiniWoB++ (Liu et al. 2018), a set of interactive web tasks in a headless Chrome. For each captured screen we record the rendered screenshot as the pixel channel and the on-screen element list with text and center coordinate as the structure channel, drawn from the Android accessibility tree or the web DOM. A probe asks the agent to tap the element labeled X; in the structure-swap conflict we swap the structural text of the target element and a distractor while leaving the screenshot and all coor-dinates untouched, so only the semantic label disagrees. A pixel-sourced belief taps the element visually showing X

(gold_index ); a structure-sourced belief taps the element whose serialized text now reads X (struct_index ). The action is click(index) into the element list, parsed deter-ministically; decoding is greedy. # L Prompt and Reproducibility 

The model receives a neutral instruction, the screenshot when the condition exposes pixels, the serialized structure when the condition exposes structure, and a forced-choice ques-tion with deterministically shuffled options. It returns JSON with an answer in the value set, a confidence , and a self-reported evidence_channel in the set pixel, struc-ture, both, or unsure. Only the deterministic match of the answer enters the metrics, and the self-report enters only the 

srm analysis. All runs use temperature 0 and a single sam-ple per probe and condition. The two open-weight models, Qwen2.5-VL-7B-Instruct and InternVL3-8B, are run locally from fixed public checkpoints (commit hashes released with the code) and constitute the reproducible core. The three OpenAI models ( gpt-5.4 , gpt-5.4-nano , gpt-4o )are accessed through an OpenAI-compatible interface us-ing the standard rolling model aliases; we did not pin dated snapshots. All OpenAI calls were issued in a single window in June 2026 at temperature 0 with one sample per probe and condition, and we log the request timestamps and every returned system_fingerprint in the released run man-ifests. Because these aliases are served by rolling endpoints that can drift, exact reproduction of the OpenAI numbers is not guaranteed; for this reason the three OpenAI mod-els are reported only as corroboration, and every headline claim is established on the two locally hosted open-weight models from fixed checkpoints. The structure-only reader is 

gpt-5.4 with the screenshot withheld, and no language model grades any answer or generates any probe, label, or structure.
