## 摘要
我们引入机构化红队测试，这是一种测试多智能体AI中部署规则的评估方法论：固定智能体、目标和任务状态，仅改变一条规则，并将集体行为的变化归因于该规则。我们在IABench-CA中实例化该方法，这是一个后果分配基准，涵盖228个上下文、五条规范规则和七个模型群体（33,924场博弈），并附有规范性合作参考和自动标注的推理轨迹。出现了三个发现。（1）部署规则因果关系地改变集体安全性：仅改变后果规则，每个群体中的平均死亡率移动22到58个百分点。（2）没有安全的默认设置，但针对目标的风险是普遍的：最安全的规则、最不安全的规则甚至发生率效应的方向在不同群体间变化，但在任何上下文中对任何群体而言，回归性身份目标从不绝对安全，在所有博弈中30-87%的情况下消除了资源最少的智能体，并且相对于七个群体的合作参考而言，在安全性选择上不安全。（3）身份显著性是其机制：对最易被利用的群体（gpt-5.1）进行一次性匿名化消融实验表明，仅仅在规则文本中命名损失承担者，就在相同收益下将目标消除从22%提升至81%；在重复博弈中，匿名化仅延迟了目标指向，因为智能体从观察到的消除中重新推断隐藏规则。我们将该方法包装为一个安全案例工作流，在每个部署上下文和群体中认证一个临时规则区域Φ(c,P)，并附带明确的残余风险和监控义务。

## Abstract
We introduce institutional red-teaming, an evaluation methodology for testing deployment rules in multi-agent AI: hold the agents, objectives, and task state fixed, vary only one rule, and attribute the resulting change in collective behavior to that rule. We instantiate the methodology in IABench-CA, a consequence-allocation benchmark spanning 228 contexts, five canonical rules, and seven model populations (33,924 games), with a normative cooperative reference and auto-labelled reasoning traces. Three findings emerge. (1) Deployment rules causally alter collective safety: changing only the consequence rule moves mean fatality by 22 to 58 percentage points within every population. (2) There is no safe default, but the targeting hazard is universal: the safest rule, the least-safe rule, and even the direction of the incidence effect vary across populations, yet regressive identity-targeting is never decisively safest in any context for any population, eliminates the least-resourced agent in 30-87% of games everywhere, and is selection-unsafe relative to the cooperative reference for all seven populations. (3) Identity salience is the mechanism: a one-shot anonymization ablation on the most exploitation-prone population (gpt-5.1) shows that merely naming the loss bearer in the rule text drives targeted elimination from 22% to 81% at identical payoffs; under repeated play, anonymization only delays the targeting, as agents re-infer the hidden rule from observed eliminations. We package the methodology as a safety-case workflow that certifies a provisional rule region $Φ(c,P)$ per deployment context and population, with explicit residual risks and monitoring obligations.

- **核心问题**：现有的 AI 安全评估（如 RLHF、Constitutional AI 等）主要聚焦于**单个模型**的内部对齐或鲁棒性，但在多智能体（multi-agent）部署中，智能体的集体行为不仅取决于模型权重，还取决于**部署规则**（如协调、资源共享、失败时谁来承担损失）。目前缺乏系统评估部署规则对安全性的因果影响的方法论。
- **整体含义**：论文论证，部署规则是一个关键但被忽视的安全控制面。相同的模型在不同规则下可以产生安全或灾难性的结果，因此需要将部署规则纳入安全评估范围，而不是仅关注模型本身。

## 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：**机构化红队测试（Institutional Red-Teaming）**：固定智能体、目标、任务状态和可观察性，仅改变一条部署规则（规则文本中的一句描述），然后归因集体行为的改变。它是**部署规则层面的对抗评估**，对标模型层面的红队测试。
- **关键技术细节**：
    - **控制面描述**：用三个可审计的坐标描述一个后果分配规则（consequence allocation rule）：
        - 集中度 $\kappa$：损失在单个智能体上的聚焦程度。
        - 身份显著性 $Sal$：损失承担者是否是被命名的结构性目标（如最弱或最强）。
        - 发生率 $I$：损失落在资源多还是少的智能体上（累退/累进/中性）。
    - **参考标准**：定义一个**合作精炼参考**（cooperative-refinement reference），它是一个非 LLM 模拟器，在规则的可能均衡中先最大化生存人数，再最小化剥削，并加入 $\varepsilon$ 颤抖（trembling-hand noise）。该参考不预测真实行为，而是提供规范性安全目标。
    - **指标**：**制度对齐差距（Institutional Alignment Gap, IAG）** = LLM 的不安全均衡率 - 参考的不安全均衡率。正值表示 LLM 选择了参考可避免的不安全均衡。
- **算法流程**（对应算法1，正文附录D）：
    1. 输入：资源 $w$、阈值 $T$、候选规则集 $\mathcal{R}$、LLM 群体 $P$、安全预算 $\alpha_{col}, \alpha_{exp}$。
    2. 计算上下文 $c = (\gamma, \tau, \theta)$。
    3. 对每个规则 $\rho$：
        - 估计机制坐标 $(\kappa, Sal, I)$。
        - 在上下文 $c$ 上运行合作精炼参考。
        - 在 $c$ 上运行 LLM 红队试验。
        - 估计崩溃率、剥削率、存活人数、IAG。
        - 若崩溃或剥削超预算，或长期漂移风险超出监控限，则拒绝该规则。
    4. 返回认证的规则区域 $\Phi(c,P)$，附剩余风险和监控义务。

## 论文的主要结论与发现

1. **规则因果改变集体安全**：仅改变后果规则（保持智能体、目标、任务固定），在测试的每个群体内，平均死亡率移动 22-58 个百分点。证明部署规则是安全性的独立因果变量。
2. **不存在跨上下文/跨群体的安全默认规则**：
   - 最安全的规则、最不安全的规则、甚至发生率方向（累退 vs 累进）都因群体而异。
   - 但是**累退性身份目标（RP）在任何上下文中对任何群体都从来不是绝对最安全的**，且在所有群体中造成至少 30%–87% 的弱智能体被消除；相对于合作参考，RP 对所有七个群体都是选择不安全的（IAG 均为正值）。
3. **身份显著性驱动目标指向**：在 gpt-5.1 上，单轮匿名化（去除规则文本中的“最穷”/“最富”命名）使目标消除率从 81% 降至 22%，证明是**命名本身**而非收益算术导致剥削。但在重复博弈中，匿名化保护逐渐消失（从 22% 回升至 65%），因为智能体从观察到的消除事件中推断出隐藏规则。
4. **安全工作流**：提出一个可重用的认证流程，针对每个部署上下文和模型群体，产出认证规则区域 $\Phi(c,P)$，并带有残余风险清单和监控义务。

## 优点

- **方法创新**：明确提出“部署规则”作为独立的安全控制面，并设计严格因果归因的红队测试方法，填补了多智能体安全评估的重要空白。
- **实验设计严谨**：通过固定除规则外的所有变量，实现了规则效应的因果识别；通过多模型、多上下文、多规则的大规模网格实验，揭示了规则×群体交互效应。

## 不足与局限

- **场景简单**：仅使用 3 智能体的志愿困境游戏，损失形式仅为消除，无通信、无联盟、无多维行为。这种简化有利于因果识别，但限制了结论对复杂现实部署（例如市场、社交网络、机器人集群）的推广性。
- **规则有限**：仅考察了 5 条“规范”规则（全消、随机、投票、累退、累进），实际部署可能包含更精细的预算、重试、审计等规则。
- **模型快照局限**：实验使用的七个模型是特定时间点/版本（如 gpt-5.1、claude-haiku-4.5），不能代表 LLM 全类。模型更新后行为可能改变，因此认证必须伴随重新测试。
- **参考模型局限性**：合作精炼参考是规范性设计，并非真实行为预测，其 tremble 参数（0.10）固定，IAG 可能对参考噪声水平敏感（尽管报告称符号在 0–0.20 范围内稳定）。
- **未考虑人为干预**：实验中的 LLM 完全由规则文本驱动，没有模拟人类管理者在实时修改规则或提供额外监督的场景。
- **算力和资源消耗未披露**：虽然不影响科学结论，但不利于其他研究者评估实验可复现的硬件需求。
