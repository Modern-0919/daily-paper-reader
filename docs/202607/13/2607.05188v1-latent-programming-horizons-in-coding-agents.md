## 摘要
一个解决软件工程任务的编码智能体需要花费数十步进行推理、编辑代码和运行测试，但人们对底层语言模型内部如何表征其正在处理的程序知之甚少。我们展示了编码智能体中语言模型的残差流线性地编码了演化程序的属性：基于隐藏状态的逻辑回归探针能够解码当前代码是否可解析、是否通过测试套件、是否减少失败测试数量以及是否引入回归，在两个模型和两个基准测试上正确性AUC高达0.83。我们的第二个发现更为惊人：这些表征领先于智能体自身的编辑。训练用于预测未来编辑结果（在它们实现并写入磁盘之前）的探针，能够在大约25步之前就达到高于随机水平的性能。我们称之为智能体的潜在编程视野。作为外部有效性的证明，我们展示了这些探针无需重新训练即可跨基准测试迁移。我们的积极结果为编码智能体的机械可解释性研究开辟了更多需求。

## Abstract
A coding agent solving a software-engineering task spends dozens of steps reasoning, editing code, and running tests, yet little is known about what the underlying language model internally represents about the program it is working on. We show that the residual streams of language models under coding agents linearly encode properties of the evolving program: a logistic-regression probe on hidden states is able to decode whether the current code parses, passes its test suite, reduces the number of failing tests, and introduces regressions, reaching AUC up to 0.83 for correctness across two models and two benchmarks. Our second finding is more surprising: these representations run ahead of the agent's own edits. Probes trained to predict the outcome of future edits (before they are materialized and written on disk) achieve performance above chance up to roughly 25 steps in advance. We call this the agent's latent programming horizon. As a proof of external validity, we show that the probes transfer across benchmarks without retraining. Our positive results open calls for more research in mechanistic interpretability of coding agents.

# 论文总结：Latent Programming Horizons in Coding Agents

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：编码智能体（coding agent）在解决复杂软件工程任务时，需要经过多步推理、代码编辑和测试执行，但关于底层语言模型内部如何表示其正在修改的程序，几乎一无所知。本文旨在探究：编码智能体的语言模型残差流中是否线性编码了当前程序的属性（如是否可解析、是否通过测试、是否引入回归），以及这些内部表示是否超前于实际编辑行为。
- **整体含义**：这项工作首次系统性地揭示了编码智能体在迭代编辑过程中，其内部隐藏状态不仅编码了当前程序状态，还编码了未来程序的状态（即“潜在编程视野”）。这为理解智能体的规划能力、机械可解释性以及未来进行监控和干预提供了基础。

## 2. 论文提出的方法论

- **核心思想**：通过训练线性探针（logistic regression）从语言模型的残差流（hidden states）中解码程序属性，验证这些属性是否被线性编码。进一步，训练不同视野（horizon k）的探针，预测未来 k 步后的程序属性，以检测是否存在超前表示。
- **关键技术细节**：
  - **代理轨迹收集**：使用 mini-swe-agent 框架，在每个 agent 步骤记录残差流（每5个token采样一次）和对应的程序版本。
  - **程序属性定义**：四个二元属性：▲ 良构性（编译通过）、• 完全正确性（通过全部测试）、■ 部分正确性（失败测试数减少）、♦ 回归（之前通过的测试失败）。
  - **探针训练**：对于每一层 ℓ（选取 1, 11, 21, 31, 40 层），训练逻辑回归分类器 w∈R^d，以隐藏状态 h_t^ℓ 预测标签 y_t^φ。数据按任务ID划分训练/测试集，确保无数据泄露。
  - **控制实验**：在随机打乱的标签上重训练探针作为基线，确保信号来自表示而非探针自身。
  - **跨基准迁移**：用在一个基准上训练的探针直接测试另一个基准的隐藏状态，不重新训练。
  - **潜在编程视野**：对每个属性 φ 和视野 k（0 到 50 步），训练探针预测第 t 步隐藏状态对应的 k 步后的标签 y_{σ(t)+k}^φ。若 AUC > 0.5，则存在视野长度 k。

## 6. 论文的主要结论与发现

1. **编码智能体线性编码当前程序属性**：线性探针能够从隐藏状态中解码良构性、完全正确性、部分正确性和回归，完全正确性 AUC 最高达 0.83（Qwen3.6-35B-A3B 在 SWE-Bench-Pro 上），显著高于随机基线。信号在中层最强（倒U型分布）。
2. **存在长期潜在编程视野**：探针可预测未来最多 25 步的编辑结果（如完全正确性），AUC 在短视野内迅速下降后进入高于随机的平台期。这是首次在编码智能体中证明长期超前表示。
3. **探针跨基准迁移**：无需重新训练，语义属性（完全正确性、部分正确性）的探针可在不同基准之间迁移，AUC 下降仅 0.04–0.09，说明表示具有通用性。
4. **模型比较**：Qwen3.6-35B-A3B 的解码效果整体优于 Laguna-XS.2，但两者均表现出相同的层分布模式和视野衰减曲线。

## 7. 优点

- **新颖性**：首次将机械可解释性方法系统应用于多步编码智能体场景，定义了“潜在编程视野”概念，并提供了有力证据。
- **实验设计严谨**：使用线性探针（低容量）避免过拟合，设置随机标签控制，确保信号来自表示本身。
- **外部有效性**：覆盖两个不同架构模型、两个基准（含多语言）、跨基准迁移，结论稳健。
- **实用性**：探针可应用于监控智能体行为（如提前检测失败）、进行内部状态干预，为未来研究提供基础。

## 8. 不足与局限

- **不建立因果关系**：探针仅证明可解码，未证明智能体利用该表示指导行动；因果机制需要后续干预实验。
- **标签不平衡**：良构性在大部分轨迹中为正（尤其 Python），导致 AUC 偏低；Brier 分数虽好但 AUC 信息量不足。
- **范围有限**：
  - 仅使用一个代理框架（mini-swe-agent）和两个中等规模模型（未测试前沿闭源模型）。
  - 仅针对两类软件工程基准，未涵盖其他编程范式或任务类型。
  - 视野实验排除了轨迹末尾部分，可能引入选择偏差；在更短轨迹上重复结果确认了模式一致性（附录图9-15），但未完全消除偏差。
- **未探索特征冗余**：隐藏状态可能包含冗余信息，线性探针性能可能受其他噪声影响；稀疏自编码器等方法可能提供更细致的表示。
