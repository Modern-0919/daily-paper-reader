## 摘要
大型语言模型编码代理越来越多地执行开放式数据建模和分析。这些代理具有随机性和自适应性，因此其自主模型发现行为无法通过单一基准运行充分表征。在本工作中，我们提出了一种实验设计和分析框架，用于系统评估这一发现过程，量化其变异性，并识别重要因素。所提出的框架将这些代理视为随机的模型发现算子，将任务特定的发现数据和优化目标映射到拟合模型上。具体来说，我们在受控实验因素下研究了两个这样的算子：Codex和Claude Code，因素包括代理的推理努力、任务、优化指标以及训练数据的构成。对于每个代理-任务-指标组合，对多个响应（如输出质量、美元成本、挂钟时间和过程复杂度）进行回归模型和推断。此外，我们开发了一种效用对齐的规范分解，以表征推理努力效应的主导方向，并评估该方向是否与性能-成本效用方向一致。所提出的框架在一个网络化单词形成游戏测试台上进行了演示，并获得了关于推理努力在成本和过程复杂度方面的深刻见解。

## Abstract
Large language model coding agents increasingly perform open-ended data modeling and analysis. These agents are stochastic and adaptive, and therefore their autonomous model discovery behavior cannot be adequately characterized by a single benchmark run. In this work, we propose an experimental design and analysis framework for systematically evaluating this discovery process, quantifying its variability, and identifying important factors. The proposed framework treats these agents as stochastic model-discovery operators, which map task-specific discovery data and an optimization target to a fitted model. Specifically, we investigate two such operators, Codex and Claude Code, under controlled experimental factors including agent's reasoning effort, task, optimization metric, and composition of training data. For each agent-task-metric combination, regression models and inference are conducted for multiple responses such as output quality, dollar cost, wall-clock time, and process complexity. Furthermore, we develop a utility-aligned canonical decomposition to characterize the dominant direction of the reasoning-effort effect and to assess whether that direction aligns with a performance-cost utility direction. The proposed framework is demonstrated on a testbed of networked word-forming games with insightful findings on reasoning effort with respect to cost and process complexity.

## 3. 实验设计

### 数据集与场景
- **数据来源**：网络化组字谜游戏（group anagram game, Cedeño-Mieles et al., 2019），包含28个会话、209名玩家、62,700条玩家-时间记录。玩家在通信网络上，每5分钟轮次内选择4种动作（空闲、回复、请求、组词）。数据分为训练和评估部分。
- **任务**：
  - **Task 1 (timestep)**：下一动作四分类预测（基于历史+网络上下文）。
  - **Task 2 (ABM)**：生成式代理基模型模拟（生成完整会话轨迹）。
- **指标**：
  - Task 1: 加权AUC（wAUC）、罕见观测相对改进均值（MRI-RO）
  - Task 2: 玩家级摘要的KL散度（KL6）、Levenshtein距离分布距离（DLD）
- **对比方法**：不直接对比其他基线方法，而是对比两个代理（Codex、Claude Code）在相同条件下的表现，并分析其内部自我评估与实际评估的差异。

### Benchmark
- 无标准外部 benchmark，论文自身构建了基于因子设计的评估流水线，将代理的发现过程作为实验对象。
- 实验使用3折交叉验证（折叠水平），每折内进行全量/部分发现体制比较。

## 5. 实验数量与充分性

- **实验数量**：计划 144 次运行（全因子设计：2×2×2×3×3×2），实际有效运行 140 次（4次因模拟有效性低于90%被剔除，均来自 a-DLD 条件）。
- **分层**：8 个分层（2 agents × 2 tasks × 2 metrics），每个分层包含 18 或 16 次运行。
- **充分性评价**：
  - **优点**：因子设计覆盖了关键处理变量（推理努力）和分块变量（折、体制），可以估计主效应。
  - **不足**：
    - **无重复**：每个配置只运行一次，无法估计纯随机误差，依赖模型假设（正态、独立同分布）。

    - **提示词不一致**：Codex Task-1 全量体制运行使用了稍有差异的提示词版本，与体制完全对齐，限制了体制因果分析（论文已诚实指出）。
    - **成本估算**：Codex 成本为 imputed 而非直接计量。
- 总体而言，实验设计系统性好，但充分性受限于无重复和小样本规模；论文将结果描述为“证明技术可行”而非“全面调查”。

## 6. 论文的主要结论与发现

1. **推理努力主要影响成本与过程复杂度，而非性能**：提高推理努力显著增加 token 消耗、代码长度、候选模型数量、运行时间，但未带来匹配的性能提升。定向效用检验表明：在 7/8 分层中更多努力显著降低效用（负斜率），Holm 校正后仍显著。
2. **UACD 结果**：所有 8 个分层中的主导典型方向都与效用方向不一致（η(₁)<0），符号检验 p=0.008，描述性支持“推理努力不利于效用”的结论。
3. **代理专业化**：代理对优化指标有强专业化，优化指标上的得分显著优于非优化指标。
4. **训练数据量影响**：更多训练数据仅在 a-KL 条件带来明显提升，其他条件差异不大。
5. **成本差异**：Claude Code 中位数成本约 $4.19，Codex 约 $0.60（相差约7倍），但两代理质量相近。
6. **自我评估偏差**：在 Task 1 上自我评估与实际评估一致，在 Task 2 生成任务上系统性地过度乐观（尤其是 DLD 条件，4次失败运行均来自该条件）。

## 7. 优点

- **方法创新性**：将实验设计引入 LLM 代理评估，系统控制变量、记录多维响应，比简单基准测试更全面。提出 UACD 方法，将多变量效应分解为易解释的典型方向，并与预设效用对齐比较。
- **多变量视角**：联合分析性能、成本、过程复杂度，揭示单变量分析（仅看性能）会错失“努力主要影响成本”的重要模式。
- **框架可推广**：方法论不限于具体游戏数据，可应用于其他代理-任务场景。
- **实验严谨性**：采用因子设计（分层+分块+处理因子），控制折叠和体制，使用标准化、正交对比编码。明确了 4 次无效运行并分析失败原因。
- **开源可复现**：提供 GitHub 仓库和评估流水线。

## 8. 不足与局限

- **实验无重复**：每个配置仅一次运行，无法估计纯噪声方差，多变量检验功效受限。
- **代理数量少**：仅两个商业代理（Codex, Claude Code），且版本固定，结论泛化性有限。
- **成本估算不精确**：Codex 成本基于 token 数乘以固定费率，而非实际 API 计费；Claude Code 成本来源未详述方法。跨代理成本比较属于“部署经济学”而非控制对比。

- **计算资源未规范报告**：未提供 GPU 型号、训练时长等信息，限制了能耗/复现性讨论。
