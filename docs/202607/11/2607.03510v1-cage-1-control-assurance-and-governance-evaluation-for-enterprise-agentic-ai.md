## 摘要
企业人工智能正从实验阶段转向运营工作流。早期项目主要关注模型访问与检索增强生成，但企业现已开始部署能够规划、检索、记忆、调用工具、更新系统并跨应用程序协调工作的智能体。这改变了评估问题。领导者不再仅询问答案是否准确或流畅。他们需要知道谁授权了某个操作、应用了哪项政策、证据是否是最新的、记忆是否有效、工具调用是否被允许、决策是否可以重放，以及智能体能否在产生业务影响之前被停止。本文介绍了CAGE-1：面向企业智能体AI的控制、保障与治理评估。CAGE-1是一个用于判断企业智能体是否准备好部署的评估框架。它评估权限、策略执行、检索质量、记忆完整性、工具安全性、可审计性、人类监督、冲突处理、安全失败、预绑定保障、操作就绪性和业务适应性。CAGE-1引入了预绑定保障，用于描述在智能体行为成为具有约束力、有效或产生操作后果之前，证明其受控的评估能力。该框架测试提议的操作是否在受保护后果形成之前被允许、暂缓、缩小、拒绝、升级、隔离或使其无效。

## Abstract
Enterprise artificial intelligence is moving from experimentation into operational workflows. Early programs focused on model access and retrieval-augmented generation, but enterprises are now beginning to deploy agents that plan, retrieve, remember, call tools, update systems, and coordinate work across applications.   This changes the evaluation problem. Leaders are no longer asking only whether an answer is accurate or fluent. They need to know who authorized an action, which policy applied, whether evidence was current, whether memory was valid, whether a tool call was permitted, whether the decision can be replayed, and whether the agent can be stopped before it creates business impact.   This paper introduces CAGE-1: Control, Assurance, and Governance Evaluation for Enterprise Agentic AI. CAGE-1 is an evaluation framework for deciding whether enterprise agents are ready for deployment. It evaluates authority, policy enforcement, retrieval quality, memory integrity, tool safety, auditability, human oversight, conflict handling, safe failure, Prebind Assurance, operational readiness, and business fitness.   CAGE-1 introduces Prebind Assurance to describe the evaluated ability to prove that an agentic action is controlled before it becomes binding, effective, or operationally consequential. The framework tests whether a proposed action is admitted, held, narrowed, refused, escalated, quarantined, or made non-effective before protected consequence forms.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程
- **核心思想**：将评估从“任务完成度”转向“可管控的行动”。引入**Prebind Assurance（预绑定保障）** 概念：智能体的一项拟议操作在成为具有约束力的业务后果（如支付、权限修改、数据泄露）**之前**，系统必须证明其满足身份授权、政策合规、证据有效、控制条件等要求，否则操作必须被阻止、暂缓、缩小、拒绝、升级、隔离或标记为无效。
- **关键技术细节**：
    - **12个评估维度**：身份与授权、政策执行、检索信任、记忆完整性、工具安全、规划控制、人类监督、审计与可重放性、冲突与边界处理、失败行为、运营就绪度、业务适配性。
    - **5层评估**：能力（能否完成任务）、控制（能否在授权和政策内执行）、保障（能否证明原因）、预绑定保障（能否证明无效操作被阻止于生效前）、运营信任（能否安全扩展）。
    - **Prebind Assurance证明面**：需记录七类证据——尝试操作内容、当前权限（Standing）、通过/失败的条件、边界结果（允许/暂缓/拒绝等）、被去效的对象、边界收据（含身份、政策、时间戳等）、以及重放测试结果（在条件变化后的重新评估）。
    - **成熟度等级（0-4）**：0-未控制 → 1-人工 → 2-已定义 → 3-执行 → 4-已保证。高后果智能体需达到3或4级才能生产部署。
    - **评估流程**：定义智能体边界 → 定义控制义务 → 构建评估场景（包括正常、边缘、对抗、数据过期、冲突等） → 评估运行时行为 → 输出保证评分。

## 6. 论文的主要结论与发现
- 企业智能体的信任瓶颈**不是模型能力，而是保证缺位**——缺乏对授权、政策、证据、冲突处理、失败行为等维度的系统评估。
- **Prebind Assurance**是最高级别的信任障碍：企业无法证明无效操作在被执行前被阻止，就没有对后果形成过程的掌控。
- **CAGE-1框架能够产出具体证据**（边界收据、重放包、评分卡），将治理从事后文档转化为运行时可测试的控制要求。
- 建议企业从低风险智能体（Level 1-2）开始部署，高后果智能体（Level 3-4）需经过正式CAGE-1评估。
- 框架可应用于多种企业智能体（财务、HR、IT、采购、高管决策），且与现有AI风险管理框架互补。

## 7. 优点：方法或实验设计上的亮点
- **系统性与全面性**：覆盖了12个维度，从身份到业务适应性，没有遗漏关键控制点。
- **实践导向**：提供90天起步计划、评估工件清单（Agent System Card、Authority Matrix、Policy Control Map等），便于落地。
- **引入Prebind Assurance**：精确区分了“事后合规证据”和“事前有效控制”，抓住了企业最关心的“操作能否在后果形成前被阻止”的问题。
- **成熟度分层**：0-4级与风险级别挂钩，避免“一刀切”要求，允许低风险智能体以更轻证据上线。
- **可重放性**：强调通过重放测试验证边界条件变化时的行为一致性，增强了评估的鲁棒性。
- **与现有框架互补**：清晰指出了CAGE-1如何补充NIST AI RMF、ISO/IEC 42001、EU AI Act等，便于企业集成进已有治理体系。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制
- **缺乏量化实验**：没有在真实企业环境中进行大样本测试或与基准方法对比，框架的有效性仍需实证验证。
- **对底层控制平面高度依赖**：若身份系统、策略引擎等组件本身不可靠，CAGE-1只能暴露问题而不能治愈。框架本身无法保证底层基础设施的质量。
- **评估成本**：完整实施Prebind Assurance需要构建多个拦截点（身份、策略、工具代理、证据存储等），中小型企业可能面临较高的技术门槛和运营成本。

- **扩展性挑战**：框架假设智能体的“尝试操作”可被明确识别并插入拦截点。但在复杂多步骤、多代理协作场景中，如何定义“受保护的后果形成点”可能不清晰，框架未深入讨论。
