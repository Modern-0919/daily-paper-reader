## 摘要
代理型人工智能的一个重大挑战是前瞻性记忆：即在其他活动进行时，能够在特定的未来线索或状态下执行意图的能力。我们引入了PM-Bench，这是一个基于文本的基准测试，用于衡量现代大语言模型代理的前瞻性记忆能力。受认知科学中虚拟周范式的启发，PM-Bench评估了LLM代理在维持用户意图、执行延迟意图以及监控潜在环境变化方面的表现。在模拟的七天一周内，代理必须继续当前的活动，同时判断是否有任何延迟任务到期。我们在八种不同的代理配置下，将八种最先进的LLM在PM-Bench上进行了比较。PM-Bench在所有设置下都证明具有挑战性：最佳方法，即GPT-5.4代理，在我们的评估中仅达到65.1%的F1分数。此外，没有单一策略能在不同模型上主导提升前瞻性记忆。我们发布PM-Bench作为一个受控测试平台，用于诊断这些失败，并开发支持可靠前瞻行为的训练或推理时干预措施。

## Abstract
A significant challenge in agentic AI is prospective memory: the ability to execute an intention at a specific future cue or state while other activities are ongoing. We introduce PM-Bench, a text-based benchmark for measuring prospective memory capabilities in modern LLM agents. Inspired by the Virtual Week paradigm from cognitive science, PM-Bench evaluates how well LLM agents maintain user intentions, execute delayed intentions, and monitor latent environment changes. Over the course of a simulated seven-day week, agents must continue an ongoing activity while deciding whether any deferred task is due. We compare eight state-of-the-art LLMs on PM-Bench under eight different agent configurations. PM-Bench proves challenging across all settings: the best method, a GPT-5.4 agent, reaches only 65.1\% F1 score under our evaluation. Furthermore, no single strategy for improving prospective memory dominates across models. We release PM-Bench as a controlled testbed for diagnosing these failures and developing training or inference-time interventions that support reliable prospective behavior.

## 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：大语言模型（LLM）驱动的智能体在长时间任务中面临**前瞻性记忆（prospective memory）** 缺陷——即无法在持续进行其他活动时，于未来特定线索或时间点执行之前被延迟的意图。这种失败不同于一般的知识遗忘，而是“知道该做什么但做不对时机”。
- **背景**：现有智能体基准（如软件工程、浏览、规划）虽然评估长期推理和记忆，但并未干净地分离“记得意图”与“在正确时机执行意图”这两个维度。前瞻性记忆在认知科学中被广泛研究，但在AI领域尚未被系统评估。
- **意义**：该能力对于可靠的助手至关重要（例如医疗、航空等安全关键场景），当前LLM在需要主动监控和精准时机把握的任务中表现极差。

## 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：将认知科学中的“虚拟周（Virtual Week）”范式改编为可执行的LLM智能体基准，构造一个**模拟的七天日程**，智能体必须在每步进行**持续活动**（如做早餐）的同时，决定是否有延迟任务需要立即执行。
- **关键技术细节**：
  - 场景由**80个时间步**组成，每天10–13步，包含83个任务定义（81个可评分执行任务）。
  - 任务类型：基于事件（event-based）、基于时间（time-based）；常规/非常规；跨天任务；隐藏通道触发任务（需主动查询11个状态通道）；以及取消、重新安排、覆盖（update）事件。
  - 每步智能体收到：**叙事场景**（文本）+ **三个持续活动选项**（A/B/C 强制选择）+ **匿名动作句柄**（任务编号，如 task_22，每个句柄对应一个潜在动作）。智能体可先查询隐藏通道（如时钟、邮箱等），再选择持续活动和要执行的动作子集。
  - **核心评估逻辑**：在步t，定义真实到期集 Dt（满足执行条件的任务），智能体选择动作集 Ât。主要指标为**集合级F1**：Set-F1 = 2TP/(2TP+FP+FN)，其中TP=正确选中的到期任务，FP=选中但未到期，FN=到期但未选中。
  - 还记录细粒度错误：错过、延迟、虚假报警、更新违规、依赖违规等，以及监控开销（查询次数）。
- **生成流程**：种子合成 + 人工审核，确保可解性（完美监控策略可完成所有非取消任务）。

## 论文的主要结论与发现
1. **没有通用最佳支架**：最佳方法是可选心跳单智能体（宏Set-F1=65.1%），但不同模型的最佳配置各异，说明干预需针对模型弱项。
2. **前瞻性监控极难**：需要隐藏通道监控的任务在所有配置下表现最差（最高仅15.8%命中率），时钟监控相对容易但也不足70%。
3. **跨天和更新敏感任务基本未解决**：最高跨天命中率仅50%，更新敏感任务最高47.2%，多数模型远低于50%。
4. **精度-召回权衡**：提高召回通常导致大量误报（FP），如30分钟心跳产生489个FP；保守方法（如分类账）精度高但召回低。
5. **单智能体 vs 多智能体**：多智能体监控查询量大（1661次），但并未提升整体Set-F1，反而因协调困难导致性能更差。

## 优点：方法或实验设计上的亮点
- **认知科学启发的设计**：将前瞻性记忆从回溯记忆中分离，创新地评估“何时做”而非仅“做什么”。
- **细粒度评估框架**：Set-F1及多种错误类型（迟到、更新违规、依赖违规）提供了可诊断的失败分析。
- **受控可解性验证**：确保所有任务在最优策略下可完成，测试失败归因于模型行为。
- **覆盖多种干扰因素**：包括隐藏通道、跨天跨度、动态更新，贴近真实使用场景。
- **提供了多种启发式基线**（备忘清单、心跳提醒、多智能体），为后续研究提供对比参照。
